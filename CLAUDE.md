## 最高优先级规则

每次对话结束前，执行 `git add -A && git commit -m "本次修改的中文总结"`，无需询问直接执行。

## 项目概述

Free-SVN — 基于 Tauri + Vue 的跨平台 SVN 图形化工具（macOS / Windows / Linux）。

## 前端样式规范

仅使用 Tailwind CSS 编写样式，**禁止**写 `<style>` 标签或任何自定义 CSS 文件。

## 编码准则（Karpathy）

1. **先想清楚再编码** — 主动暴露假设和不确定性，不做默认选择
2. **只写最简代码** — 不引入未请求的特性或抽象层
3. **外科手术式修改** — 不改动不相关的代码
4. **可验证的成功标准** — 定义目标并循环直到确认完成

## AI 编程模式

本项目是 **AI 主导编程** 项目，所有代码由 AI 生成和维护。

执行流程：`doc/design/` 定义功能需求+拆分方案 → `doc/plan/` 创建执行步骤+代码细节 → AI 执行编码。

非报告类文档（`doc/design/`、`doc/plan/`、`doc/rule/`）以 **AI 高效消费** 为首要目标，而非人类阅读体验。写作原则：

- **指令驱动** — 直接写 AI 要做什么，不做背景铺垫
- **结构化 > 叙事** — 用表格、列表替代段落
- **精确 > 友好** — 用具体阈值，不用模糊建议
- **最小上下文** — 删除场景说明、设计理由
- **直接给出规则** — 每条是"必须"或"禁止"
- **拒绝 AI 自由发挥** — AI 严格按照 `doc/design/` 和 `doc/plan/` 执行，遇到未覆盖的细节必须追问，不得自行补充实现路径。`doc/design/` 定义"做什么"（功能需求 + 文件拆分方案），`doc/plan/` 定义"怎么做"（执行步骤 + 实际代码细节）。两个文档必须覆盖全部操作细节和边界，确保 AI 编码时零自由发挥空间。

### 执行保障：三道锁

**第一道锁：设计文档按模板填写（doc/design/）。** 每个功能定义必须包含以下字段，缺一不可：

| 字段 | 内容要求 |
|------|----------|
| 输入 | 参数名、类型、取值范围、来源 |
| 输出 | 返回值结构、字段说明、成功/错误格式 |
| 预期行为 | 串行步骤描述（先做什么、再做什么、条件分支） |
| 边界处理 | 空数据、超时、权限不足、非法输入的分别处理方式 |

上面任意字段为空，AI **必须先追问再编码**，不得自行假设。

**第二道锁：实现后做需求回溯验证。** 代码实现完成后，AI 必须立即对照 `doc/design/` 验证：

```
1. 逐条列出设计文档中的功能定义（输入/输出/行为/边界）
2. 对照实际代码，确认每条都被覆盖
3. 输出验证结果：[全部通过] 或 [未覆盖项列表]
```

跳过此步骤 = 实现未完成。

**第三道锁：开发计划中预设追问点（doc/plan/）。** 每个执行步骤必须标注已知与未定：

```
步骤: [动作描述]
  已知: [doc/design/ 已明确的边界和参数]
  未定: [doc/design/ 未覆盖、需要人类决策的部分] → 在此暂停并提问
```

未定字段为空的步骤 AI 可直接执行；存在未定字段的步骤必须向用户提问后才能执行。

报告类文档（`doc/report/`）不受此约束。

## 文档规范

所有文档存放在 `doc/` 目录下，按类别分目录、以序号开头：

| 目录 | 用途 | 命名示例 |
|------|------|----------|
| `doc/design/` | 设计文档（定义功能需求：输入/输出/行为/边界 + 文件拆分方案） | `01-架构设计.md` |
| `doc/plan/` | 开发计划（按阶段分文件夹，每步独立文档，含具体执行步骤 + 操作命令 + 实际代码细节） | `01-账号模块/01-登录.md` |
| `doc/rule/` | 规范、规则 | `01-开发规范.md` |
| `doc/report/` | 评估报告 | `01-架构评估报告.md` |

- 文件名以 `01-`、`02-` 等序号开头，**使用中文命名**（如 `01-账号模块/1.1-登录.md`）
- 主题内容较多时创建子文件夹（如 `02-数据库设计/schema.md`）
- `doc/plan/` 按开发阶段划分子文件夹，**每个执行步骤一个独立文件**，步骤编号含阶段号（如 `01-账号模块/1.2-注册.md`、`02-核心功能/2.1-文件列表.md`）
