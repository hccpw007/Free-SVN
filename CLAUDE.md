## 最高优先级规则

每次对话结束前，执行 `git add -A && git commit -m "本次修改的中文总结"`，无需询问直接执行。

## 项目概述

Free-SVN — 基于 Tauri + Vue 的跨平台 SVN 图形化工具（macOS / Windows / Linux）。

## 前端样式规范

仅使用 Tailwind CSS 编写样式，**禁止**写 `<style>` 标签或任何自定义 CSS 文件。

## 编码准则（Karpathy）

1. **先想清楚再编码** — 主动暴露假设和不确定性，不做默认选择
2. **只写最简代码** — 不引入未请求的特性或抽象层
3. **外科手术式修改** — 不改动不相关的代码
4. **可验证的成功标准** — 定义目标并循环直到确认完成

## AI 编程模式

本项目是 **AI 主导编程** 项目，所有代码由 AI 生成和维护。非报告类文档（`doc/design/`、`doc/plan/`、`doc/rule/`）以 **AI 高效消费** 为首要目标，而非人类阅读体验。写作原则：

- **指令驱动** — 直接写 AI 要做什么，不做背景铺垫
- **结构化 > 叙事** — 用表格、列表替代段落
- **精确 > 友好** — 用具体阈值，不用模糊建议
- **最小上下文** — 删除场景说明、设计理由
- **直接给出规则** — 每条是"必须"或"禁止"

报告类文档（`doc/report/`）不受此约束。

## 文档规范

所有文档存放在 `doc/` 目录下，按类别分目录、以序号开头：

| 目录 | 用途 | 命名示例 |
|------|------|----------|
| `doc/design/` | 项目设计文档 | `01-架构设计.md` |
| `doc/plan/` | 开发计划 | `01-开发计划.md` |
| `doc/rule/` | 规范、规则 | `01-开发规范.md` |
| `doc/report/` | 评估报告 | `01-架构评估报告.md` |

- 文件名以 `01-`、`02-` 等序号开头
- 主题内容较多时创建子文件夹（如 `02-数据库设计/schema.md`）
