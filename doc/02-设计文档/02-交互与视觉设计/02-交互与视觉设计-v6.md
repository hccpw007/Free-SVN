**认证输入区交互规则：**

| 元素 | 交互规则 |
|------|---------|
| 用户名 | 文本输入框，placeholder "输入 SVN 用户名" |
| 密码 | 密码输入框 `type="password"`，右侧 👁 按钮切换显示/隐藏，placeholder "输入密码" |
| 👁 显示/隐藏 | 点击切换图标状态：👁（隐藏密码）⇄ 👁‍🗨（显示密码），`aria-label` 随状态切换 |
| 保存凭据 | `<el-checkbox>`，文案"保存凭据到 ~/.subversion/auth/" |
| 分割线 | 水平线 `border-t border-slate-200 dark:border-slate-700`，中间灰色文字"认证信息（可选，留空走系统缓存凭据）" |
| 区域折叠 | 用户名/密码/保存凭据默认展开显示，不折叠（方便用户发现） |
| 校验逻辑 | 点击[检出]时：用户名为空 && 密码为空 → 走无凭据模式（`--non-interactive`）；用户名非空 && 密码为空 → [检出]按钮禁用（tooltip "请输入密码"）；用户名为空 && 密码非空 → [检出]按钮禁用（tooltip "请输入用户名"） |

**（v5 新增）认证重试对话框 — AuthDialog mode="retry"：**

```
┌─ 认证失败 ──────────────────────────────────────┐
│                                                    │
│  SVN 服务器需要认证                                │
│  仓库: svn://example.com/project                  │
│  操作: 更新到最新版本                              │
│                                                    │
│  用户名       [________________]                   │
│  密码         [________________]      [👁 显示]    │
│  ☐ 保存凭据到 ~/.subversion/auth/                  │
│                                                    │
│  ⚠️ 认证失败，请检查用户名和密码                     │
│                                                    │
│         [取消操作]          [重试]                  │
└────────────────────────────────────────────────────┘
```

**尺寸：** 宽度 440px，`<el-dialog>` 组件。

**视觉元素说明：**

| 元素 | 规范 |
|------|------|
| 标题 | "认证失败"，dialog title |
| 仓库信息 | `font-mono text-xs`，仓库 URL 显示在服务器说明下方 |
| 操作信息 | `text-xs text-slate-500`，显示失败的操作名称 |
| 错误提示 | 红色区域 `bg-red-50 dark:bg-red-900/20`，`text-red-600 dark:text-red-400`，带 `XCircle` 图标 |
| 成功提示 | 绿色区域 `bg-green-50 dark:bg-green-900/20`，`text-green-700 dark:text-green-400`，带 `CheckCircle` 图标 |
| 密码按钮 | 密码输入框右侧绝对定位，`text-slate-400 hover:text-slate-600` |

**交互规则：**

| 条件 | 行为 |
|------|------|
| 服务端操作返回 `SVN_AUTH_FAILED` | 暂停操作流程，弹出对话框，携带失败操作的信息（仓库 URL、操作类型、错误信息） |
| 用户输入凭据 → [重试] | 使用新凭据重新执行原始操作 |
| [重试]按钮状态 | 用户名和密码均非空时启用，否则禁用并 `opacity-50 cursor-not-allowed` |
| [重试]点击后 | 按钮进入 loading 态（spinner + "重试中..."），用户名/密码/保存复选框禁用 |
| 重试成功 | 状态提示区域显示绿色成功信息，0.8 秒后自动关闭对话框 |
| 重试失败（再次认证错误） | 保留对话框，状态提示区域更新红色错误信息 |
| [取消操作] | 关闭对话框，终止原始操作 |
| Esc 键 | 关闭对话框（同取消操作） |
| 遮罩点击 | 禁止关闭（因操作进行中） |
| 回车键 | 在用户名或密码输入框中按 Enter → 触发重试（等价于点击[重试]） |

```
┌─ 凭据管理 ──────────────────────────────────────┐
│                                                    │
│  当前工作副本                                      │
│  仓库: svn://example.com/project                  │
│                                                    │
│  ── 认证信息 ──                                    │
│  用户名    [________________]                      │
│  密码      [________________]       [👁 显示]      │
│  ☐ 更新 ~/.subversion/auth/ 缓存                   │
│                                                    │
│  [测试连接]  [清除缓存]               [关闭]       │
│                                                    │
└────────────────────────────────────────────────────┘
```

**尺寸：** 宽度 480px，`<el-dialog>` 组件。

**视觉元素说明：**

| 元素 | 规范 |
|------|------|
| 标题 | "凭据管理"，dialog title |
| 仓库信息 | 与 retry 模式同，`font-mono text-xs` |
| 底部按钮区 | 左右分布：左侧 [测试连接] [清除缓存] ← → 右侧 [关闭] |
| [测试连接] | `<el-button size="small">`，次要按钮样式（outline） |
| [清除缓存] | `<el-button size="small">`，次要按钮样式 |
| [关闭] | `<el-button>`，默认样式 |

**交互规则：**

| 操作 | 行为 | 视觉反馈 |
|------|------|---------|
| [测试连接] | 调用后端 `test_connection` → 执行 `svn info --non-interactive --username xxx --password-from-stdin` | 按钮 loading → 状态区域显示成功/失败信息 |
| [清除缓存] | 删除 `~/.subversion/auth/svn.simple/` 中匹配当前仓库 realm 的文件 | 按钮 loading → 状态区域显示"已清除 N 个凭据缓存文件" |
| [关闭] | 仅关闭对话框，输入的凭据保留在会话凭据池中 | 正常关闭 |
| 用户名/密码空时 [测试连接] | 禁用，tooltip "请输入用户名和密码" | `opacity-50 cursor-not-allowed` |

**状态提示区域（两种模式共用）：**
- 位于凭据输入区下方、按钮区上方
- 成功时：绿色背景 `bg-green-50 dark:bg-green-900/20` + `CheckCircle` 图标
- 失败时：红色背景 `bg-red-50 dark:bg-red-900/20` + `XCircle` 图标
- 无状态时：隐藏（`v-if="statusMessage"`）

## 4. 交互规范

### 4.1 工具栏按钮控制

### 4.2 操作反馈（v5 新增认证相关反馈）

在 v4 基础上增加以下反馈：

| 触发 | 反馈形式 | 持续时间 | 样式 |
|------|---------|----------|------|
| **认证失败（服务端操作）** | **弹认证重试对话框** | **用户交互后关闭** | **`AuthDialog` mode="retry"，非 toast** |
| **凭据保存成功** | toast | 3s 自动关闭 | green-500 |
| **凭据保存失败** | toast | 持续，手动关闭 | red-500 |
| **测试连接成功** | 对话框内状态提示 | 显示到下次操作或关闭 | 绿色内嵌提示 |
| **测试连接失败** | 对话框内状态提示 | 显示到下次操作或关闭 | 红色内嵌提示 |
| **清除缓存完成** | 对话框内状态提示 | 显示到下次操作或关闭 | 绿色内嵌提示 |
| **清除缓存失败** | toast | 持续，手动关闭 | red-500 |

**认证失败的特殊处理（区别于普通错误）：**

- 所有服务端操作（update/commit/log/diff/switch/merge/checkout/export 等）遇到认证错误时，**不显示 toast**，而是弹出 AuthDialog（`mode="retry"`）
- 重试成功 → 自动继续操作流程 → 显示操作原本的成功/失败反馈
- 取消 → 操作视为已取消，不额外显示反馈

### 4.4 键盘快捷键（v5 新增认证相关）
