# 操作进度提示设计 v1

## 1. 概述

SVN 服务端文件传输操作（检出/更新/提交/切换/合并/导出）需要一个**非模态信息弹窗**，实时展示传输进度、速度、文件列表和取消功能。

## 2. 设计原则

- **非模态**：弹窗不阻塞应用其他功能操作（查看日志、差异等）
- **仅禁用工作空间切换**：弹窗打开期间不允许切换工作空间
- **信息清晰**：三种状态分明的文件展示
- **拖拽定位**：用户可拖拽标题栏调整弹窗位置

## 3. 适用范围

| 操作 | SVN 命令 | 文件信息来源 |
|------|----------|------------|
| 检出 | `svn checkout` | stdout：`A    path/to/file` |
| 更新 | `svn update` | stdout：`U/A/D path/to/file` |
| 提交 | `svn commit` | stdout：`Sending    path/to/file` → `Committed revision N` |
| 切换 | `svn switch` | stdout：类似 update 的 `U/A/D` 行 |
| 合并 | `svn merge` | stdout：类似 update 的 `U/A/D` + 冲突行 |
| 导出 | `svn export` | stdout：类似 checkout 的 `A    path/to/file` |

## 4. 弹窗布局

### 4.1 整体结构（从上到下）

```
┌─ 检出进行中 (85%) ────────────────────── 可拖拽标题栏
│  ████████████████████████████░░░░  85%   进度条
│  ⚡ 2.3 MB/s  │  已完成 47 个  │  待传输 ~167 个  │ 00:12  统计栏
├──────────────────────────────────────────┤
│  ✓  src/main.rs                  暗淡     │
│  ✓  src/lib.rs                   暗淡     │
│  ✓  src/components/App.tsx       暗淡     │
│  ⋯  src/components/Sidebar.tsx   高亮     │  文件滚动区
│  ⋯  src/components/Navbar.tsx    高亮     │  （深色终端风格）
│  ▶  (等待中，167 个文件)          正常     │
│  ▶  (等待中)                     正常     │
│  ▶  (等待中)                     正常     │
├──────────────────────────────────────────┤
│  操作完成后自动关闭              [取消]   │  底栏
└──────────────────────────────────────────┘
```

### 4.2 各模块详细说明

#### 标题栏
- 内容格式：`{操作名}进行中 ({百分比}%)`
- 可拖拽整窗移动
- 关闭按钮：操作完成后自动关闭，运行中不可手动关闭

#### 进度条
- 绿色渐变（#22c55e → #16a34a）
- 百分比右侧数字
- 百分比来源：SVN 命令行输出的百分比行 `  45%  1234K  1.2MB/s`
- 若无百分比行，则根据已传输文件数/估算总数推算

#### 统计栏
- 四项信息横排：传输速度 / 已完成文件数 / 已耗时间 / 待传输估算数量
- 传输速度：从 SVN stderr 行解析（`  45%  1234K  1.2MB/s` 中的速度信息）
- 待传输估算：取最近 N 个文件的到达间隔的平均时间，除以总估算时间得出

#### 文件滚动区

**三种文件状态：**

| 状态 | 图标 | 深色模式 | 浅色模式 |
|------|------|---------|---------|
| 已完成 | ✓ | 暗淡灰色（text-slate-600） | 暗淡灰色（text-slate-400） |
| 进行中 | ⋯ | 黄色高亮（text-amber-400）+ 背景高亮（bg-slate-700） | 橙色高亮（text-amber-600），无背景色 |
| 待传输 | ▶ | 正常色（text-slate-300） | 正常色（text-slate-600） |

- 深色模式背景 bg-slate-900，浅色模式背景 bg-white
- 等宽字体（font-mono）
- 通过 Tailwind `dark:` 前缀切换

#### 底栏
- 左侧文字提示："操作完成后自动关闭"
- 右侧取消按钮

## 5. 技术方案

### 5.1 后端改造

**核心改造**：`src-tauri/src/svn/executor.rs` 在现有 `run_svn_sync` 实现基础上增加**逐行读取 stdout/stderr** 的进度函数。

#### 改造方案

新增 `run_svn_with_progress()` 函数（保留原 `run_svn` 以保证向后兼容）：

```rust
async fn run_svn_with_progress(
    args: &[&str],
    cwd: &str,
    credentials: Option<&SvnCredentials>,
    app_handle: AppHandle,
    operation: &str,
) -> Result<String, AppError>
```

执行流程：
1. `Command::new(svn_path)` + `stdout(Stdio::piped())` + `stderr(Stdio::piped())`
2. `BufReader::new(reader)` 逐行读取 stdout
3. 每行解析后通过 `app_handle.emit("operation:line", payload)` 推送
4. 同时逐行读取 stderr（并行线程）提取速度/百分比信息
5. 整体完成后推送 `operation:completed`

#### 进度事件 payload 定义

```typescript
// 新增事件: operation:line — 每行 SVN 输出
interface OperationLine {
  operation: string    // "checkout" | "update" | "commit" | ...
  line: string         // 原始输出行（如 "A    src/main.rs"）
  status: 'completed' | 'in_progress'
  // completed: 该行已处理完毕
  // in_progress: 当前行正在传输
}

// 增强 operation:progress — 进度更新
interface OperationProgress {
  operation: string
  percent: number       // 0-100
  completedCount: number // 已完成文件数
  pendingCount: number   // 待传输估算数
  speed: string         // "2.3 MB/s"
  elapsed: string       // "00:00:12"
  currentLines: string[] // 当前进行中的文件路径（可能并行）
}
```

#### 各操作的行解析规则

| 操作 | 匹配模式 | 文件路径提取 |
|------|---------|------------|
| checkout | `^(A)\s+(.+)$` | 第 2 组 |
| update | `^(U\|A\|D)\s+(.+)$` | 第 2 组 |
| commit | `^Sending\s+(.+)$` | 第 1 组 |
| switch | `^(U\|A\|D)\s+(.+)$` | 第 2 组 |
| merge | `^(U\|A\|D\|C)\s+(.+)$` | 第 2 组 |
| export | `^(A)\s+(.+)$` | 第 2 组 |

### 5.3 前端改造

#### 新增组件：`ProgressOverlay.vue`

监听 `operation:started`/`operation:line`/`operation:progress`/`operation:completed`/`operation:error` 事件。

数据流：
```
Tauri Event → svnStore.progress / svnStore.lines → ProgressOverlay 渲染
```

组件定位：挂载在 `MainLayout.vue` 中，位于 RouterView 上方。

### 5.5 工作空间切换锁定

- `workspaceStore.switchWorkspace()` 在 `svnStore.showProgress === true` 时直接 return
- 工具栏"切换工作区"按钮在 `svnStore.showProgress` 时 disabled

## 6. 边界情况

| 场景 | 处理方式 |
|------|---------|
| 操作失败 | 推送 `operation:error`，弹窗显示错误信息和关闭按钮 |
| 用户取消 | 调用 `cancel_operation`，显示"操作已取消"，弹窗可关闭 |
| 大量文件（万级） | 文件列表最多保留 1000 行，超出后丢弃最旧的已完成行 |
| 文件路径过长 | 省略中间部分，hover 显示完整路径 |
| 统计栏数字显示 | 超过 1000 显示 "1.2k"，超过 1000000 显示 "1.2M" |
| 操作瞬间完成 | 进度条动画 300ms 确保用户看到 "100%" 的视觉反馈 |
