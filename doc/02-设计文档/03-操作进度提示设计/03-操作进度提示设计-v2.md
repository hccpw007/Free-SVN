# 操作进度提示设计 v2

## 1. 概述

SVN 服务端文件传输操作（检出/更新/提交/切换/合并/导出）需要一个**非模态信息弹窗**，实时展示传输进度、速度、文件列表和取消功能。

v2 修订内容：根据评估报告（10-操作进度提示设计评估报告-v1）全面修订，涉及事件协议协作关系、commit Transmitting file data 阶段、后端双管道线程方案、取消竞争条件、Lucide 图标规范、非模态实现方式、边界情况补充等 25 项改进。

## 2. 设计原则

- **非模态**：弹窗不阻塞应用其他功能操作（查看日志、差异等）
- **仅禁用工作空间切换**：弹窗打开期间不允许切换工作空间
- **信息清晰**：三种状态分明的文件展示
- **拖拽定位**：用户可拖拽标题栏调整弹窗位置

## 3. 适用范围

| 操作 | SVN 命令 | 文件信息来源 | 进度行来源 |
|------|----------|------------|-----------|
| 检出 | `svn checkout` | stdout：`A    path/to/file` | stderr：`  22%   567K   1.2MB/s` |
| 更新 | `svn update` | stdout：`U/A/D path/to/file` | stderr：同上格式 |
| 提交 | `svn commit` | stdout：`Sending    path/to/file` → `Committed revision N` | stdout 行 + stderr Transmitting：`Transmitting file data .` → `   ...  45%` |
| 切换 | `svn switch` | stdout：类似 update 的 `U/A/D` 行 | stderr：同上格式 |
| 合并 | `svn merge` | stdout：类似 update 的 `U/A/D` + 冲突行 | stderr：同上格式 |
| 导出 | `svn export` | stdout：类似 checkout 的 `A    path/to/file` | stderr：同上格式 |

commit 操作的特殊阶段：在 Sending 阶段之后进入 Transmitting file data 阶段，此时输出 `Transmitting file data ...` 行，后续的 `   ...  45%` 行包含百分比进度（正则：`^\s*\.\.\.\s(\d+)%\s`）。此阶段的行不计入文件列表，仅用于更新进度百分比。

### 4.2 各模块详细说明

#### 标题栏
- 内容格式：`{操作名}进行中 ({百分比}%)`
- 可拖拽整窗移动
- 关闭按钮：操作完成后自动关闭，运行中不可手动关闭
- 拖拽约束：面板不可拖出视口范围；默认位置居中偏上；位置信息不持久化存储

#### 进度条
- 绿色渐变（#22c55e → #16a34a）
- 百分比右侧数字
- 百分比来源：
  - 稳定来源：SVN stderr 的百分比行 `  45%  1234K  1.2MB/s`
  - commit Transmitting file data 阶段：`   ...  45%` 行
  - 回退估算：若无百分比行，根据已传输文件数/估算总数推算
- 操作瞬间完成（单次输出即结束）：从 0% → 100% 以 300ms 过渡动画展现，完成后延迟 500ms 自动关闭弹窗

#### 统计栏
- 四项信息横排：传输速度 / 已完成文件数 / 已耗时间 / 待传输估算数量
- 传输速度：从 SVN stderr 行解析（完整正则：`...\s+(\d+K?)\s+([0-9.]+ (KB/s|MB/s))\s+(\S+)`）
- 待传输估算：取最近 N（10-20）个文件的到达间隔的平均时间，除以总估算时间得出。首次估算在收到第 10 条文件行后触发
- 响应式：窄屏（< 640px）时自动折叠为两行（第一行：已完成数 + 待传输估算；第二行：速度 + 耗时）
- 数字格式化：超过 1000 显示 "1.2k"，超过 1000000 显示 "1.2M"

#### 文件滚动区

**三种文件状态：**

| 状态 | 图标（Lucide） | 深色模式 | 浅色模式 |
|------|---------------|---------|---------|
| 已完成 | `Check`（◆） | 暗淡灰色（text-slate-600） | 暗淡灰色（text-slate-400） |
| 进行中 | `Loader`（⟳，旋转动画） | 黄色高亮（text-amber-400）+ 背景高亮（bg-slate-700） | 橙色高亮（text-amber-600），无背景色 |
| 待传输 | `MoveRight`（▸） | 正常色（text-slate-300） | 正常色（text-slate-600） |

- 文件路径过长处理：前 20 字符 + `...` + 后 15 字符，hover 时 tooltip 显示完整路径
- 已完成路径用 `line-through` 删除线表示结束
- 深色模式背景 bg-slate-900，浅色模式背景 bg-white
- 等宽字体（font-mono）
- 通过 Tailwind `dark:` 前缀切换

> **设计例外说明**：文件区域使用深色终端背景色 `bg-slate-900`（深色模式）/ `bg-slate-950`（深色模式增强）是刻意为之，意在模拟终端操作的真实视觉体验。这与项目整体标准背景色 `bg-slate-800` 不一致，作为设计例外此处保留。

#### 底栏
- 左侧文字提示："操作完成后自动关闭"
- 右侧取消按钮

取消按钮交互流程：
1. 用户点击取消 → 按钮变为 loading 状态，文案变为 "正在取消中..."
2. 后端终止 SVN 进程 → 推送 `operation:cancelled` 事件
3. 弹窗显示"操作已取消"状态 → 500ms 后显示成功 toast "操作已取消"
4. 弹窗变为可关闭状态（显示关闭按钮）

## 5. 技术方案

### 5.1 后端改造

**核心改造**：`src-tauri/src/svn/executor.rs` 在现有 `run_svn_sync` 实现基础上增加**双管道逐行读取 stdout + stderr** 的进度函数。

#### 双管道读取策略

使用两个独立线程分别读取 stdout 和 stderr，通过 channel 同步：

```
stdout 线程 ──→ line channel ──→ 解析文件行 → emit("operation:line")
                                   ↓
stderr 线程 ──→ progress channel ──→ 解析百分比/速度 → emit("operation:progress")
```

- stdout 线程：`BufReader::new(stdout)` 逐行读取，匹配文件操作行（`A/U/D/Sending...`），通过 channel 发送到主处理循环
- stderr 线程：`BufReader::new(stderr)` 逐行读取，匹配进度行（百分比/速度），通过 channel 发送到主处理循环
- 两个线程通过独立的 `mpsc::channel` 与主线程同步

#### 取消机制与竞争条件处理

改用 `Arc<Mutex>` 在线程间共享取消标志和子进程句柄：

```rust
struct SharedState {
    cancelled: bool,
    child: Option<Child>,
}
let shared = Arc::new(Mutex::new(SharedState { cancelled: false, child: None }));
```

- stdout/stderr 两线程每次读取一行前检查 `shared.cancelled`，若为 true 则终止读取
- 取消操作：设置 `cancelled = true`，然后 kill 子进程
- 文件解析循环中每行也检查 CANCELLED 标志，避免在取消后继续处理

#### 新增函数

```rust
async fn run_svn_with_progress(
    args: &[&str],
    cwd: &str,
    credentials: Option<&SvnCredentials>,
    app_handle: AppHandle,
    operation: &str,
) -> Result<String, AppError>
```

执行流程：
1. `Command::new(svn_path)` + `stdout(Stdio::piped())` + `stderr(Stdio::piped())`
2. 启动两个异步线程分别读取 stdout 和 stderr
3. 每解析一行 stdout 后通过 `app_handle.emit("operation:line", payload)` 推送
4. 每解析一行 stderr 后通过 `app_handle.emit("operation:progress", payload)` 推送（带节流，不超过 200ms 推送一次）
5. 整体完成后推送 `operation:completed` 或 `operation:error`

**事件生命周期**：
```
operation:started → (operation:line × N 并行 + operation:progress 节流) → operation:completed
                                                                           → operation:error
                                                                           → operation:cancelled
```

说明：`operation:line` 是新增事件，与现有的 `operation:started` / `operation:completed` / `operation:error` **共存**而非替换。步骤中的 `operation:line` 携带单行文件信息，节流的 `operation:progress` 携带整体进度。现有只使用 `operation:started`/`operation:completed`/`operation:error` 的旧流程不受影响。

#### 错误处理与恢复

- **stderr 线程出错**：stderr 线程若因管道损坏异常退出，主处理循环检测到后降级运行——进度百分比回退到按文件数估算模式（已传输/已传输+待传输估算）。速度显示改为 "计算中..."。操作继续执行，不中断用户操作。
- **管道损坏恢复**：若 stdout 或 stderr 管道意外关闭（如 SVN 进程异常退出），使用 `try_wait()` 检查子进程状态。若子进程已退出但未返回完整结果，推送异常退出错误。
- **双线程 panic 安全**：任一线程 panic 不会导致另一线程挂起。通过 `catch_unwind` 包装线程体，panic 时关闭对应 channel 让主循环自然结束。

#### AppHandle 兼容性矩阵

当前各 command 是否持有 AppHandle：

| Command | 当前持有 AppHandle？ | 需追加？ |
|---------|-------------------|---------|
| `checkout_repo` | 否 | 是（添加参数） |
| `update_workspace` | 否 | 是（添加参数） |
| `create_commit` | 否 | 是（添加参数） |
| `merge_branch` | 否 | 是（添加参数） |
| `switch_branch` | 否 | 是（添加参数） |

#### 速度数据提取正则

SVN 进度行格式：`  45%   1234K   1.2MB/s   00:12`

完整正则匹配模式：
```
...\s+(\d+K?)\s+([0-9.]+ (KB/s|MB/s))\s+(\S+)
```

分组说明：
- 组 1：传输字节数（如 `1234K`）—— `...` 前缀前有至少三个空格
- 组 2 + 组 3：速度（如 `1.2 MB/s`）
- 组 4：剩余时间（如 `00:12`）

commit Transmitting file data 阶段进度正则：
```
^\s*\.\.\.\s(\d+)%\s
```
匹配 `   ...  45%` 格式，提取组 1 为百分比值。

#### 文件列表淘汰策略

当 `fileLines` 数量超过 1000 时，在一次批量操作中淘汰最旧的 200 行已完成（status=completed）行。不能逐行淘汰以避免 DOM 频繁更新。

#### 各操作的行解析规则

| 操作 | 匹配模式 | 文件路径提取 |
|------|---------|------------|
| checkout | `^(A)\s+(.+)$` | 第 2 组 |
| update | `^(U\|A\|D)\s+(.+)$` | 第 2 组 |
| commit | `^Sending\s+(.+)$` | 第 1 组 |
| switch | `^(U\|A\|D)\s+(.+)$` | 第 2 组 |
| merge | `^(U\|A\|D\|C)\s+(.+)$` | 第 2 组 |
| export | `^(A)\s+(.+)$` | 第 2 组 |

### 5.3 前端改造

#### 新增组件：`ProgressOverlay.vue`

> 命名说明：项目中已有一个功能不同的 `ProgressOverlay.vue` 组件，该组件改名为 `ProgressPanel.vue`。新的进度弹窗组件使用 `ProgressOverlay.vue` 名称。

监听 `operation:started` / `operation:line` / `operation:progress` / `operation:completed` / `operation:error` / `operation:cancelled` 事件。

**组件挂载方式**：使用 `<Teleport to="body">` 将组件渲染到 body 最顶层，确保覆盖在所有其他 UI 之上。

**非模态实现**：
- 最外层容器：`fixed inset-0 pointer-events-none`（点击穿透，不阻塞其他操作）
- 内层拖拽面板：`pointer-events-auto`（仅在面板区域内可交互）
- 不使用 `<el-dialog>` 等模态组件

**数据流**：
```
Tauri Event → useProgressOverlay composable → svnStore.progress / svnStore.lines → ProgressOverlay 渲染
```

#### 图标使用

严格遵循项目规范，使用 Lucide 图标替代 Unicode 符号：

| 状态 | 原 Unicode | Lucide 图标组件 |
|------|-----------|----------------|
| 已完成 | ✓ | `<Check class="..." />` |
| 进行中 | ⋯ | `<Loader class="animate-spin ..." />` |
| 待传输 | ▶ | `<MoveRight class="..." />` |

#### 路径省略

```typescript
function truncatePath(path: string): string {
  if (path.length <= 38) return path
  return path.slice(0, 20) + '...' + path.slice(-15)

```

hover 时通过 `title` 属性或 Element Plus `el-tooltip` 显示完整路径。

### 5.5 事件协议前后端同步

`OperationProgress` 结构体在 Rust 和 TypeScript 两端必须完全一致：

**Rust 端**（`src-tauri/src/svn/types.rs`）：

```rust
#[derive(Clone, Serialize)]
pub struct OperationProgress {
    pub operation: String,
    pub percent: f64,
    pub completed_count: i32,
    pub pending_count: i32,
    pub speed: String,
    pub elapsed: String,
    pub current_lines: Vec<String>,
}
```

**TypeScript 端**（`src/types/svn.ts`）：

```typescript
interface OperationProgress {
  operation: string
  percent: number
  completedCount: number
  pendingCount: number
  speed: string
  elapsed: string
  currentLines: string[]
}
```

> 注意：Rust 端使用 `snake_case`（`completed_count`），通过 `#[serde(rename_all = "camelCase")]` 自动转换为 TypeScript 的 `camelCase`。确保两端字段一一对应，顺序无关。

## 6. 边界情况

| 场景 | 处理方式 |
|------|---------|
| 操作失败 | 推送 `operation:error`，弹窗显示错误信息和关闭按钮 |
| 用户取消 — 取消后反馈 | 点击取消 → loading + "正在取消中..." → 后端终止进程 → `operation:cancelled` → 弹窗显示"操作已取消" → 500ms 后显示成功 toast "操作已取消"，弹窗可关闭 |
| 大量文件（万级） | 文件列表最多保留 1000 行，超过时一次性淘汰最旧的 200 行已完成行 |
| 文件路径过长 | 前 20 字符 + `...` + 后 15 字符，hover tooltip 显示完整路径 |
| 统计栏数字显示 | 超过 1000 显示 "1.2k"，超过 1000000 显示 "1.2M" |
| 操作瞬间完成 | 进度条 0% → 100% 以 300ms 过渡动画，完成后延迟 500ms 自动关闭弹窗 |
| 10 秒无响应 | 从首次操作启动开始计时，若连续 10 秒未收到任何输出行（包括 stdout 和 stderr），状态栏显示"正在连接服务器..."，进度条为不确定状态 |
| 认证失败 + 进度弹窗 | 进度弹窗保持打开（不关闭），AuthDialog 模态弹窗叠加在进度弹窗上方。认证完成后进度弹窗继续更新，认证取消时进度弹窗也关闭并显示已取消 toast |
| 网络中断恢复 | SVN 命令超时（60s）后推送 `operation:error`，弹窗显示错误信息，用户可重试 |
