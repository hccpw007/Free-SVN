# 02-交互与视觉设计

> Free-SVN 的第一版 UI/UX 设计规范，与 `01-业务功能设计-v2.md` 配套使用

## 1. 设计系统

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **清晰优先** | 信息层级明确，操作意图一目了然。开发者工具第一要义是效率，不是装饰 |
| **技术支持** | 采用开发者熟悉的视觉语言（终端绿、代码字体、暗色模式）传递技术气质 |
| **克制** | 不过度设计。文件列表、差异对比、提交面板——让数据本身成为界面主角 |
| **一致性** | 跨页面使用统一组件、颜色、间距、动效规范 |

### 1.5 间距系统

采用 4px 基准递增系统：

| Token | 值 | 用法 |
|-------|-----|------|
| `space-1` | 4px | 极微间距 |
| `space-2` | 8px | 元素内间距、图标与文字间隙 |
| `space-3` | 12px | 按钮内边距 |
| `space-4` | 16px | 列表行间距、卡牌内边距 |
| `space-5` | 20px | 分组间距 |
| `space-6` | 24px | 区块间距 |
| `space-8` | 32px | 页面级间距 |

### 1.6 圆角与阴影

| Token | 值 | 用法 |
|-------|-----|------|
| `radius-sm` | 4px | 输入框、小型控件 |
| `radius-md` | 6px | 按钮、对话框 |
| `radius-lg` | 8px | 卡片、面板 |
| `shadow-sm` | `0 1px 2px rgba(0,0,0,0.05)` | 卡片阴影（亮色）|
| `shadow-md` | `0 4px 6px rgba(0,0,0,0.07)` | 对话框阴影 |
| `shadow-lg` | `0 10px 25px rgba(0,0,0,0.12)` | 模态遮罩层 |

暗色模式下阴影减淡：`rgba(0,0,0,0.3)` 至 `rgba(0,0,0,0.5)`。

### 1.7 图标风格

- 使用 **Lucide** 图标库（与 Tailwind CSS 生态一致）
- 统一 1.5px 描边宽度
- 图标尺寸：16px（内联）/ 20px（按钮）/ 24px（状态指示）
- **禁止使用 emoji 作为功能图标**

### 1.8 动画基础

| 场景 | 持续时间 | 缓动函数 | 说明 |
|------|---------|----------|------|
| 悬停状态 | 150ms | ease-out | 按钮、行、操作项的 hover |
| 面板展开/收起 | 200ms | ease-in-out | Diff 面板、侧面板 |
| 对话框出现 | 200ms | ease-out + scale(0.95→1) | 模态对话框 |
| toast 提示 | 300ms | ease-out | 操作反馈 |
| 列表刷新 | 150ms | ease-out | 文件列表行更新 |
| 进度动画 | 持续 | linear | 进度条 |

## 2. 整体布局架构

### 2.1 MainLayout 结构

```
┌──────────────────────────────────────────────────────────┐
│  TopBar (48px) - 路径/切换/分支             暗色切换 ⚙️  │
├──────────────────────────────────────────────────────────┤
│  Toolbar (40px) - 按钮组                                  │
│  提交  更新  日志  切换  分支/标签  合并  清理  导出     │
├──────────────────────────────────────────────────────────┤
│                                                          │
│               <RouterView /> (灵活高度)                    │
│                                                          │
│                                                          │
├──────────────────────────────────────────────────────────┤
│  StatusBar (28px) - 版本号 | 最新提交 | SVN 版本        │
└──────────────────────────────────────────────────────────┘
```

- TopBar、Toolbar、StatusBar 均固定在 MainLayout 中
- `<RouterView />` 区域高度自动填充剩余空间
- 所有页面（settings 除外）共享此布局

|------|--------|--------|
| 窗口宽度 | 800px | 1100-1400px |
| 窗口高度 | 500px | 700-900px |

## 5. 暗色模式

### 5.1 策略

- **默认跟随系统**：通过 CSS `prefers-color-scheme` media query
- **手动切换**：顶部状态栏右侧提供暗色/亮色切换按钮
- **持久化**：用户选择存入 `tauri-plugin-store`，覆盖系统默认
- **实现方式**：Tailwind CSS `dark:` 变体 + CSS 变量

### 5.2 实现思路

```css
/* 使用 Tailwind dark 变体 */
.dark .card {
  @apply bg-slate-800 border-slate-700;
}

/* 或使用 CSS 变量双主题 */
:root {
  --bg-primary: #F8FAFC;
  --text-primary: #0F172A;
}
.dark {
  --bg-primary: #0F172A;
  --text-primary: #F8FAFC;
}
```

### 5.3 暗色特殊处理

| 元素 | 暗色适配 |
|------|---------|
| Diff 删除行 | 透明度 `bg-red-900/30` 替代红色底色 |
| Diff 新增行 | 透明度 `bg-green-900/30` 替代绿色底色 |
| 对话框遮罩 | `rgba(0,0,0,0.6)` 更深以突出前景 |
| 阴影 | 暗色下几乎不可见，使用边框替代 |
| 状态图标颜色 | 调亮一级（见 §1.3 表格）|
| 输入框 | `bg-slate-800 border-slate-600` |

## 6. 无障碍设计

| 要求 | 实现 |
|------|------|
| 对比度 | 文字 4.5:1 最小对比度（亮色/暗色均满足）|
| 键盘导航 | 所有操作均支持键盘完成（见 §4.4 快捷键）|
| 焦点指示 | 2px blue-400 outline 在所有可交互元素上 |
| 颜色非唯一指示 | 文件状态使用颜色 + 图标双通道传递 |
| 屏幕阅读器 | `<el-table>` 默认支持 ARIA；图标按钮加 `aria-label` |
| 动画尊重 | `prefers-reduced-motion` 时禁用非必要动画 |
| 文本缩放 | 支持浏览器/系统字体缩放（使用 rem/em 而非 px）|
