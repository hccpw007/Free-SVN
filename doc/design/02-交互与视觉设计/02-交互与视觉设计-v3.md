# 02-交互与视觉设计

> Free-SVN 的 UI/UX 设计规范 v3，与 `01-业务功能设计-v2.md` 配套使用

## 1. 设计系统

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **清晰优先** | 信息层级明确，操作意图一目了然。开发者工具第一要义是效率，不是装饰 |
| **技术支持** | 采用开发者熟悉的视觉语言（终端绿、代码字体、暗色模式）传递技术气质 |
| **克制** | 不过度设计。文件列表、差异对比、提交面板——让数据本身成为界面主角 |
| **一致性** | 跨页面使用统一组件、颜色、间距、动效规范 |

### 1.2 色彩体系

### 1.4 字体排版

| 层次 | 字重 | Tailwind 类 | fallback 栈 | 行高 | 用法 |
|------|------|-------------|-------------|------|------|
| Display | `Inter 700` | `text-3xl font-bold` | `'Inter', system-ui, -apple-system, sans-serif` | 1.3 | 欢迎页标题 |
| H1 | `Inter 600` | `text-xl font-semibold` | 同上 | 1.4 | 页面标题 |
| H2 | `Inter 600` | `text-base font-semibold` | 同上 | 1.4 | 面板标题 |
| Body | `Inter 400` | `text-sm` | 同上 | 1.5 | 正文、文件列表内容 |
| Body Small | `Inter 400` | `text-sm` | 同上 | 1.5 | 次要文字、状态栏 |
| Mono | `JetBrains Mono 400` | `text-sm font-mono` | `'JetBrains Mono', 'Fira Code', 'Cascadia Code', 'Consolas', monospace` | 1.5 | 文件路径、diff 代码、版本号 |
| Mono Small | `JetBrains Mono 400` | `text-xs font-mono` | 同上 | 1.4 | diff 行内文字 |
| Label | `Inter 500` | `text-sm font-medium` | `'Inter', system-ui, -apple-system, sans-serif` | 1.4 | 按钮文字、标签 |
| Tooltip | `Inter 400` | `text-xs` | 同上 | 1.3 | 工具提示 |

**字体加载策略：**

因 Free-SVN 是 Tauri 桌面应用（可能离线使用），**不采用 Google Fonts CDN `@import` 方式加载字体**。改为以下方案：

- **Inter 字体**：使用系统字体栈 `system-ui, -apple-system, 'Segoe UI', Roboto, sans-serif` —— macOS / Windows / Linux 三平台均有高质量系统界面字体，无需额外加载
- **JetBrains Mono 字体**：将字体文件打包到 Tauri 资源中（`resources/fonts/JetBrainsMono-Regular.ttf`），在 `src/style.css` 中通过 `@font-face` 加载：

```css
@font-face {
  font-family: 'JetBrains Mono';
  src: url('/fonts/JetBrainsMono-Regular.woff2') format('woff2');
  font-weight: 400;
  font-style: normal;
}
@font-face {
  font-family: 'JetBrains Mono';
  src: url('/fonts/JetBrainsMono-Medium.woff2') format('woff2');
  font-weight: 500;
  font-style: normal;
}
```

Tailwind 的 `font-sans` 和 `font-mono` 已在 Tailwind 配置中映射为上述字体栈。

### 1.5 间距系统

采用 4px 基准递增系统，使用 Tailwind 内置间距类：

| Token | 值 | Tailwind 类 | 用法 |
|-------|-----|-------------|------|
| `space-1` | 4px | `p-1` `gap-1` | 极微间距 |
| `space-2` | 8px | `p-2` `gap-2` | 元素内间距、图标与文字间隙 |
| `space-3` | 12px | `p-3` `gap-3` | 按钮内边距 |
| `space-4` | 16px | `p-4` `gap-4` | 列表行间距、卡牌内边距 |
| `space-5` | 20px | `p-5` `gap-5` | 分组间距 |
| `space-6` | 24px | `p-6` `gap-6` | 区块间距 |
| `space-8` | 32px | `p-8` `gap-8` | 页面级间距 |

### 1.6 圆角与阴影

| Token | 值 | Tailwind 类 | 用法 |
|-------|-----|-------------|------|
| `radius-sm` | 4px | `rounded-sm` | 输入框、小型控件 |
| `radius-md` | 6px | `rounded-md` | 按钮、对话框 |
| `radius-lg` | 8px | `rounded-lg` | 卡片、面板 |
| `shadow-sm` | `0 1px 2px rgba(0,0,0,0.05)` | `shadow-sm` | 卡片阴影（亮色）|
| `shadow-md` | `0 4px 6px rgba(0,0,0,0.07)` | `shadow-md` | 对话框阴影 |
| `shadow-lg` | `0 10px 25px rgba(0,0,0,0.12)` | `shadow-lg` | 模态遮罩层 |

暗色模式下阴影减淡：`rgba(0,0,0,0.3)` 至 `rgba(0,0,0,0.5)`——使用 Tailwind 的 `dark:shadow-*` 变体。

### 1.7 图标风格

- 使用 **Lucide** 图标库（与 Tailwind CSS 生态一致）
- 统一 1.5px 描边宽度
- 图标尺寸：16px（内联）/ 20px（按钮）/ 24px（状态指示）
- **禁止使用 emoji 作为功能图标**

### 1.8 动画基础

| 场景 | 持续时间 | 缓动函数 | 说明 |
|------|---------|----------|------|
| 悬停状态 | 150ms | ease-out | 按钮、行、操作项的 hover |
| 面板展开/收起 | 200ms | ease-in-out | Diff 面板、侧面板 |
| 对话框出现 | 200ms | ease-out + scale(0.95→1) | 模态对话框 |
| toast 提示 | 300ms | ease-out | 操作反馈 |
| 列表刷新 | 150ms | ease-out | 文件列表行更新 |
| 进度动画 | 持续 | linear | 进度条 |

|------|--------|--------|

**面板尺寸与交互：**

- 折叠时反方向滑出

**diff2html 渲染规则：**
- 背景：亮色使用 slate-50，暗色使用 `dark:bg-slate-800`
- 删除行：红色底色 `bg-red-50`（亮色）/ `dark:bg-red-900/30`（暗色）

- 行号列：固定宽度 60px，灰色文字
- 差异块导航按钮置于底部，左右居中

**Diff 内容交互：**
- 支持 `Ctrl+F` 在 diff 内容中搜索关键词，匹配行高亮
- 代码行过长时不自动换行，水平滚动查看

## 4. 交互规范

### 4.1 工具栏按钮控制

工具栏按钮根据上下文启用/禁用，并显示 tooltip 说明原因：

| 按钮 | 禁用条件 | tooltip |
|------|---------|---------|
| 提交 | 无变更时 | "当前工作副本无变更" |
| 更新 | 工作副本不可用 | "无可用工作副本" |
| 日志 | 工作副本不可用 | "无可用工作副本" |
| 切换 | 工作副本不可用 | "无可用工作副本" |
| 分支/标签 | 工作副本不可用 | "无可用工作副本" |
| 合并 | 工作副本不可用 | "无可用工作副本" |
| 清理 | 工作副本不可用 | "无可用工作副本" |
| 导出 | 工作副本不可用 | "无可用工作副本" |
| ⚙️ 设置 | 始终可用 | — |

**工具栏溢出：** 当窗口宽度导致按钮放不下时，超出的按钮自动折叠到末尾的 `···` 更多菜单中。溢出阈值动态计算，至少保留"提交"和"设置"两个按钮在可见区域。禁用态使用 `opacity-50 cursor-not-allowed`。

### 4.2 操作反馈

| 触发 | 反馈形式 | 持续时间 | 样式 |
|------|---------|----------|------|
| 提交成功 | toast | 3s 自动关闭 | green-500 左侧图标 |
| 提交失败 | toast | 持续，手动关闭 | red-500 |
| 更新完成 | 系统通知 | 系统默认 | 含变更摘要 |
| 更新失败 | toast | 持续，手动关闭 | red-500，显示错误信息 |

| 删除文件 | 确认对话框 → toast | — | 确认用 danger 强调 |
| 检出中 | ProgressOverlay（可取消） | 操作结束 | 进度条 + 阶段文字 + [取消] 按钮 |
| 检出失败 | ProgressOverlay 错误态 → toast | — | 错误摘要 + [重试] / [关闭] |
| 切换分支成功 | toast | 3s 自动关闭 | green-500 + 自动刷新列表 |
| 切换分支失败 | toast | 持续，手动关闭 | red-500，保持当前分支 |

| 清理失败 | toast | 持续，手动关闭 | red-500 |
| 解锁成功 | toast | 3s 自动关闭 | green-500 |
| 忽略成功 | toast | 3s 自动关闭 | green-500 |
| 网络超时 | toast | 持续 | red-500 + 重试按钮 |
| 右键菜单更新 | 系统通知 | 5s | 显示文件变更数 |

**toast 样式要求：**
- Element Plus `<el-message>` 组件
- 成功：绿色 `#22C55E` 圆角图标 + 成功文字
- 错误：红色 `#EF4444` + 错误描述
- 警告：琥珀色 `#F59E0B` + 警告文字
- 所有 toast 默认 3s 自动关闭，错误和警告类型持续显示直到用户手动关闭

### 4.3 确认对话框

所有破坏性操作（还原、删除、切换分支等）必须使用确认对话框。模板见 §3.7。确认按钮使用 `type="danger"` 样式。

## 5. 暗色模式

### 5.1 策略

- **默认跟随系统**：通过 CSS `prefers-color-scheme` media query
- **手动切换**：TopBar 右侧提供暗色/亮色切换按钮
- **持久化**：用户选择存入 `tauri-plugin-store`，覆盖系统默认
- **实现方式**：仅使用 Tailwind `dark:` 变体，不使用自定义 CSS 变量或 `<style>` 标签

### 5.2 实现示例

所有暗色适配通过 Tailwind `dark:` 变体实现：

```html
<!-- 卡片组件：亮色 white，暗色 slate-800 -->
<div class="bg-white dark:bg-slate-800 border-slate-200 dark:border-slate-700 rounded-lg shadow-sm dark:shadow-none">
  <!-- 标题：亮色 slate-900，暗色 slate-50 -->
  <h2 class="text-slate-900 dark:text-slate-50">面板标题</h2>
  <!-- 次要文字：亮色 slate-500，暗色 slate-400 -->
  <p class="text-slate-500 dark:text-slate-400">辅助信息</p>
</div>
```

### 5.3 暗色特殊处理

| 元素 | 暗色适配 |
|------|---------|
| Diff 删除行 | 透明度 `dark:bg-red-900/30` 替代红色底色 |
| Diff 新增行 | 透明度 `dark:bg-green-900/30` 替代绿色底色 |
| 对话框遮罩 | `dark:bg-black/60` 更深以突出前景 |
| 阴影 | 暗色下几乎不可见，使用 `dark:shadow-none` + 边框替代 |
| 状态图标颜色 | 调亮一级（见 §1.3 表格）|
| 输入框 | `dark:bg-slate-800 dark:border-slate-600` |

## 6. 无障碍设计

| 要求 | 实现 |
|------|------|
| 对比度 | 文字 4.5:1 最小对比度（亮色/暗色均满足）|
| 键盘导航 | 所有操作均支持键盘完成（见 §4.4 快捷键）|
| 焦点指示 | 2px blue-400 outline 在所有可交互元素上 |
| 颜色非唯一指示 | 文件状态使用颜色 + 图标双通道传递 |
| 屏幕阅读器 | `<el-table>` 默认支持 ARIA；图标按钮加 `aria-label`；操作按钮列建议加 `aria-label="对 xxx 文件执行差异对比"` |
| 动画尊重 | `prefers-reduced-motion` 时禁用非必要动画 |
| 文本缩放 | 支持浏览器/系统字体缩放（使用 Tailwind 类而非 px 硬编码）|

## 8. Element Plus 组件使用约定

| 原生组件 | 使用场景 | 定制 |
|---------|---------|------|
| `<el-table>` | 文件列表、日志表 | 紧凑模式 `size="small"`，行高 40px |
| `<el-dialog>` | 所有对话框 | 关闭按钮 + 遮罩点击关闭 |
| `<el-input>` | 所有输入框 | `size="default"` |
| `<el-select>` | 下拉选择 | 全局使用 |
| `<el-button>` | 所有按钮 | 主操作使用 `type="primary"`（映射为 green）|
| `<el-message>` | toast 反馈 | 成功/错误/警告三类 |
| `<el-progress>` | 进度条 | 线性样式 |
| `<el-switch>` | 开关（设置页）| — |
| `<el-tooltip>` | 工具栏按钮禁用解释 | `effect="dark"` |

> **注意**：Element Plus 的主题色需通过 `@theme` 在 `src/style.css` 中覆盖：
> ```css
> @theme {
>   --color-primary: #22C55E;
>   --color-danger: #EF4444;
>   --color-warning: #F59E0B;
> }
> ```
