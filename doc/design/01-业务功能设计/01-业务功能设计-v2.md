# 01-业务功能设计

> Free-SVN — 基于 Tauri + Vue 的跨平台 SVN 图形化工具

## 1. 概述

### 1.1 目标

提供一个跨平台（macOS / Windows / Linux）的 SVN 图形化客户端，核心体验对标 TortoiseSVN，包含系统右键菜单快捷操作和主窗口完整功能。

### 1.2 核心理念

- **单工作副本模式** — 每次只关注一个工作副本，切换通过下拉菜单选择已检出的副本
- **变更文件列表为默认视图** — 打开软件直接看到所有变更，提升效率
- **调用系统 svn CLI** — 不依赖 svn 库绑定，用户只需要安装了 SVN 客户端
- **内置差异查看器** — 默认基于 `diff2html` + `highlight.js` 实现语法高亮 diff，可选配外部工具

### 1.3 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + TypeScript + Vite + Tailwind CSS + Element Plus + Pinia + Vue Router |
| 后端 | Tauri 2 (Rust) + serde + thiserror |
| 存储 | tauri-plugin-store（本地配置持久化） |
| I18n | vue-i18n（按需引入） |
| SVN | 调用系统 `svn` CLI 解析 XML/文本输出 |

### 2.1 前置依赖

- 用户必须预装 SVN 命令行客户端（`svn` 命令在 PATH 中，或在设置中指定路径）
- 首次启动检测 svn 是否可用，不可用时提示用户安装

### 2.2 SVN 认证交互策略

svn CLI 可能触发交互式认证提示（用户名/密码），处理策略：

- **所有 SVN 命令设置超时**（默认 60 秒，可配置），超时后终止进程并返回超时错误
- **不处理交互式认证提示** — 建议用户使用 SSH key 或 `~/.subversion/auth/` 缓存凭据，或配置 `~/.subversion/servers` 中的用户名/密码
- 如果检测到 svn 进程在等待 stdin 输入（认证提示），超时退出并提示用户配置凭据缓存
- 不支持在 UI 中输入 SVN 密码后传递给命令行（避免密码泄露风险）

## 3. 系统右键菜单

### 3.1 菜单内容

右键菜单按目标位置区分：

**在 SVN 工作副本目录上右键（完整菜单）：**

```
SVN 提交...              → 打开主窗口提交面板
SVN 更新                 → 后台执行 + 系统通知
SVN 查看日志             → 打开主窗口日志视图
SVN 更新到版本...        → 打开版本选择对话框
───────────────────────────────────────────
SVN 差异对比             → 打开差异视图
SVN 还原...              → 确认后还原
───────────────────────────────────────────
SVN 切换(Switch)...      → 打开切换分支对话框
SVN 分支/标签...         → 打开分支/标签创建对话框
SVN 合并...              → 打开合并向导
───────────────────────────────────────────
SVN 清理                 → 后台执行 + 系统通知
SVN 导出...              → 选择目录后导出
```

**在普通文件夹/空目录上右键（仅检出）：**

```
SVN 检出...              → 打开检出对话框，目标目录自动填入
```

**在工作副本内文件上右键（跟目录菜单相同，但操作作用于当前文件）：**

同目录菜单，但提交/差异等操作限定为选中文件。

### 3.2 行为逻辑

| 条件 | 行为 |
|------|------|
| 主窗口未打开 | 启动主窗口，设置当前工作副本为该目录 |
| 主窗口已打开，但当前副本不同 | 切换到该目录作为当前工作副本 |
| 操作无需主窗口交互（更新/清理） | 后台执行，结果通过系统通知展示 |
| 操作需主窗口交互（提交/日志/差异） | 焦点回到主窗口，打开对应视图 |

### 3.3 跨平台实现

| 平台 | 实现方式 |
|------|----------|
| macOS | Finder 扩展 / Quick Action / 通过 Tauri 注册 URL scheme 回调 |
| Windows | 注册表 Shell 扩展（右键菜单 GUID 注册） |
| Linux | Nautilus 脚本 / Thunar UCA 配置 |

### 4.2 顶部状态栏

| 元素 | 说明 |

### 4.3 SVN 工具栏

每个按钮的含义和启用条件：

| 按钮 | 操作 | 启用条件 |
|------|------|----------|
| 提交 | 打开提交面板 | 有未提交的变更 |
| 更新 | 执行 svn update | 有工作副本 |
| 日志 | 打开日志视图 | 有工作副本 |
| 切换 | 切换到其他分支/URL | 有工作副本 |
| 分支/标签 | 创建分支/标签 | 有工作副本 |

| ⚙️ 设置 | 打开设置页面 | 始终可用 |

### 4.4 变更文件列表

**文件状态图标：**

| 图标 | 含义 | SVN 状态 |
|------|------|----------|
| `M` | 已修改 | modified |
| `✚` | 已新增 | added |
| `!` | 冲突 | conflicted |
| `🗑` | 已删除 | deleted |
| `?` | 未加入版本控制 | unversioned |
| `-` | 目录锁定 | locked |

**操作列按钮（按文件状态动态显示）：**

| 文件状态 | 按钮组合 |
|----------|----------|
| 已修改 (M) | [差异] [还原] |
| 新增 (✚) | [差异] [忽略] [删除] |
| 冲突 (!) | [差异] [合并] |
| 删除 (🗑) | [差异] [忽略] [还原] |
| 未加入 (?) | [差异] [忽略] |
| 锁定 (-) | [差异] [解锁] |

- **差异**：底部面板展开差异对比视图
- **还原**：恢复到未修改状态（需要确认）
- **忽略**：添加到 svn:ignore 列表（弹出选择忽略模式）
- **删除**：直接删除新增文件（需要确认）
- **合并**：打开冲突解决界面

**搜索与筛选：**
- 搜索框：按文件名过滤
- 筛选下拉：全部文件 / 已修改 / 新增 / 冲突 / 已删除 / 未加入版本控制

**全选功能：**
- 顶部复选框全选/取消全选
- 底部显示"已选 N 个文件"，提交时使用选择状态

**自动刷新与边界处理：**
- 文件列表在以下情况自动刷新：提交后、更新后、还原后、切换工作副本后
- 如果检测到工作副本路径不存在或被删除/移动，显示提示信息并切换到欢迎页（保留该路径在历史记录中供用户修正）
- 主窗口获得焦点时（`window focus` 事件），自动重新执行 `svn status` 刷新列表，以同步外部修改

### 4.5 底部状态栏

| 元素 | 说明 |
|------|------|
| 版本号 | 当前工作副本的 SVN revision |
| 最新提交 | 最近一次提交的时间 |

### 5.2 主视图（变更文件列表）

默认视图，见 4.4 节。用户双击文件行进入该文件的差异对比视图，点击"提交"进入提交面板。

### 5.3 差异对比视图（内嵌面板）

**技术选型：** 使用 `diff2html` 渲染差异对比，`highlight.js` 提供语法高亮。两者与 Vue 3 兼容性好，无需自建 diff 引擎。

**形态**：在主窗口底部展开（主列表下方，不影响文件列表宽度），非独立页面。

```
┌─ 差异对比: src/views/Home.vue → ────────── [外部工具打开] [✕关闭] ─┐
│ 版本: 1023 → 1024                   作者: username                     │
│ ┌─────────────────────────────────────────────────────────────────┐  │
│ │ @@ -10,7 +10,9 @@                                              │  │
│ │  <template>                                                     │  │
│ │  -  <h1>旧标题</h1>                                             │  │
│ │  +  <h1>新标题</h1>                                             │  │
│ │  +  <p>新增段落</p>                                             │  │
│ │  </template>                                                    │  │
│ │                                                                 │  │
│ │  [◀ 上一个差异]  [下一个差异 ▶]                                 │  │
│ └─────────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────┘
```

**特性：**
- 语法高亮（按文件扩展名判断语言）
- 行号显示
- 删除行红色底色、新增行绿色底色
- 逐块导航（上/下一个差异块）
- 可配置外部工具打开
- 双击文件行打开差异 / 操作列的"差异"按钮也打开此面板

### 5.6 检出对话框

**形态**：模态对话框。

| 输入项 | 类型 | 说明 |
|--------|------|------|
| 仓库 URL | 文本输入 | SVN 仓库地址（svn:// / https:// / svn+ssh://） |
| 检出到 | 路径输入 + 浏览 | 自动填入右键文件夹路径或默认检出目录 |
| 检出深度 | 下拉选择 | 完全递归 / 仅当前目录 / 仅文件 / 仅子项 |
| 仅此目录（不包括子目录） | 复选框 | 相当于 `--depth empty` |

操作：点击"检出" → 后台执行 + 进度显示 → 完成后切换到该工作副本。

### 5.7 设置页面

**形态**：独立路由或右侧面板。

| 分类 | 设置项 | 类型 | 说明 |
|------|--------|------|------|
| SVN | SVN 可执行路径 | 文件路径输入 | 默认 PATH 查找，可手动指定 |
| SVN | 默认检出目录 | 目录路径输入 | 新建检出的默认保存位置 |
| SVN | 全局忽略模式 | 文本输入 | 空格分隔的忽略模式 |
| 差异 | 差异工具 | 下拉选择 | 内置 / VS Code / Beyond Compare / Kaleidoscope / 自定义 |
| 差异 | 合并工具 | 下拉选择 | 同上 |
| 界面 | 显示未版本控制的文件 | 开关 | 控制 `?` 文件是否显示在列表中 |
| 界面 | 语言 | 下拉选择 | 跟随系统 / 简体中文 / English / 日本語 / 한국어 / 其他 |
| 通用 | 开机自启动 | 开关 | 后台常驻以支持右键菜单 |
| 通用 | 关于 | 信息展示 | 版本号、许可证、仓库地址 |

**外部工具命令行参数：**

| 工具 | 差异查看命令 | 合并命令 |
|------|-------------|---------|
| VS Code | `code --diff <file1> <file2>` | `code <file>`（手动编辑）|
| Beyond Compare | `bcomp <file1> <file2>` | `bcomp <file1> <file2> <file3> <output>` |
| Kaleidoscope | `ksdiff <file1> <file2>` | `ksdiff --merge <file1> <file2> <file3> <output>` |
| 自定义 | 用户输入命令模板 | 用户输入命令模板 |

### 5.8 切换(Switch)对话框

**形态**：模态对话框。

## 6. SVN 操作与后端 Commands

### 6.1 完整操作清单与输入输出

以下每个操作对应一个 Tauri Command，输出通过 `serde_json` 序列化返回给前端。

| 操作 | 输入参数 | 输出 | 优先级 |
|------|---------|------|--------|
| **status** | path | 文件状态列表（每个文件的状态、路径、属性变更等） | MVP |
| **info** | path | 工作副本信息（URL、版本、分支、作者等） | MVP |
| **checkout** | url, targetPath, depth, ignoreExternals | 成功/失败 + 版本号 | MVP |
| **commit** | paths[], message, keepLocks | 新版本号 + 提交结果 | MVP |
| **update** | path, revision?, depth? | 新版本号 + 更新结果（冲突/已修改） | MVP |
| **log** | path, limit, revision?, search? | 版本历史列表 + 每个版本的变更文件 | MVP |
| **diff** | path, revision1?, revision2? | 差异文本（可格式化为统一 diff） | MVP |
| **add** | path[] | 每个文件添加结果 | MVP |
| **delete** | path[], keepLocal? | 每个文件删除结果 | MVP |
| **revert** | path[] | 每个文件还原结果 | MVP |
| **resolve** | path, resolution | 解决冲突结果 | MVP |
| **switch** | path, targetUrl, ignoreAncestry? | 切换结果 | 后续迭代 |
| **copy**(branch/tag) | srcUrl, dstUrl, message, revision? | 操作结果 | 后续迭代 |
| **merge** | srcUrl, revStart, revEnd, targetPath | 合并结果（冲突文件列表）| 后续迭代 |
| **cleanup** | path | 清理结果 | MVP |
| **export** | path, targetDir, revision? | 导出结果 | 后续迭代 |
| **ignore** | path, pattern | 添加 svn:ignore 结果 | MVP |
| **blame** | path, revision? | 每行归属信息（版本 + 作者 + 日期 + 行号）| 后续迭代 |
| **relocate** | path, fromUrl, toUrl | 重定位结果 | 后续迭代 |
| **property** | path, propName?, action? | 属性列表或操作结果 | 后续迭代 |
| **lock** | path[], message? | 锁定结果 | 后续迭代 |
| **unlock** | path[] | 解锁结果 | 后续迭代 |

### 6.2 实现策略

**CLI 调用方式**：
```rust
// 统一执行模式
fn svn_exec(command: &str, args: &[&str], cwd: &Path) -> Result<SvnOutput, SvnError>
```

- 调用 `svn` 命令时使用 `--xml` 参数获取 XML 输出，通过 serde 解析
- 设置超时（默认 60 秒，大操作可配置）
- 异步执行（Tauri 支持 async command）
- 长操作通过 Tauri event 推送进度到前端

**错误处理**：
```rust
#[derive(Debug, thiserror::Error)]
pub enum SvnError {
    #[error("SVN 命令执行失败: {0}")]
    ExecutionFailed(String),
    #[error("SVN 输出解析失败: {0}")]
    ParseFailed(String),
    #[error("操作超时")]
    Timeout,
    #[error("SVN 未安装或不在 PATH 中")]
    SvnNotFound,
    #[error("非 SVN 工作副本")]
    NotWorkingCopy,
}
```

## 7. 路由设计

路由使用 MainLayout 作为父布局，包含顶部状态栏、SVN 工具栏和底部状态栏：

```
/                           → 重定向到 /workspace
/workspace                  → 主视图（变更文件列表 / 欢迎页空状态）
/workspace/commit           → 提交面板
/workspace/log              → 日志页面
/workspace/diff?file=xxx    → 差异对比（可路由以支持外部唤起）
/workspace/resolve?file=xxx → 冲突解决页面
/settings                   → 设置页面
```

- `workspace` 前缀下的页面共享 MainLayout（顶栏 + 工具栏 + 底栏完整布局）
- `settings` 页面独立于工作副本上下文，可独立布局

以下操作用模态对话框实现，不占用路由：

- 检出（/checkout → 模态对话框）
- 分支/标签（/branch-tag → 模态对话框）
- 合并向导（/merge → 模态对话框）
- 切换（/switch → 模态对话框）
- 更新到版本（/update-revision → 模态对话框）

## 8. 数据流

### 8.1 前端状态管理 (Pinia Store)

```
stores/
  ├── workspace.ts    -- 当前工作副本信息、最近副本列表
  ├── fileList.ts     -- 文件状态列表、选中状态、筛选/搜索
  ├── settings.ts     -- 设置项
  └── svn.ts          -- SVN 操作封装（invoke commands）
```

### 8.2 主要数据流

```
用户点击"提交"

  → Pinia: svnStore.commit(params)
    → Tauri invoke('svn_commit', { path, files, message })
      → Rust: 执行 svn commit CLI
      → 解析 XML 输出 → 返回 { revision, errors? }
    → Pinia store 更新
  → 变更文件列表刷新（auto refresh）
  → 显示提交结果
```

```
右键文件夹 → SVN 更新

  → Tauri 后台 process: svn update
    → 完成 → 系统通知显示结果
  → 如果主窗口已打开且是当前工作副本
    → 自动刷新变更文件列表
```

## 10. 非功能需求

| 需求 | 要求 |
|------|------|
| 性能 | 变更文件列表加载 ≤ 1 秒（10000 文件以内）|
| 性能 | 差异对比渲染 ≤ 2 秒 |
| 性能 | 10000+ 文件场景：使用虚拟滚动（仅渲染可见行），增量加载状态 |
| 响应 | 长操作（更新/提交/检出）显示实时进度 |
| 健壮性 | 所有 SVN 命令设置超时（默认 60s），超时后终止进程并返回友好错误 |
| 健壮性 | 所有 SVN 操作的错误信息友好展示，不抛原始 stderr |
| 健壮性 | 工作副本路径失效时提示用户，不崩溃 |
| 健壮性 | 主窗口获得焦点时自动刷新文件列表 |
| 安装 | 右键菜单注册/清理在安装/卸载时处理 |
| 国际化 | I18n 覆盖所有 UI 文案，初始化至少支持中/英 |
| 跨平台 | macOS / Windows / Linux 三平台覆盖 |
