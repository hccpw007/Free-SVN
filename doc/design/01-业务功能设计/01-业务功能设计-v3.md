# 01-业务功能设计 v3

> Free-SVN — 基于 Tauri + Vue 的跨平台 SVN 图形化工具
> 更新说明：v3 基于专家评估报告补充并发控制、取消机制、系统托盘、单实例防护、日志体系、许可证合规等缺失项

## 1. 概述

### 1.1 目标

提供一个跨平台（macOS / Windows / Linux）的 SVN 图形化客户端，核心体验对标 TortoiseSVN，包含系统右键菜单快捷操作和主窗口完整功能。

### 1.2 核心理念

- **单工作副本模式** — 每次只关注一个工作副本，切换通过下拉菜单选择已检出的副本
- **变更文件列表为默认视图** — 打开软件直接看到所有变更，提升效率
- **捆绑 svn CLI** — 安装包内置各平台 svn 可执行文件，应用仅使用内置 svn；用户安装即用，零额外步骤
- **内置差异查看器** — 默认基于 `diff2html` + `highlight.js` 实现语法高亮 diff，可选配外部工具

### 1.3 技术栈

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + TypeScript + Vite + Tailwind CSS + Element Plus + Pinia + Vue Router |
| 后端 | Tauri 2 (Rust) + serde + thiserror |
| 存储 | tauri-plugin-store（本地配置持久化） |
| I18n | vue-i18n（按需引入） |
| SVN | 调用内置 `svn` CLI 解析 XML/文本输出 |

### 2.1 前置依赖

- **内置 svn**：安装包捆绑对应平台的 svn 可执行文件（约 2-3MB），应用仅使用内置 svn，不依赖系统安装
- **首次启动**：检测内置 svn 是否可执行，异常时提示用户重新安装 Free-SVN

### 2.2 SVN 认证交互策略

svn CLI 可能触发交互式认证提示（用户名/密码），处理策略：

- **所有 SVN 命令设置超时**（默认 60 秒，可配置），超时后终止进程并返回超时错误
- **不处理交互式认证提示** — 建议用户使用 SSH key 或 `~/.subversion/auth/` 缓存凭据，或配置 `~/.subversion/servers` 中的用户名/密码
- 如果检测到 svn 进程在等待 stdin 输入（认证提示），超时退出并提示用户配置凭据缓存
- 不支持在 UI 中输入 SVN 密码后传递给命令行（避免密码泄露风险）
- **首次认证引导**：超时/认证失败时，提示信息应区分"网络问题"和"认证问题"，并给出配置指引链接或简要说明（如"请配置 SSH key 或 ~/.subversion/auth/ 凭据"）

### 2.3 内置 svn 版本维护与许可证合规

| 方面 | 策略 |
|------|------|
| 更新时机 | 内置 svn 跟随 Free-SVN 发版更新，不提供应用内动态升级 |
| 平台支持 | macOS：使用 Homebrew 或官方编译的二进制（Apache 2.0）；Windows：使用官方 Apache 二进制（Apache 2.0），**不采用 SlikSVN**（专有许可不可再分发）；Linux：使用静态编译二进制（Apache 2.0）|
| 打包方式 | 各平台 svn 二进制存放在 `resources/svn/{platform}/` 目录，由 Tauri `resources` 配置打包 |
| 版本显示 | 底部状态栏和关于页显示内置 svn 版本号 |
| 许可证声明 | 应用关于页面和发行说明中声明 Apache 2.0 许可证和 Subversion 版权归属 |
| 合规检查 | 发行前确认所有捆绑二进制的许可证允许再分发 |

### 2.4 并发操作控制策略

**背景**：多个 SVN 操作同时执行可能导致 `.svn/` 元数据损坏、工作副本状态不一致。

**控制规则**：

| 分类 | 操作 | 并发策略 | 说明 |
|------|------|---------|------|
| 只读操作 | status, info, diff, log, blame | 可并发执行 | 多个只读操作可同时运行，不互斥 |
| 写操作 | commit, update, switch, merge, checkout, cleanup, revert, resolve, add, delete, ignore, lock, unlock, copy, export, relocate | 互斥执行 | 同一时间每个工作副本只允许一个写操作 |
| 杂项 | property | 互斥执行 | 属性修改归类为写操作 |

**实现方式**：
1. 后端（Rust）：使用 `Mutex` 包裹操作执行器，写操作加锁，只读操作跳过锁
2. 前端：写操作进行时，对应工具栏按钮和菜单项禁用，ProgressOverlay 显示中
3. 队列行为：写操作进行中触发的新写操作 → 提示"当前有操作正在进行"（不排队）
4. 写操作锁仅作用于**同一工作副本**；不同工作副本的操作互不影响
5. `status` 等只读操作若在写操作执行期间被调用，返回上次缓存的列表或等待写操作完成（前端优先显示缓存，写操作完成后自动刷新）

### 2.5 长操作取消机制

**支持取消的操作**：checkout、update、export、merge（commit 是原子操作，不可取消）

**取消交互**：
- ProgressOverlay 上显示 [取消] 按钮
- 点击取消后 → 按钮变为"正在取消..."（禁用态）→ 发送终止信号

**取消实现**：
1. 前端点击取消 → invoke('cancel_operation') → Tauri event `operation:cancelled` 通知后端
2. Rust 侧发送 `SIGINT`/`SIGTERM` 终止 svn 进程
3. 进程终止后自动执行 `svn cleanup` 清理工作副本状态
4. 取消完成后 → Tauri event 通知前端 → 恢复 UI 状态
5. 清理过程本身不可取消

**取消后状态**：
- 如果 svn 已部分写入（如 update 已更新部分文件），cleanup 会恢复到一致状态
- 前端刷新文件列表反映当前实际状态

### 2.6 操作前置条件校验

每个操作在执行前必须经过前置条件检查，不通过则按钮禁用或弹出提示：

| 操作 | 前置条件 | 不满足时行为 |
|------|---------|-------------|
| 提交 | 至少选中一个文件、有工作副本 | 提交按钮禁用 |
| 更新 | 有工作副本 | 按钮禁用 |
| 日志 | 有工作副本 | 按钮禁用 |
| 差异 | 已选中一个文件 | 差异按钮禁用 |
| 检出 | URL 非空、目标路径非空 | 检出按钮禁用 |
| 切换 | 目标 URL 非空、有工作副本 | 按钮禁用 |
| 还原 | 至少选中一个文件 | 还原按钮禁用 |
| 合并 | 源 URL 非空、有工作副本 | 合并按钮禁用 |
| 清理 | 有工作副本 | 按钮禁用 |
| 导出 | 目标路径非空、有工作副本 | 导出按钮禁用 |
| 忽略 | 至少选中一个可忽略的文件 | 忽略按钮隐藏 |
| 解锁 | 至少选中一个锁定文件 | 解锁按钮隐藏 |

### 2.7 持久化存储异常恢复

tauri-plugin-store 持久化 JSON 文件可能因写入中断、用户误编辑等原因损坏。

**恢复策略**：
- 读取 store 时使用 try_deserialize，失败时：
  1. 将损坏文件备份为 `.store.json.bak`
  2. 重置为默认配置
  3. 应用正常启动，不崩溃
- 常规操作不需要备份数据（SVN 工作副本是源数据，丢失 store 只是丢失最近列表）
- 跨版本升级时，store 中不存在的字段采用默认值，不做 migration（tauri-plugin-store 自然处理）

## 3. 系统右键菜单

### 3.1 菜单内容

右键菜单按目标位置区分：

**在 SVN 工作副本目录上右键（完整菜单）：**

```
SVN 提交...              → 打开主窗口提交面板
SVN 更新                 → 后台执行 + 系统通知
SVN 查看日志             → 打开主窗口日志视图
SVN 更新到版本...        → 打开版本选择对话框
───────────────────────────────────────────
SVN 差异对比             → 打开差异视图
SVN 还原...              → 确认后还原
───────────────────────────────────────────
SVN 切换(Switch)...      → 打开切换分支对话框
SVN 分支/标签...         → 打开分支/标签创建对话框
SVN 合并...              → 打开合并向导
───────────────────────────────────────────
SVN 清理                 → 后台执行 + 系统通知
SVN 导出...              → 选择目录后导出
```

**在普通文件夹/空目录上右键（仅检出）：**

```
SVN 检出...              → 打开检出对话框，目标目录自动填入
```

**在工作副本内文件上右键（跟目录菜单相同，但操作作用于当前文件）：**

同目录菜单，但提交/差异等操作限定为选中文件。

### 3.2 行为逻辑

| 条件 | 行为 |
|------|------|
| 主窗口未打开 | 启动主窗口，设置当前工作副本为该目录 |
| 主窗口已打开，但当前副本不同 | 切换到该目录作为当前工作副本 |
| 操作无需主窗口交互（更新/清理） | 后台执行，结果通过系统通知展示 |
| 操作需主窗口交互（提交/日志/差异） | 焦点回到主窗口，打开对应视图 |

### 3.3 跨平台实现

| 平台 | 实现方式 |
|------|----------|
| macOS | Finder 扩展 / Quick Action / 通过 Tauri 注册 URL scheme 回调 |
| Windows | 注册表 Shell 扩展（右键菜单 GUID 注册） |
| Linux | Nautilus 脚本 / Thunar UCA 配置 |

### 3.4 系统托盘与后台模式

**设计背景**：§5.7 设置了"开机自启动"，但关闭窗口行为未定义。如果自启动后没有后台模式，启动就打开全屏窗口，用户体验不好。

**关闭窗口行为**：
- 用户点击关闭按钮（红绿灯 / [×]）→ **最小化到系统托盘**，应用继续运行
- 用户右键托盘图标 → 选择"退出"→ 完全退出
- macOS 上：点击 Dock 关闭窗口按钮 → 同关闭按钮行为（最小化到托盘）

**托盘图标**：
- 默认图标：应用图标的小型版本（单色/简约版）
- 操作进行时：图标显示小徽章或动画表示有后台操作

**托盘右键菜单**：

```
显示主窗口
──────────────────
SVN 更新当前工作副本      → 后台执行 + 系统通知
SVN 清理当前工作副本       → 后台执行 + 系统通知
──────────────────
关于 Free-SVN
退出
```

**后台操作反馈**：
- 后台操作（更新/清理）完成时 → 系统通知（macOS Notification Center / Windows Toast / Linux notify-send）
- 用户点击通知 → 激活主窗口并定位到相关结果

**开机自启动的关系**：
- 自启动时应用启动 → 最小化到托盘（不打开主窗口）
- 若有关联右键菜单操作传递给首实例的参数，则打开主窗口并跳转

### 3.5 单实例运行防护

**策略**：使用 `tauri-plugin-single-instance` 确保只运行一个实例。

**第二实例启动行为**：
1. 如果主窗口最小化到托盘 → 激活主窗口并前置
2. 如果第二实例携带了右键菜单参数（如 --workspace=/path）→ 传递给首实例处理，首实例切换到该工作副本
3. 忽略第二实例的其他参数

**为什么不允许多实例**：
- 两个实例操作同一工作副本 → `.svn/` 元数据损坏
- 后台右键菜单操作与主窗口操作冲突
- macOS / Windows 的用户直觉是"一个应用一个窗口"

### 3.6 窗口状态持久化

**策略**：每次关闭窗口时记录位置和大小，下次打开时恢复。

| 持久化项 | 说明 |
|---------|------|
| 窗口位置 (x, y) | 保存并恢复 |
| 窗口大小 (width, height) | 保存并恢复 |
| 是否最大化 | 记录是否最大化状态 |
| 多显示器 | 如果上次关闭的显示器已不存在，回退到主显示器居中显示 |

使用 `tauri-plugin-window-state` 实现。

### 4.4 变更文件列表

**文件状态图标：**

| 图标 | 含义 | SVN 状态 |
|------|------|----------|
| `M` | 已修改 | modified |
| `✚` | 已新增 | added |
| `!` | 冲突 | conflicted |
| `🗑` | 已删除 | deleted |
| `?` | 未加入版本控制 | unversioned |
| `-` | 目录锁定 | locked |

**操作列按钮（按文件状态动态显示）：**

| 文件状态 | 按钮组合 |
|----------|----------|
| 已修改 (M) | [差异] [还原] |
| 新增 (✚) | [差异] [忽略] [删除] |
| 冲突 (!) | [差异] [合并] |
| 删除 (🗑) | [差异] [忽略] [还原] |
| 未加入 (?) | [差异] [忽略] |
| 锁定 (-) | [差异] [解锁] |

- **差异**：底部面板展开差异对比视图
- **还原**：恢复到未修改状态（需要确认）
- **忽略**：添加到 svn:ignore 列表（弹出选择忽略模式）；如果该文件已匹配已有 svn:ignore 规则，显示"取消忽略"选项
- **删除**：直接删除新增文件（需要确认）
- **合并**：打开冲突解决界面

**右键菜单（文件行）：**
- 复制文件路径
- 复制相对路径
- 在文件管理器中显示
- 用外部编辑器打开

**搜索与筛选：**
- 搜索框：按文件名过滤
- 筛选下拉：全部文件 / 已修改 / 新增 / 冲突 / 已删除 / 未加入版本控制

**全选功能：**
- 顶部复选框全选/取消全选
- 底部显示"已选 N 个文件"，提交时使用选择状态

**自动刷新与边界处理：**
- 文件列表在以下情况自动刷新：提交后、更新后、还原后、切换工作副本后
- 如果检测到工作副本路径不存在或被删除/移动，显示提示信息并切换到欢迎页（保留该路径在历史记录中供用户修正）
- 主窗口获得焦点时（`window focus` 事件），自动重新执行 `svn status` 刷新列表，以同步外部修改
- **已知局限**：不通过文件系统事件监听实时刷新（避免 notify crate 在跨平台上的复杂度）。窗口 focus 刷新已覆盖"用户在终端修改后切回应用"的核心场景。后续迭代可增加文件监听。

### 4.5 底部状态栏

|------|------|
| 版本号 | 当前工作副本的 SVN revision |
| 最新提交 | 最近一次提交的时间 |
| SVN 版本 | 本机安装的 svn 版本号 |

## 6. SVN 操作与后端 Commands

### 6.1 完整操作清单与输入输出

以下每个操作对应一个 Tauri Command，输出通过 `serde_json` 序列化返回给前端。

| 操作 | 输入参数 | 输出 | 优先级 |
|------|---------|------|--------|
| **status** | path | 文件状态列表（每个文件的状态、路径、属性变更等） | MVP |
| **info** | path | 工作副本信息（URL、版本、分支、作者等） | MVP |
| **checkout** | url, targetPath, depth, ignoreExternals | 成功/失败 + 版本号 | MVP |
| **commit** | paths[], message, keepLocks | 新版本号 + 提交结果 | MVP |
| **update** | path, revision?, depth? | 新版本号 + 更新结果（冲突/已修改） | MVP |
| **log** | path, limit, revision?, search? | 版本历史列表 + 每个版本的变更文件 | MVP |
| **diff** | path, revision1?, revision2? | 差异文本（可格式化为统一 diff） | MVP |
| **add** | path[] | 每个文件添加结果 | MVP |
| **delete** | path[], keepLocal? | 每个文件删除结果 | MVP |
| **revert** | path[] | 每个文件还原结果 | MVP |

| **copy**(branch/tag) | srcUrl, dstUrl, message, revision? | 操作结果 | 后续迭代 |

### 6.2 实现策略

**CLI 调用方式**：
```rust
// 统一执行模式
fn svn_exec(command: &str, args: &[&str], cwd: &Path) -> Result<SvnOutput, SvnError>
```

- 使用内置 svn 可执行文件（运行时通过 Tauri `resource_dir()` 获取路径）
- 调用 `svn` 命令时使用 `--xml` 参数获取 XML 输出，通过 serde 解析
- 设置超时（默认 60 秒，大操作可配置）
- 异步执行（Tauri 支持 async command）
- 长操作通过 Tauri event 推送进度到前端
- 写操作通过 Mutex 互斥（参见 §2.4）

**操作取消实现**：
```rust
// 取消信号，通过 AtomicBool 在 executor 中检测
static CANCELLED: AtomicBool = AtomicBool::new(false);

// 取消操作 command
#[tauri::command]
fn cancel_operation() -> Result<(), SvnError> {
    CANCELLED.store(true, Ordering::SeqCst);
    // 终止当前 svn 进程
    kill_current_process();
    // 执行 cleanup
    svn_exec("cleanup", &[], &current_path)?;
    // 重置取消标志
    CANCELLED.store(false, Ordering::SeqCst);
    Ok(())
}

// 在长操作循环中检测
fn check_cancelled() -> bool {
    CANCELLED.load(Ordering::SeqCst)
}
```

**错误处理**：
```rust
#[derive(Debug, thiserror::Error)]
pub enum SvnError {
    #[error("SVN_EXEC_FAILED")]
    ExecutionFailed(String),
    #[error("SVN_PARSE_FAILED")]
    ParseFailed(String),
    #[error("SVN_TIMEOUT")]
    Timeout,
    #[error("SVN_NOT_FOUND")]
    SvnNotFound,
    #[error("SVN_NOT_WORKING_COPY")]
    NotWorkingCopy,
    #[error("SVN_OPERATION_IN_PROGRESS")]
    OperationInProgress,
    #[error("SVN_CANCELLED")]
    Cancelled,
}
```

**错误消息处理原则**：
- 后端返回 **错误码 key**（如 `SVN_TIMEOUT`）而非中文文字
- 前端根据当前 locale 将错误码渲染为用户界面语言
- 仅在日志中记录原始 stderr 供诊断

### 6.3 日志与诊断体系

**设计背景**：应用需要日志体系来支持用户问题诊断和开发者调试，这是生产级桌面应用的基本要求。

**日志存储**：
- 位置：使用 `tauri::api::path::app_log_dir()`（macOS: `~/Library/Logs/com.free-svn/`，Windows: `%APPDATA%/Free-SVN/logs/`）
- 文件：`free-svn.log`（当前日志），`free-svn.log.1`、`free-svn.log.2`（轮转备份）

**日志级别**：

| 级别 | 记录内容 |
|------|---------|
| ERROR | SVN 命令执行失败、解析失败、超时、未捕获异常 |
| WARN | 非致命问题（如 store 损坏已恢复、右键菜单注册失败） |
| INFO | SVN 命令执行（命令 + 参数 + 耗时）、操作开始/完成、窗口事件 |
| DEBUG | 原始 svn stdout/stderr、Tauri event 详情、状态变更日志 |

**日志轮转**：
- 单文件最大 5MB
- 保留最近 3 个日志文件
- 应用启动时检查并轮转

**用户诊断入口**：
- 设置页 → 关于 → [导出日志]
- 导出内容包含：当前日志文件 + 应用版本 + 系统信息 + 内置 svn 版本
- 用户点击 [导出日志] → 弹出系统保存对话框 → 生成 `.zip` 文件

**后端日志指令**（Rust 侧新增 command）：
```rust
#[tauri::command]
fn get_logs() -> Result<String, SvnError>  // 返回日志内容
#[tauri::command]
fn export_logs(target_path: String) -> Result<(), SvnError>  // 导出日志到指定路径
```

### 6.4 操作队列与并发控制

参见 §2.4 并发操作控制策略。后端实现要点：

1. 使用 `Mutex<Option<JoinHandle>>` 包裹当前写操作的句柄
2. 写操作开始时占用锁，try_lock 失败则返回 `SvnError::OperationInProgress`
3. 只读操作（status/info/diff/log/blame）跳过锁检查
4. 取消操作（cancel_operation）是特殊操作，不经过锁

```
/                           → 重定向到 /workspace
/workspace                  → 主视图（变更文件列表 / 欢迎页空状态）
/workspace/commit           → 提交面板
/workspace/log              → 日志页面
/workspace/diff?file=xxx    → 差异对比（可路由以支持外部唤起）
/workspace/resolve?file=xxx → 冲突解决页面
/settings                   → 设置页面
```

- `workspace` 前缀下的页面共享 MainLayout（顶栏 + 工具栏 + 底栏完整布局）
- `settings` 页面独立于工作副本上下文，可独立布局

以下操作用模态对话框实现，不占用路由：

- 检出（/checkout → 模态对话框）
- 分支/标签（/branch-tag → 模态对话框）
- 合并向导（/merge → 模态对话框）
- 切换（/switch → 模态对话框）
- 更新到版本（/update-revision → 模态对话框）

## 8. 数据流

### 8.1 前端状态管理 (Pinia Store)

```
stores/
  ├── workspace.ts    -- 当前工作副本信息、最近副本列表
  ├── fileList.ts     -- 文件状态列表、选中状态、筛选/搜索
  ├── settings.ts     -- 设置项
  └── svn.ts          -- SVN 操作封装（invoke commands）
```

### 8.2 主要数据流

```
用户点击"提交"

  → Pinia: svnStore.commit(params)
    → Tauri invoke('svn_commit', { path, files, message })
      → Rust: 操作队列检查（是否有写操作进行中）
        → 操作队列有空位 → 占用写操作锁 → 执行 svn commit CLI
        → 操作队列忙 → 返回 SvnError::OperationInProgress
      → 解析 XML 输出 → 返回 { revision, errors? }
    → Pinia store 更新
  → 变更文件列表刷新（auto refresh）
  → 显示提交结果
```

```
右键文件夹 → SVN 更新（后台操作）

  → Tauri 单实例检测（参见 §3.5）
  → 如果主窗口关闭 → 最小化到托盘执行
  → 写操作锁检查
  → svn update 执行（可取消）
    → 完成 → 系统通知显示结果
  → 如果主窗口已打开且是当前工作副本
    → 自动刷新变更文件列表
```

```
长操作取消

  → 用户点击 ProgressOverlay 的 [取消] 按钮
  → invoke('cancel_operation')
    → 设置 CANCELLED 标志
    → 终止 svn 进程
    → 执行 svn cleanup
    → 重置 CANCELLED 标志
    → 返回取消结果
  → Tauri event 'operation:completed' 通知前端
  → 刷新文件列表
  → ProgressOverlay 关闭
```

### 8.3 IPC 异常处理

前端调用 `invoke()` 时可能遇到以下异常：

| 异常场景 | 处理方式 |
|---------|---------|
| Rust command panic | 前端 catch 后显示通用错误提示，记录日志 |
| JSON 序列化/反序列化失败 | 同上 |
| Tauri 窗口关闭但后台操作进行中 | 不影响后台操作，操作完成通过系统通知反馈 |
| invoke 超时 | 前端超时提示，询问用户是否重试 |

**统一错误处理**：
- `svn.ts`（services 层）封装 `invoke()` 调用，对每个调用统一 try-catch
- 错误统一转换为中文/英文友好消息，不抛原始异常
- 严重错误（command panic）写入日志

## 10. 非功能需求

| 需求 | 要求 |
|------|------|
| 性能 | 变更文件列表加载 ≤ 1 秒（10000 文件以内）|
| 性能 | 差异对比渲染 ≤ 2 秒 |
| 性能 | 10000+ 文件场景：使用虚拟滚动（仅渲染可见行），增量加载状态 |
| 响应 | 长操作（更新/提交/检出）显示实时进度，且支持取消（见 §2.5）|
| 健壮性 | 所有 SVN 命令设置超时（默认 60s），超时后终止进程并返回友好错误 |
| 健壮性 | 所有 SVN 操作的错误信息友好展示，使用错误码 + 前端 i18n 翻译，不抛原始 stderr |
| 健壮性 | 工作副本路径失效时提示用户，不崩溃 |
| 健壮性 | 主窗口获得焦点时自动刷新文件列表 |
| 健壮性 | 写操作互斥执行，防止并发操作损坏工作副本（见 §2.4）|
| 健壮性 | 持久化存储损坏时自动恢复为默认配置（见 §2.7）|
| 健壮性 | 单实例运行，防止多实例冲突（见 §3.5）|
| 可观测 | 应用日志记录到系统日志目录，支持导出诊断信息（见 §6.3）|
| 可观测 | INFO 级别记录每次 SVN 命令的执行和耗时 |
| 安装 | 右键菜单注册/清理在安装/卸载时处理 |
| 国际化 | 所有 UI 文案通过 vue-i18n 管理，后端错误使用错误码由前端翻译 |
| 国际化 | 初始化至少支持中/英，后端错误码不硬编码具体语言 |
| 跨平台 | macOS / Windows / Linux 三平台覆盖 |
| 跨平台 | 快捷键统一使用 `CmdOrCtrl` 修饰键 |
| 跨平台 | 所有捆绑的 svn 二进制确认许可证允许再分发（§2.3）|
