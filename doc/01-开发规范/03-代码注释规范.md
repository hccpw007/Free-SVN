# 03-多语言代码注释规范

适用于本项目中所有语言的统一注释规范，覆盖前端（TypeScript / Vue / React）和后端（Rust / Java）。

---

# 01-总则

## 注释的目的

注释是代码评估、AI 快速定位功能的重要手段。好的注释让读者**无需逆向推导**即可理解代码的意图。注释不是代码的翻译——不应说"做了什么"，而是说"为什么这么做"。

## 统一原则（所有语言通用）

1. **功能变更必须同步更新注释** — 过期注释比没注释更危险。修改代码时，立即检查并更新相关注释
2. **不注释不言自明的代码** — 变量名、函数名本身已清晰表达用途时，不添加无信息量的注释（如 `// 索引` / `let i = 0` 这样的噪声注释）
3. **注释聚焦"为什么"，而非"是什么"** — 代码本身表达"做了什么"，注释表达"为什么这么做"或"有什么约束"
4. **不注释掉代码** — 不需要的代码直接删除，git 历史可找回

# 02-核心规则（语言无关）

以下规则是所有语言的共同基线，各语言章节在此基础上补充差异。

## 规则 1：结构体 / Class / 接口定义上方必须加注释

简短说明用 `//` 单行注释，较长说明允许使用 `/* */` 块注释。

```typescript
// 表示工作副本中的一个变更文件
interface FileItem {
```

```rust
/// 表示 SVN 命令执行后的输出结果
pub struct SvnOutput {
```

```java
/*
 * 负责与 SVN CLI 交互的执行器。
 * 封装了命令构建、超时控制、取消检测和错误处理。
 */
public class SvnExecutor {
```

## 规则 2：函数 / 方法定义上方必须加注释说明功能

函数有参数、返回值时，需要说明参数和返回值的用途。

**TypeScript：**

```typescript
// 根据文件路径获取差异内容，支持指定两个版本进行比较
// @param filePath - 要比较的文件路径
// @param oldRevision - 旧版本号（可选，省略时与本地工作副本比较）
// @param newRevision - 新版本号（可选）
// @returns 包含差异文本和统计信息的对象
async function getFileDiff(
  filePath: string,
  oldRevision?: number,
  newRevision?: number
): Promise<DiffResult> {
```

**Rust（使用 `///` doc comment，Rust 惯例）：**

```rust
/// 根据文件路径获取差异内容
///
/// # 参数
/// - `file_path`: 要比较的文件路径
/// - `old_revision`: 旧版本号（可选，省略时与本地工作副本比较）
/// - `new_revision`: 新版本号
///
/// # 返回
/// 包含差异文本和统计信息的 `DiffResult`
pub async fn get_file_diff(
    file_path: &str,
    old_revision: Option<u64>,
    new_revision: Option<u64>,
) -> Result<DiffResult, AppError> {
```

**Java：**

```java
/**
 * 根据文件路径获取差异内容
 *
 * @param filePath     要比较的文件路径
 * @param oldRevision  旧版本号，可选，省略时与本地工作副本比较
 * @param newRevision  新版本号
 * @return 包含差异文本和统计信息的 DiffResult
 * @throws AppException 执行失败时抛出
 */
public DiffResult getFileDiff(String filePath,
                              @Nullable Long oldRevision,
                              @Nullable Long newRevision) throws AppException {
```

## 规则 3：变量定义上方加注释说明用途

所有变量在定义时在上方加 `//` 注释说明用途。

**注意：** 当变量名 + 类型 + 赋值已足够清晰时，可以省略注释。判断标准：
- 变量名自文档化：`const userName = getUserName()` — 可省略
- 临时累加器/循环索引：`let i = 0` — 可省略
- 初始化值本身已说明用途：`const TIMEOUT_MS = 5000` — 可省略
- 复杂逻辑的中间结果、标志位、魔法数字、有业务含义的配置值 — **必须注释**

```typescript
// 已验证通过的用户会话信息
const session = await validateToken(token);

// 是否为管理员用户（同时也是部门负责人）
const isAdmin = user.role === 'admin' && user.isDeptHead;

// 计算当前操作预计剩余时间（秒）
const estimatedRemaining = (total - processed) * avgTimePerItem;
```

```rust
// 从请求头中提取的认证令牌
let token = extract_token(req)?;

// 是否为管理员用户（同时也是部门负责人）
let is_admin = user.role == Role::Admin && user.is_dept_head;
```

```java
// 计算当前操作预计剩余时间（秒）
long estimatedRemaining = (total - processed) * avgTimePerItem;
```

## 规则 4：常量和配置值上方必须加注释

```typescript
// 最大重试次数
const MAX_RETRY = 3;
// API 超时时间（毫秒）
const TIMEOUT_MS = 5000;
```

```rust
// SVN 命令默认超时秒数
const DEFAULT_TIMEOUT_SECS: u64 = 60;
// 单个日志文件的最大字节数
const MAX_LOG_FILE_SIZE: u64 = 5 * 1024 * 1024;
```

## 规则 5：所有控制流必须加注释说明条件

### if / else if / else

`if` 的注释放在**正上方**，`else` 的注释放在**右侧同行**。

```typescript
// 如果用户已登录且未过期
if (user.isLoggedIn && !user.isExpired) {
  return loadDashboard();

// 用户已登录但会话已过期
} else if (user.isLoggedIn && user.isExpired) {
  return showSessionExpired();

// 用户未登录（包括游客和未认证用户）
} else {
  return redirectToLogin();
}
```

```rust
// 如果文件存在且可读
if path.exists() && path.is_readable() {
    return read_file(path);

// 文件不存在，返回空结果
} else {
    return Ok(String::new());
}
```

### switch / match

注释放在 `switch` / `match` 正上方：

```typescript
// 根据文件状态确定显示颜色和操作按钮
switch (file.status) {
```

```rust
// 根据错误码类型确定前端显示方式
match error {
```

### for / while / do-while

注释放在循环语句正上方：

```typescript
// 遍历每个变更文件，累计修改行数
for (const file of changedFiles) {
```

```rust
// 逐行解析 SVN XML 输出
while let Ok(line) = reader.read_line(&mut buffer) {
```

### try-catch / finally

注释放在 `try` 正上方，`catch` 的注释放在右侧同行：

```typescript
// 执行 SVN 命令并将结果写入缓存
try {
  const result = await executeSvnCommand(args);

// 命令执行失败，回退到缓存数据
} catch (e: unknown) {
  console.error('[SvnService] 命令执行失败:', e);
  return loadFromCache();
}
```

```rust
// 尝试解析 SVN XML 输出
match serde_xml_rs::from_str::<SvnLog>(&xml_str) {
    Ok(log) => Ok(log),

    // 解析失败，记录原始 XML 以便诊断
    Err(e) => {
        log::error!("XML 解析失败: {}, 原始输出: {}", e, xml_str);
        Err(AppError::ParseFailed(e.to_string()))
    }
}
```

## 规则 6：注释前的空行规则

`//` 注释的上一行如果是同级的代码（而非上一段注释或空白），注释上方必须空行。

```typescript
const a = 1;

// 这是新的代码段，上方有空行
if (a > 0) {
```

多条 `//` 注释连续排列时，第一条的上方如果是同级代码才需空行，后续注释紧随即可：

```typescript
// 验证用户身份
// 检查会话是否过期
// 加载用户权限
```

## 规则 7：枚举 / 联合体每个变体上方加注释

```typescript
// 文件在 SVN 工作副本中的状态
export enum FileStatus {
  // 已修改（M）
  Modified = 'modified',
  // 已新增（A）
  Added = 'added',
  // 已删除（D）
  Deleted = 'deleted',
}
```

```rust
/// SVN 操作错误码
#[derive(Error, Debug)]
pub enum AppError {
    /// SVN 命令执行失败
    #[error("SVN_EXEC_FAILED: {0}")]
    SvnCommand(String),

    /// SVN XML 输出解析失败
    #[error("SVN_PARSE_FAILED: {0}")]
    ParseFailed(String),

    /// 操作被用户取消
    #[error("SVN_CANCELLED")]
    Cancelled,
}
```

# 03-前端注释规范（TypeScript / Vue / React）

## 注释格式

- **默认使用 `//` 单行注释**，禁止无理由使用 `/* */` 块注释
- 较长的类/接口说明或参数描述较复杂时，允许使用 `/** */` doc block
- JSDoc 风格的 `@param`、`@returns` 仅在参数较多（≥3 个）时使用

## Vue 组件

每个 `.vue` 文件的 `<script setup>` 顶部必须加注释说明组件的功能职责：

```vue
<script setup lang="ts">
// 文件变更列表表格组件：显示当前工作副本的变更文件，
// 支持多选、排序、搜索筛选、行操作按钮
interface Props {
  files: FileItem[]
  loading: boolean
}

const props = withDefaults(defineProps<Props>(), {
  loading: false,
})

const emit = defineEmits<{
  select: [paths: string[]]
  diff: [path: string]
}>()

// 当前选中的文件路径集合
const selectedPaths = ref<Set<string>>(new Set())

// 如果用户尚未选中任何文件，默认全选
if (selectedPaths.value.size === 0) {
  selectedPaths.value = new Set(props.files.map(f => f.path))
}
</script>
```

## React 组件

每个组件函数 / class 上方必须加注释：

```tsx
// 差异查看器组件：以 side-by-side 或 unified 模式显示文件差异
// 支持语法高亮和行号
function DiffViewer({ diffText, viewMode }: DiffViewerProps) {
  // 根据 viewMode 计算显示的差异内容
  const displayLines = useMemo(() => {
    // 如果是 unified 模式，过滤掉多余的空行
    if (viewMode === 'unified') {
      return diffText.split('\n').filter(line => line.trim() !== '');
    }
    return diffText.split('\n');
  }, [diffText, viewMode]);
```

## React Hooks

每个 hook 上方必须加注释说明其职责：

```tsx
// 监听文件变更事件，自动刷新文件列表
// @param refreshFn - 刷新的回调函数
// @returns 一个取消监听的清理函数
function useFileWatcher(refreshFn: () => void) {
```

## Pinia Store / 状态管理

每个 store 定义上方加注释：

```ts
// 工作副本状态管理：当前路径、分支名、版本号、最近打开记录
export const useWorkspaceStore = defineStore('workspace', () => {
```

## Ref / Reactive / useState

所有 `ref()`、`reactive()`、`useState()` 定义上方加注释：

```typescript
// 当前选中的文件路径集合
const selectedPaths = ref<Set<string>>(new Set())

// 文件列表加载状态
const [isLoading, setIsLoading] = useState(false)

// 搜索防抖定时器句柄
const searchTimer = ref<ReturnType<typeof setTimeout> | null>(null)
```

# 04-Rust 后端注释规范

## 注释格式

Rust 的注释格式需要兼顾生态惯例和本规范要求：

| 场景 | 格式 | 说明 |
|------|------|------|
| `struct` / `enum` / `trait` 定义 | `///` | 使用 doc comment，Rust 标准惯例 |
| `pub fn` 公开函数 | `///` | 使用 doc comment，生成 API 文档 |
| 私有函数 / `pub(crate)` | `///` 或 `//` | 复杂函数用 `///`，简单用 `//` |
| 模块级文档 | `//!` | 写在 `mod.rs` 或模块文件顶部 |
| 变量 / 常量 | `//` | 行内注释 |
| 控制流 | `//` | 正上方或右侧同行 |
| 行尾补充说明 | `//` | 简短说明放在行尾 |

## 函数注释

`pub fn` 使用 `///` doc comment，遵循 Rust 惯例：

```rust
/// 根据工作副本路径获取文件变更列表
///
/// # 参数
/// - `path`: 工作副本根路径
///
/// # 返回
/// 文件变更列表，每项包含路径、状态、工作副本版本号
///
/// # 错误
/// 返回 `AppError::SvnCommand` 当 SVN CLI 执行失败
pub async fn get_status(path: &str) -> Result<Vec<FileItem>, AppError> {
```

私有函数用 `//`：

```rust
// 解析 SVN status --xml 输出的单行记录
fn parse_status_entry(entry: &XmlEntry) -> Result<FileItem, AppError> {
```

## Struct / Enum 注释

```rust
/// SVN 操作进度信息
pub struct OperationProgress {
    /// 操作类型（checkout / update / export / merge）
    pub operation: String,
    /// 进度百分比（0-100）
    pub percent: u8,
    /// 当前阶段描述文字
    pub stage: String,
    /// 已处理的文件数
    pub file_count: u32,
}
```

## 控制流示例

```rust
// 检查当前操作是否已被用户取消
if CANCELLED.load(Ordering::SeqCst) {
    return Err(AppError::Cancelled);

// 操作未被取消，执行 SVN 命令
} else {
    // 构建 SVN 命令参数列表
    let mut args = vec!["status", "--xml"];
    // 如果指定了深度参数
    if let Some(depth) = depth {
        args.push("--depth");
        args.push(depth);
    }
}
```

## 允许的省略场景

以下场景注释可省略：
- getter 方法：`fn get_name(&self) -> &str`
- trait 实现中的显而易见的实现：`fn fmt(&self, f: &mut Formatter) -> fmt::Result`
- 测试函数：`#[test] fn test_parse_status_xml()`（测试函数名自文档化）
- 极其简单的 P0 基础设施函数（如返回常量的函数）

# 05-Java 后端注释规范

## 注释格式

| 场景 | 格式 | 说明 |
|------|------|------|
| Class / Interface 定义 | `/** */` | Javadoc 风格 |
| `public` 方法 | `/** */` | Javadoc 风格，含 `@param`、`@return`、`@throws` |

## 允许的省略场景

以下场景注释可省略：

# 06-注释位置汇总

| 元素 | 注释位置 | 示例 |
|------|----------|------|
| Class / Struct | 上方 | `// 描述\nclass Foo {` |
| 函数 / 方法 | 上方 | `// 描述\nfunction foo() {` |
| 变量 / 常量 | 上方 | `// 描述\nconst x = 1;` |
| `if` | 正上方 | `// 条件\nif (x) {` |
| `else` / `else if` | 右侧同行 | `} else { // 条件` |
| `switch / match` | 正上方 | `// 条件\nswitch (x) {` |
| `for` | 正上方 | `// 条件\nfor (...) {` |
| `while` | 正上方 | `// 条件\nwhile (...) {` |
| `try` | 正上方 | `// 描述\ntry {` |
| `catch` | 右侧同行 | `} catch { // 条件` |
| 枚举变体 | 上方或右侧 | `// 描述\n Modified,` |

# 07-常见反模式

| 反模式 | 问题 | 正解 |
|--------|------|------|
| 注释与代码重复 | 噪声，增加维护负担 | "设置计数器" 不如不写 |
| `if` 注释写在同行 | 违反位置规则 | 注释写在 `if` 正上方 |
| `else` 注释写在上一行 | 违反位置规则 | 注释写在 `else` 右侧同行 |
| 注释与上方代码无空行 | 视觉上归属模糊 | 注释上一行是同级代码时加空行 |
| 控制流无注释 | 读者需逆向推导条件意图 | 每个控制流加注释说明 |
| 注释掉的代码 | 死代码污染 | 删除，git 历史可找回 |
| 用 `/* */` 替换 `//` 做行注释 | 破坏视觉连续性 | 统一用 `//` |
| 修改代码但不更新注释 | 留言不正确，误导后来者 | 修改代码时同步更新注释 |

# 08-各语言注释格式速查表

| 场景 | TypeScript (Vue/React) | Rust | Java |
|------|----------------------|------|------|
| 行内注释 | `//` | `//` | `//` |
| Class / Struct | `//` 或 `/** */` | `///` | `/** */` |
| 公开函数 | `//` 或 `/** */` | `///` | `/** */` |
| 私有函数 | `//` | `//` | `//` |
| 模块 / 文件 | `//` | `//!` | `/** */` |
| 枚举变体 | `//` | `///` | `/** */` 或 `//` |
| 接口 / Trait | `//` 或 `/** */` | `///` | `/** */` |
| 控制流 | `//` | `//` | `//` |
| 参数说明 | `@param` 或行内 | `/// # 参数` | `@param` |
| 返回值说明 | `@returns` 或行内 | `/// # 返回` | `@return` |
