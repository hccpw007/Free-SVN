## §2 v2 新增内容质量评估

### 2.1 新增外部工具管理模块 (2.8)

| 子模块 | 实现内容 | 对齐依据 |
|--------|---------|---------|
| `tools/differ.rs` | 外部差异工具调用（不设超时、不捕获输出、分离进程） | 后端规范 "外部工具执行规范" |
| `tools/merger.rs` | 外部合并工具调用（mine/base/theirs/output 四个参数） | 业务设计 §5.7 外部合并路径传递 |
| `tools/resolver.rs` | 工具路径检测 + 失败回退到内置 diff2html + toast 提示 | 后端规范 §04 工具回退策略 |

**评价**：模块划分清晰，与现有 svn/ 模块职责分离正确。回退策略与设置页"外部工具不可用时回退到内置工具"开关联动，设计合理。

### 2.2 认证检测逻辑 (2.2)

**实现要点**：超时后检查 stderr 是否含 auth 关键词（"Authentication realm"/"Password:"/"Username:"）→ 返回 `SVN_AUTH_FAILED` 而非 `SVN_TIMEOUT`。

**评价**：
- 关键词覆盖了 svn CLI 常见的三种认证提示场景
- 错误码区分正确——让前端能分别显示"网络超时"和"认证失败"两种不同引导
- 错误码表新增 `SvnAuthFailed` 变体完整（Rust 枚举 + TS 类型 + i18n 翻译 + 验收标准）
- 局限（可接受）：基于关键词匹配而非交互式 stdin 检测，极端场景（如非英文 locale）可能漏检

### 2.3 status 缓存策略 (4.2)

**实现要点**：`fileListStore` 增加 `lastCachedFiles` ref，`isOperationRunning=true` 时 `refresh()` 返回缓存 + 轮询等待写操作完成后刷新。

**评价**：
- 与业务设计 §2.4 要求的"返回上次缓存的列表或等待写操作完成"完全对齐
- 缓存范围限定在写操作期间，不影响正常状态的实时刷新
- 轮询策略确保写操作完成后能自动获取最新结果
- 未指定轮询间隔——建议编码时使用 500ms~1000ms 间隔

### 2.4 事件协议表 (新增 §技术参数)

**内容**：完整列出 `operation:started/progress/completed/error` 四个事件的方向、用途和使用位置。

**评价**：所有长操作步骤（3B.1/3B.2/3C.1/3C.4）已逐个标注使用哪些事件，消除了 v1 报告指出的引用不完整问题。

### 2.5 错误码表 (新增 §技术参数)

**评价**：13 个错误码（含新增的 `SVN_AUTH_FAILED`）全部列出 Rust 变体、用途和前端行为，与后端规范 §03 表完全对齐。
