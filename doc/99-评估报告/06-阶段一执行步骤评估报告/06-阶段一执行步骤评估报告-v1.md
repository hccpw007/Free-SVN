#### ❌ P1 缺项（必须修复）

| # | 缺项 | 来源 | 具体影响 |
|---|------|------|---------|
| **1.3-A** | **未配置 `vite.config.ts` 的 `@` 路径别名** | 开发计划 v2 1.4 | 后续所有 `import xx from '@/components/xxx'` 将报模块未找到错误。AI 如果没有此配置会自动改用相对路径，但前后不一致 |
| **1.3-B** | **未确认 `tsconfig.json` strict 模式** | 开发计划 v2 1.4 | Tauri 初始化后 `strict` 可能为 false。未开启 strict 则 TS 类型检查宽松，null/undefined 隐患被遗漏 |
| **1.3-C** | **未确认 `App.vue` 为仅 `<RouterView />` 模式** | 开发计划 v2 1.4 | Tauri+Vue 初始化模板默认生成带内容的 App.vue（计数器示例），AI 可能保留模板内容，导致 `RouterView` 嵌套异常 |

#### ⚠️ P2 建议

| # | 建议 | 说明 |
|---|------|------|
| 1.3-D | 字体文件存放路径不明确 | style.css `@font-face` 的 `src: url('/fonts/xxx.woff2')` 指向 Vite 的 `public/fonts/` 目录，但步骤 1.2 创建的是 `src/fonts/`。需要明确字体文件应放在 `public/fonts/` 才能被 `url('/fonts/...')` 正确解析 |

## §3 改进建议汇总

### 必须修复（P1 — 修复后 AI 才能正确编码）

| # | 修复位置 | 建议修改 |
|---|---------|---------|
| **1.3-A** | 步骤 1.3 的涉及文件中增加 `vite.config.ts`（**修改**） | 追加 `@` 路径别名：`resolve: { alias: { '@': __dirname + '/src' } }`（注意 Vite 6 使用 `import.meta.url` 或 `path.resolve`）。确保 `import xx from '@/...'` 可解析 |
| **1.3-B** | 步骤 1.3 的涉及文件中增加 `tsconfig.json` 和/或 `tsconfig.app.json`（**修改**） | 确认 `compilerOptions: { strict: true }`；如果使用 `@` 别名还需配置 `paths: { "@/*": ["src/*"] }` 和 `baseUrl: "."` |
| **1.3-C** | 步骤 1.3 中增加一句说明或独立子步骤 | 确保 `App.vue` 内容仅为 `<template><RouterView /></template>`，删除初始化模板的多余内容。可放在 main.ts 修改之后 |

### 建议优化（P2）

| # | 修复位置 | 建议 |
|---|---------|------|
| 1.1-A | 步骤 1.1 Cargo.toml 追加列表 | 显式加 `log = "0.4"` |
| 1.3-D | 步骤 1.2 或 1.3 明确字体路径 | 确认字体文件放在 `public/fonts/` 目录（Vite 静态资源），步骤 1.2 改为创建 `public/fonts/` 而不是 `src/fonts/` |

## §4 AI 可编码性专项评估

依据 CLAUDE.md 的评估标准——"每个执行计划内容都能够让 AI 编写出想要的效果不走样"：

| 标准 | 评估 |
|------|------|
| 涉及文件路径是否完整 | ✅ 每步都列出所有文件路径 |
| 操作细节是否具体而非需求目标 | ✅ 代码块包含具体代码内容（Rust struct/fn、TS config、JSON 翻译）|
| 验收标准是否可验证 | ✅ 所有标准可执行命令或精确检查 |
| AI 是否有自由发挥空间 | ⚠️ 1.3-A/B/C 三个缺项会让 AI 作出默认假设（保留模板 App.vue、使用相对路径 import、宽松 tsconfig），必须修复 |
| 操作顺序是否合理 | ✅ 后端 → 前端依赖 → 前端配置 → 构建验证，无循环依赖 |
| todo!() 占位是否合理 | ✅ 所有 `todo!()` 返回值类型与函数签名一致，Rust 编译器可接受 |
