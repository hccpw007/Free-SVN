## 二、问题列表

### 🔴 P1 — 建议编码前确认（可能影响编码正确性）

**P1-01：FileListTable 中 el-table selection 列与手动 Set 管理的选中状态冲突**

- **涉及步骤**：`6.2/FileListTable.vue`
- **问题描述**：FileListTable 模板使用了 `<el-table-column type="selection" />`（el-table 内置的复选框列，自带选中状态管理），但同时 `fileListStore.selectedPaths` 使用 `Set<string>` 手动管理和维护选中路径。这两个机制会相互干扰：el-table 的 `selection` 列的选中状态不会自动同步到 `Set`，反之 `toggleSelect()` / `toggleSelectAll()` 手动操作 `Set` 不会反映到 el-table 的 UI 上。
- **影响范围**：全选/取消全选/提交面板的选择状态传递均依赖此选中状态，可能导致提交时缺少或多选文件
- **建议修复**：选择以下方案之一：① 使用 el-table 的 `@selection-change` 事件同步到 `Set`；② 完全使用自定义复选框 + `Set` 管理（移除 `type="selection"`）；③ 混合方案：el-table 提供 UI，通过 `@selection-change` 将变化同步到 `fileListStore.selectedPaths`

**P1-02：Tailwind 4 的 arbitrary variants 语法兼容性**

- **涉及步骤**：`6.3/DiffViewer.vue`
- **问题描述**：DiffViewer 中使用 `[&_.d2h-file-wrapper]:border-0` 的 arbitrary group 语法来隐藏 diff2html 的文件头边框，但项目中标注使用 Tailwind 4。Tailwind 4 中 arbitrary variant 的语法已变更为 `@` 前缀（`@[&_.d2h-file-wrapper]:border-0`），旧的 `[&...]` 语法在 Tailwind 4 中可能不再生效。
- **影响范围**：diff 面板外观可能不符合设计
- **建议修复**：在编码阶段先确认 Tailwind 4 实际版本和 arbitrary variant 支持情况。如果 Tailwind 4 不支持此类选择器，改用 `style.css` 中的全局 CSS 覆盖 diff2html 默认样式

### 🟡 P2 — 建议编码中注意（不影响启动但影响正确性）

**P2-01：ResolveConflict 组件的 `viewDiff` emit 未声明**

- **涉及步骤**：`6.8/ResolveConflict.vue`
- **问题描述**：ResolveConflict 对话框的模板中使用 `$emit('viewDiff', selectedFile)` 来通知父组件打开差异对比，但 `defineEmits` 中没有声明 `viewDiff` 事件。TypeScript 在严格模式下会报类型错误。
- **建议修复**：在 `defineEmits<{ viewDiff: [path: string] }>()` 中声明

**P2-02：ResolveConflict 无冲突文件时的 UI 状态缺失**

- **涉及步骤**：`6.8/ResolveConflict.vue`
- **问题描述**：`conflictFiles` 为空时对话框仍可打开但呈现无数据状态。
- **建议修复**：在 ResolveConflict 的 `onMounted` 中检查 `conflictFiles.length`，为空时自动关闭

**P2-03：IgnoreDialog 的 svn:ignore 设置在目录上而非文件上**

- **涉及步骤**：`6.8/IgnoreDialog.vue`
- **问题描述**：`svn:ignore` 属性只能设置在**目录**上。如果用户选中的是文件路径，则 propset 目标应该是其父目录。
- **建议修复**：在 IgnoreDialog 或 `setIgnore` command 中，对文件路径自动取父目录

**P2-04：CommitPage 的 svnStore.commit() 返回类型与使用不一致**

- **涉及步骤**：`6.4/CommitPage.vue`
- **问题描述**：CommitPage 中假设 commit 返回版本号数字，但 `stores/svn.ts` 中声明返回 `OperationResult` 对象。
- **建议修复**：统一 CommitPage 中对 commit 返回值的处理

**P2-05：services/svn.ts 的 translateError 抛出 string 而非 Error**

- **涉及步骤**：`7.2/services-svn完整封装.md`
- **问题描述**：`throw translateError(msg)` 抛出字符串而非 `Error` 对象。如果调用方使用 `err instanceof Error` 判断，字符串异常会导致 catch 分支无法正确捕获。
- **建议修复**：改为 `throw new Error(translateError(msg))`

**P2-06：CommitForm 的最近日志下拉 v-model 绑定使用不当**

- **涉及步骤**：`6.4/CommitForm.vue`
- **问题描述**：最近日志下拉使用 `v-model="recentMessages[0]"` 绑定到数组第一个元素，v-model 变更不会正确触发选中状态更新。
- **建议修复**：使用独立的 `selectedRecentMessage` ref

## 七、综合建议

### 7.1 编码前建议确认的要点

| 优先级 | 问题 | 影响范围 | 建议 |
|--------|------|---------|------|
| P1-01 | FileListTable 选中状态管理冲突 | 全选/提交选择传递 | 编码时选择 el-table `@selection-change` + Set 同步方案 |
| P1-02 | Tailwind 4 arbitrary variant 兼容性 | DiffViewer 外观 | 编码时先验证 Tailwind 4 版本 |

### 7.2 编码中建议注意的要点

| 优先级 | 问题 | 影响范围 | 建议 |
|--------|------|---------|------|
| P2-01 | ResolveConflict emit 未声明 | 编译错误 | 在 defineEmits 中声明 |
| P2-02 | 无冲突文件状态 | UI 异常 | onMounted 检查长度 |
| P2-03 | Ignore svn:ignore 目标应为目录 | 忽略功能异常 | 自动取文件父目录 |
| P2-04 | commit 返回类型不一致 | Toast 显示异常 | 统一处理 |
| P2-05 | translateError 抛出 string | 错误处理异常 | 改为 throw new Error() |
| P2-06 | CommitForm v-model 绑定 | 下拉不工作 | 改用独立 ref |
