## 二、问题列表

### 🔴 P1 — 必须修正（影响编码正确性）

**P1-NEW-01（关键）：executor.rs 中 `run_svn` 未将 credentials 传递到 `run_svn_sync`**

- **涉及步骤**：`2.2 SVN CLI 执行核心`
- **文件**：`src-tauri/src/svn/executor.rs`
- **问题描述**：`run_svn()` 函数的 `credentials: Option<&SvnCredentials>` 参数在构造 `all_args` 时正确使用（追加 `--username`、移除 `--non-interactive`），但在 `timeout` + `spawn_blocking` 闭包中调用 `run_svn_sync()` 时**始终传入 `None`**：

```rust
// run_svn() 中的关键代码行
let has_creds = credentials.is_some();  // ← 定义了但从未使用的变量
let result = timeout(
    Duration::from_secs(timeout_secs),
    spawn_blocking(move || {
        run_svn_sync(&svn_path, &all_args, &cwd, None)  // ← BUG: 始终传 None
    }),
).await;
```

- **根因**：`credentials` 是引用类型（`Option<&SvnCredentials>`），不能直接捕获到 `spawn_blocking` 的 `'static` 闭包中。`SvnCredentials` 已派生 `Clone`，需要在进入闭包前克隆。
- **影响范围**：**所有带凭据的 SVN 操作**（检出认证、认证重试、凭据管理的测试连接/保存凭据）均无法正常工作。密码通过 `--password-from-stdin` 管道传递的机制完全失效。svn 会在无 `--non-interactive` 且无 stdin 密码的情况下交互式等待输入，最终超时。
- **修复方案**：
  1. 在 `spawn_blocking` 前克隆 credentials：`let creds = credentials.cloned();`
  2. 在闭包中传入 `creds.as_ref()`：`run_svn_sync(&svn_path, &all_args, &cwd, creds.as_ref())`
  3. 删除未使用的 `has_creds` 变量

### 🟡 P2 — 建议修正（影响代码质量或一致性）

**P2-NEW-01：`stores/fileList.ts` 缺少 `applyFilter()` 和 `clearFilter()` 方法**

- **涉及步骤**：`4.2 Pinia 状态管理` → 被 `6.1 HomePage` 引用
- **问题描述**：`6.1-HomePage.vue` 的搜索框模板中调用了 `fileListStore.applyFilter()` 和 `fileListStore.clearFilter()`，但 `4.2` 定义的 `useFileListStore` 中**未实现这两个方法**。`filteredFiles` 虽然是一个 `computed` 响应式计算属性，但 `applyFilter()` 不存在会在 Vue 模板执行时报运行时错误。`clearFilter()` 也不存在（应该重置 `searchQuery` 和 `filterStatus`）。
- **建议修复**：在 `4.2/fileList.ts` 中添加 `applyFilter()` 和 `clearFilter()`（重置 `searchQuery` 和 `filterStatus`）方法。

**P2-NEW-02：`stores/svn.ts` commit 方法的参数与 `CommitPage.vue` 调用不匹配**

- **涉及步骤**：`4.2 stores/svn.ts` vs `6.4 CommitPage`
- **问题描述**：`4.2` 中 `svnStore` 的 `commit` 方法签名为 `commit(paths: string[], message: string, keepLocks?: boolean)`（接受 3 个分开参数），但 `6.4 CommitPage.vue` 中调用方式为：`svnStore.commit({ paths: [...], message: '...', keepLocks: false })`（传单个对象）。TypeScript 编译时会报类型错误。类似的调用不一致也存在于 `getLog`。
- **建议修复**：统一为对象参数模式，并更新 `4.2` 中的函数签名。

**P2-NEW-03：MergeWizard 前端使用 `MergeResult` 类型与后端返回的 `OperationResult` 不匹配**

- **涉及步骤**：`6.7 MergeWizard` vs `4.2 stores/svn.ts` vs `3.3 branch_ops.rs`
- **问题描述**：`6.7 MergeWizard.vue` 中调用 `svnStore.mergeBranch()` 后访问 `r.conflicts?.length`，但后端 `merge_branch` command 返回 `OperationResult`，不含 `conflicts` 字段。
- **建议修复**：统一返回类型或在 `detail` 中获取冲突信息。

**P2-NEW-04：`executor.rs` 中 `CURRENT_CHILD` 未在 `run_svn_sync()` 中设置**

- **涉及步骤**：`2.2 SVN CLI 执行核心`
- **问题描述**：`cancel_operation` 命令依赖 `CURRENT_CHILD` 全局变量来 kill 子进程。但 `run_svn_sync()` 中**没有将 spawn 出的 child 保存到 `CURRENT_CHILD`**。
- **建议修复**：将 `child` 保存到 `CURRENT_CHILD`。

**P2-NEW-05：Step 4.2 stores/settings.ts 直接使用 `invoke` 与 Step 7.2 的 "唯一 invoke 入口" 原则矛盾**

- **涉及步骤**：`4.2 stores/settings.ts` vs `7.2 services/svn.ts`
- **问题描述**：Step 4.2 标注"本 store 在步骤 7.2 完成后应重构"，但 Step 7.2 中未包含 settings store 的重构说明。
- **建议修复**：在 7.2 中补充 settings store 的重构步骤。

**P2-NEW-06：Step 6.6 CheckoutDialog 缺少完整的认证输入校验代码**

- **涉及步骤**：`6.6 对话框组一 CheckoutDialog`
- **问题描述**：交互设计 v5 §3.7 定义了的认证输入校验逻辑步骤中仅用描述性文字提及，未提供具体代码。
- **建议修复**：补充认证输入区校验逻辑的伪代码或函数签名描述。
