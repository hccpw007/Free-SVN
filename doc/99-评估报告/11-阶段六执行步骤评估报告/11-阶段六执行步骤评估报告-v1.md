#### P1 — 右键菜单"在文件管理器中显示"/"用外部编辑器打开"为空函数

开发计划 v2 6A.3 明确要求此两项调用 `Tauri shell.open`。但步骤代码中这两个按钮只执行 `ctxMenu.show = false`，无实际的文件管理/编辑器打开行为。AI 按此编码将产出无功能的按钮。

**纠正：** 需导入 `@tauri-apps/plugin-shell` 并调用 `open()`。

#### P1 — "复制文件路径"和"复制相对路径"调用同一函数

两个右键菜单按钮都调用了 `copyFullPath(ctxMenu.file.path)`。`file.path` 为相对路径。
- "复制文件路径"应拼接 `workspaceStore.currentPath + '/' + file.path`
- "复制相对路径"才直接复制当前值

#### P2 — 缺少 Shift+Click / Ctrl+Click 多选

开发计划 v2 6A.3 要求范围选择和切换选。`handleRowClick` 仅实现了 toggle。文件选择 completable 在阶段七实现，但步骤未标注哪些部分依赖阶段七。

#### P2 — `border-l-3` 在 Tailwind 中不精确

Tailwind v3 中无 `border-l-3`（默认 0/2/4/8），v4 中 `3` 对应间距 12px 非 3px 边框。建议改为 `border-l-[3px]`。

#### P1 — `#expand` 插槽放在 `</el-table>` 外部

```vue
</el-table>
<template #expand="{ row }">
  <LogDetail />
</template>
```
`#expand` 是 ElTable 插槽，必须位于 `<el-table>` 内部。当前放在 `</el-table>` 后面，Vue 不会渲染。应改为：
```vue
<el-table>
  ...列定义...
  <template #expand="{ row }"><LogDetail /></template>
</el-table>
```

#### P2 — 使用 `el-table` 而非 `el-table-v2`（虚拟滚动）

开发计划 v2 6C.1 明确要求 `El-table-v2` 虚拟滚动，视觉设计 §3.5 要求"日志列表使用虚拟滚动（Element Plus 虚拟化表格）"。步骤使用了普通 `el-table`，大数据量时性能不足。

视觉设计 §3.5 要求"展开行有过渡动画（200ms ease-out）"。步骤未配置 ElTable 展开行动画。

#### P1 — ConfirmDialog `defineProps` 定义两次（编译错误）

```vue
defineProps<{ title: string; ... }>()                    // 第一次
withDefaults(defineProps<{ title: string; ... }>(), {...}) // 第二次
```

两次 `defineProps` 在 `<script setup>` 中是无效语法，会产生编译错误。应合并为：
```vue
const props = withDefaults(defineProps<{ ... }>(), { ... })
```

#### P2 — CheckoutDialog 缺少"仅此目录"复选框

视觉设计 §3.7 检出对话框中有"仅此目录（不包括子目录）"复选框（对应 `--depth empty`），但步骤只提到 `ignoreExternals`，"仅此目录"缺失。
