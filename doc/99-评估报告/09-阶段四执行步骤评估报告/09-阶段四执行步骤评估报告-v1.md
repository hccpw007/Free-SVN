## §2 关键发现

### 🔴 P1 — 状态筛选值不匹配（4.2B fileList.ts）

```ts
const statusMap: Record<string, string[]> = {
  modified: ['M'],            // ← 单字母
  added: ['A', '+'],
  conflicted: ['C', '!'],
  deleted: ['D', '🗑'],
  unversioned: ['?'],
}
```

但阶段二 `parse_status()` 解析 `svn status --xml`，`FileItem.status` 的值是 **XML 属性全称**：

```rust
// parser.rs
#[serde(rename = "@item")]
item: String,  // → "modified", "added", "conflicted", "deleted", "unversioned"
```

**影响**：每个筛选条件都匹配不到任何文件，筛选下拉完全失效。

**原因**：视觉设计文档 §3.2 的 `M`/`✚`/`!`/`🗑`/`?` 是 **UI 展示图标**，不是底层数据值。底层数据来自 `svn status --xml` 的 `@item` 属性。

**修复**：
```ts
const statusMap: Record<string, string[]> = {
  modified: ['modified'],
  added: ['added'],
  conflicted: ['conflicted'],
  deleted: ['deleted'],
  unversioned: ['unversioned'],
}
```

---

### 🟡 P2 — 缺少 locale 动态切换逻辑

settings store 有 `language` 字段，i18n `createI18n({ locale: 'zh-CN' })` 初始化为中文。但步骤没有实现 settings.language → vue-i18n locale 的联动。

**影响**：用户在设置页切换语言后，实际 UI 语言不会变化。

**建议**：在 settings store 的 `load()` 或 App.vue 的 `onMounted` 中增加：
```ts
import { useI18n } from 'vue-i18n'
const { locale } = useI18n()
if (settingsStore.language && settingsStore.language !== 'system') {
  locale.value = settingsStore.language
}
```
