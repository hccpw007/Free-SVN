## §2 关键变更代码审查

### 2.2 executor.rs — run_svn_sync() 重写

**Before（编译错误）：**
```rust
*guard = Some(child.try_wait().ok().flatten()
    .map(|_| child)        // child moved here
    .unwrap_or(child));    // use of moved value
let output = child.wait_with_output()?;  // again
```

**After（正确）：**
```rust
// 同步等待子进程完成（不保存 child 到 CURRENT_CHILD）
// 超时由外层 run_svn() 的 tokio::time::timeout 控制
let output = child.wait_with_output()?;
```

✅ **编译通过。** 增加注释说明超时由外层控制。无所有权冲突。

### 2.2 executor.rs — get_svn_path() 简化

**Before（Tauri 1 API，Release 编译失败）：**
```rust
#[cfg(not(debug_assertions))]
{ let resource_dir = tauri::api::path::resource_dir()... }
```

**After（无 API 依赖）：**
```rust
pub fn get_svn_path() -> PathBuf {
    PathBuf::from("svn")  // 阶段七改为内置路径
}
```

✅ **Debug/Release 均编译通过。**

## §3 已知设计取舍（确认可接受）

| 取舍 | 说明 | 可接受理由 |
|------|------|-----------|
| 超时场景无法检测认证提示 | CURRENT_CHILD 未存 child 句柄，超时后无法读 stderr | ✅ `--non-interactive` 使认证失败立即退出+输出错误，通过常规错误路径捕获 |
| 超时后子进程可能 orphan | `kill_current_process()` 读 CURRENT_CHILD 为 None 时无操作 | ✅ 短命令很少超时；长操作在阶段三有独立的 child 句柄管理 |
