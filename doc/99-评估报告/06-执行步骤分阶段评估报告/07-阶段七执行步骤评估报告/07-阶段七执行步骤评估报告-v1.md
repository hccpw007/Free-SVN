#### P2 — useKeyboardShortcuts 缺少 unregister 机制

`allShortcuts` 是模块级全局数组，`register()` 向其中 push 条目但无对应 `unregister()`。组件卸载后 handler 持续留在数组中响应事件。当组件被销毁后重新创建（如热替换），handler 会重复注册。

**建议：** 在 Shortcut 中添加 `id` 字段 + 返回 `unregister(id)` 函数。

#### P2 — useFileSelection 第 83 行存在无操作表达式

```ts
items[idx].path; s.has(items[idx].path) ? s.delete(items[idx].path) : s.add(items[idx].path)
```
`items[idx].path;` 取值后丢弃结果，是重构残留。应删除该行。

#### P2 — useOperationGuard 中 Pinia store 调用位置不规范

```ts
const blocked = computed(() => useFileListStore().isOperationRunning)
```
Pinia 推荐顶层调用 store，改为：
```ts
const fileListStore = useFileListStore()
const blocked = computed(() => fileListStore.isOperationRunning)
```

#### P2 — 示例代码中缺少 try-catch 和错误码翻译

开发计划 v2 7.5 明确要求"统一 try-catch + 错误码翻译(i18n key)"，但示例代码：
```ts
export async function getStatus(path: string): Promise<FileItem[]> {
  return invoke('get_status', { path })
}
```
无任何 try-catch。AI 按此字面编码将缺失统一错误处理。

**建议：** 添加通用包装函数：
```ts
async function wrappedInvoke<T>(cmd: string, args?: Record<string, unknown>): Promise<T> {
  try { return await invoke<T>(cmd, args) }
  catch (e) { throw translateError(e) }
}
```

#### P2 — 三个平台具体注册/清理代码完全缺失

mod.rs 有完整 Rust 代码（`register/unregister` + `#[cfg]`），但 macos.rs / windows.rs / linux.rs **完全无代码**。三平台各自的原生注册方式：
- macOS：`/Library/Services/` 或 `.workflow` 文件
- Windows：注册表 `HKEY_CLASSES_ROOT\Directory\Background\shell\` 键值
- Linux：Nautilus 脚本 `~/.local/share/nautilus/scripts/`

是步骤中最核心的工作量，但完全留空。
