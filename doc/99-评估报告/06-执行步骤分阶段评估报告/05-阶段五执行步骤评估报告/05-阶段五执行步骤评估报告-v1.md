### 步骤 5.2 — TopBar 顶部状态栏组件

| 检查维度 | 结果 | 说明 |
|---------|------|------|
| 涉及文件完整性 | ✅ | 唯一文件 `src/components/common/TopBar.vue` |
| 操作细节可执行性 | ❌ | 存在 P1 问题 |
| 验收标准可验证性 | ⚠️ | 存在 P2 遗漏项 |

#### P1 — Lucide 图标使用方式与项目 tech stack 不匹配

项目 CLAUDE.md 明确标注前端使用 `lucide-vue-next`，但步骤 5.2 的代码中使用 `<svg data-lucide="git-branch" />` 方式（这是 `lucide` Web 组件库的用法，与 vue-next 不兼容）。

- **影响**：AI 按此编码会产生编译错误或图标完全不显示
- **纠正**：应改为 `lucide-vue-next` 的 Vue 组件方式：
  ```vue
  <script setup lang="ts">
  import { GitBranch, Moon, Sun } from 'lucide-vue-next'
  </script>
  ```
- **波及步骤**：此问题同时影响 5.2（TopBar）、5.3（ToolBar）

#### P2 — 无工作副本时缺少"Free-SVN"应用名显示

验收标准第 7 条要求"无工作副本时仅显示 Free-SVN"，但代码中 `v-if="workspaceStore.currentPath"` 分支没有对应的 `v-else` 显示应用名。当前代码无工作副本时左边区域仅显示 git-branch 图标。

- **建议**：添加 `v-else` 分支显示应用名：
  ```vue
  <template v-else>
    <span class="text-sm font-semibold text-slate-800 dark:text-slate-200">
      Free-SVN
    </span>
  </template>
  ```

#### P2 — clipboard fallback 未实现

`copyPath` 函数的 catch 块只有注释 `// fallback: 使用 textarea 方案` 但没有实现代码。桌面应用中 `navigator.clipboard` 因权限问题在 Tauri 中可能不工作，需要有实际的 fallback 实现。

#### P1 — 同 5.2，Lucide 图标使用方式

步骤 5.3 同样使用 `<svg data-lucide="..." />` 方式，需改为 `lucide-vue-next` 组件方式。因与 5.2 为同一问题，恢复代价低（全局替换）。

#### P2 — 按钮禁用 tooltip 未区分具体原因

交互设计 §4.1 要求不同按钮有不同的禁用 tooltip：
- 提交 → "当前工作副本无变更"
- 更新/日志/切换等 → "无可用工作副本"

但步骤 5.3 实现了统一的 `"操作进行中，请等待"` 和 `btn.label`，缺少对"无变更"场景的细分。建议为提交按钮增加 `workspaceStore.hasChanges` 判断（需在 workspace store 或 fileList store 中增加该状态）。
