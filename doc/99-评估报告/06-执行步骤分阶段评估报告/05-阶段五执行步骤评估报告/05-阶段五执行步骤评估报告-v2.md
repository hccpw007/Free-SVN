## v1 问题修复验证

### P1 — Lucide 图标使用方式 ❌ → ✅

| 步骤 | v1 状态 | v2 状态 | 修复内容 |
|------|---------|---------|---------|
| 5.2 TopBar | `<svg data-lucide="...">` | `<GitBranch /> <Moon /> <Sun />` | 改用 `lucide-vue-next` Vue 组件方式，正确 import |
| 5.3 ToolBar | `<svg data-lucide="...">` | `<component :is="iconMap[]">` + `MoreHorizontal` 组件 | 动态组件模式，全部 11 个图标枚举在 `iconMap` |

### P2 — 无工作副本时缺少"Free-SVN"文字 ❌ → ✅

步骤 5.2 添加了 `v-else` 分支：
```vue
<template v-else>
  <span class="text-sm font-semibold text-slate-800 dark:text-slate-200">Free-SVN</span>
</template>
```
验收标准同步新增第 7 条。

### P2 — clipboard fallback 未实现 ❌ → ✅

步骤 5.2 `copyPath` 的 catch 块新增完整 textarea fallback：
```ts
const textarea = document.createElement('textarea')
textarea.value = workspaceStore.currentPath
// ...
document.execCommand('copy')
```

### P2 — 按钮禁用 tooltip 未区分 ❌ → ✅

步骤 5.3 新增三层 tooltip 判定逻辑（`getDisabledTooltip()`）：
1. `globallyDisabled` → "操作进行中，请等待"
2. `wpUnavailable` → "无可用工作副本"
3. 提交按钮单独检查 `fileListStore.files.length === 0` → "当前工作副本无变更"
4. 其他按钮统一使用两步判定

验收标准新增第 7 条："提交按钮 tooltip 区分三种场景"。

### P2 — StatusBar 缺少 tooltip ❌ → ✅

步骤 5.4 为三段信息全部添加了 `:title` 属性：
```vue
<span :title="commitTime">{{ commitTime }}</span>
<span :title="svnVer">{{ svnVer }}</span>
```
验收标准同步更新为"三段均有 title 悬停 tooltip"。
