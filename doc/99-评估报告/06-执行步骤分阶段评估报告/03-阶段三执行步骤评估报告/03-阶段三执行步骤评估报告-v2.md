## §2 关键变更代码审查

### `update_workspace()` 锁序修复（3.2B）

**Before（死锁风险）：**
```rust
state.try_lock()?;      // ← 先锁
let info = parse_info(&run_svn(&["info","--xml"]).await?).await?;
check_network(&info.url).await?;  // ← 后检测，失败→锁永不放
```

**After（正确）：**
```rust
// 先获取远程 URL 检测网络（info 只读操作不需要写锁）
let info_xml = svn::executor::run_svn(&["info", "--xml"], &params.path).await?;
let info = svn::parser::parse_info(&info_xml)?;
svn::executor::check_network(&info.url).await?;

state.try_lock()?;     // ← 所有检查通过后才锁
```

✅ 与 `checkout_repo`（3.2A）锁序一致，均遵循"先检测网络 → 后获取锁"的模式。
