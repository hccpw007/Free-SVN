## 操作细节

### 5.3A — 新建 `src/components/common/ToolBar.vue`

高度 40px（h-10），背景 `bg-white dark:bg-slate-800`，底部 1px 边框。使用 `<script setup lang="ts">` 语法，Lucide 图标（无 emoji）。

**完整按钮列表（11 个）：**

| 标签（i18n key） | Lucide 图标 | 操作 | 禁用条件 | 禁用 tooltip |
|------|------------|------|---------|-------------|
| toolbar.commit | GitCommit | router.push('/workspace/commit') | 无工作副本 / 无变更 | "无可用工作副本" / "当前工作副本无变更" |
| toolbar.diff | FileDiff | router.push('/workspace/diff') | 无工作副本 / 未选中文件 | "无可用工作副本" / "请先选中文件" |
| toolbar.update | RefreshCw | svnStore.update() | 无工作副本 | "无可用工作副本" |
| toolbar.updateToRevision | ArrowUpCircle | 打开 UpdateRevisionDialog | 无工作副本 | "无可用工作副本" |
| toolbar.log | History | router.push('/workspace/log') | 无工作副本 | "无可用工作副本" |
| toolbar.switch | GitBranch | 打开 SwitchDialog | 无工作副本 | "无可用工作副本" |
| toolbar.branchTag | Layers | 打开 BranchTagDialog | 无工作副本 | "无可用工作副本" |
| toolbar.merge | GitMerge | 打开 MergeWizard | 无工作副本 | "无可用工作副本" |
| toolbar.cleanup | Broom | svnStore.cleanup() | 无工作副本 | "无可用工作副本" |
| toolbar.export | Package | 打开 ExportDialog | 无工作副本 | "无可用工作副本" |
| toolbar.settings | Settings | router.push('/settings') | 始终可用 | — |
| toolbar.moreOperations | MoreHorizontal | — | 同按钮 | — |

**溢出策略：** ResizeObserver 监听容器宽度，每个按钮约 88px，至少保留"提交"和"设置"可见。
**禁用态规则（四层判断）：**
1. `isOperationRunning=true` → 全部禁用，tooltip="操作进行中，请等待"（最高优先级）
2. 无工作副本 → 全部禁用（设置除外），tooltip="无可用工作副本"
3. 提交按钮：无工作副本 + 无变更时 → tooltip="当前工作副本无变更"
4. diff 按钮：无工作副本 + 未选中文件（`fileListStore.selectedPaths` 为空）时 → tooltip="请先选中文件"

**核心代码（全组件骨架，使用 `lucide-vue-next` 组件模式）：**

```vue
<script setup lang="ts">
import { ref, computed, onMounted, onUnmounted, type Component } from 'vue'
import { useRouter } from 'vue-router'
import { GitCommit, FileDiff, RefreshCw, ArrowUpCircle, History, GitBranch, Layers, GitMerge, Broom, Package, Settings, MoreHorizontal } from 'lucide-vue-next'
import { useWorkspaceStore } from '@/stores/workspace'
import { useFileListStore } from '@/stores/fileList'
import { useSvnStore } from '@/stores/svn'
import { useI18n } from 'vue-i18n'

const { t } = useI18n()
const router = useRouter()
const workspaceStore = useWorkspaceStore()
const fileListStore = useFileListStore()
const svnStore = useSvnStore()

// 工具栏按钮事件（父组件处理对话框打开）
const emit = defineEmits<{
  'open-dialog': [dialog: string]
}>()

const toolbarRef = ref<HTMLElement | null>(null)
const overflowCount = ref(0)

// 严格索引类型：定义 IconKey 联合类型，用于 iconMap 的类型安全索引
// 取值必须与下面 iconMap 中的键名完全一致（大小写敏感）
type IconKey = 'GitCommit' | 'FileDiff' | 'RefreshCw' | 'ArrowUpCircle' | 'History'
  | 'GitBranch' | 'Layers' | 'GitMerge' | 'Broom' | 'Package'
  | 'Settings' | 'MoreHorizontal'

// lucide-vue-next 图标组件映射（用于动态渲染）
// Record<IconKey, Component> 确保 iconKey 只能是上述联合类型中的值，无需 as 类型断言
// 具体包含以下 12 个图标组件（按对应按钮优先级从高到低）：
//   GitCommit       — 提交按钮
//   FileDiff        — 差异比较按钮（v5 新增，对应 diff 按钮）
//   RefreshCw       — 更新按钮
//   ArrowUpCircle   — 更新到版本按钮
//   History         — 日志按钮
//   GitBranch       — 切换分支按钮
//   Layers          — 分支/标签按钮
//   GitMerge        — 合并按钮
//   Broom           — 清理按钮
//   Package         — 导出按钮
//   Settings        — 设置按钮
//   MoreHorizontal  — 溢出菜单按钮
const iconMap: Record<IconKey, Component> = {
  GitCommit, FileDiff, RefreshCw, ArrowUpCircle, History,
  GitBranch, Layers, GitMerge, Broom, Package,
  Settings, MoreHorizontal,
}

interface ToolbarButton {
  key: string; label: string; iconKey: IconKey      // iconKey 对应 iconMap 中的键名，类型为 IconKey 联合类型
  action: () => void
  priority: number
  getDisabledTooltip: () => string                  // 计算禁用 tooltip
}

const wpUnavailable = computed(() => !workspaceStore.currentPath)
const globallyDisabled = computed(() => fileListStore.isOperationRunning)

const buttons = computed<ToolbarButton[]>(() => [
  // i18n key 说明：以下按钮标签（toolbar.*）和禁用 tooltip key 已在语言包中预定义
  // 定义位置：src/locales/zh-CN.json 和 src/locales/en.json
  //
  // toolbar key 定义：
  // toolbar.commit = "提交" / "Commit"
  // toolbar.diff = "差异比较" / "Diff"
  // toolbar.update = "更新" / "Update"
  // toolbar.updateToRevision = "更新到版本" / "Update to Revision"
  // toolbar.log = "日志" / "Log"
  // toolbar.switch = "切换" / "Switch"
  // toolbar.branchTag = "分支/标签" / "Branch/Tag"
  // toolbar.merge = "合并" / "Merge"
  // toolbar.cleanup = "清理" / "Cleanup"
  // toolbar.export = "导出" / "Export"
  // toolbar.settings = "设置" / "Settings"
  // toolbar.moreOperations = "更多操作" / "More Operations"
  // toolbar.operationInProgress = "操作进行中，请等待" / "Operation in progress, please wait"
  // toolbar.noWorkingCopy = "无可用工作副本" / "No working copy available"
  // toolbar.noChanges = "当前工作副本无变更" / "No changes in working copy"
  // toolbar.noSelection = "请先选中文件" / "Please select a file first"
  {
    key: 'commit', label: 'toolbar.commit', iconKey: 'GitCommit',
    action: () => router.push('/workspace/commit'), priority: 8,
    getDisabledTooltip: () => {
      if (globallyDisabled.value) return t('toolbar.operationInProgress')
      if (wpUnavailable.value) return t('toolbar.noWorkingCopy')
      // 有工作副本且无操作进行中时，判断是否有已版本化的文件变更
      // 注意：showUnversioned=false 时 files 中仍包含已版本化文件，
      // 因此需过滤掉 'unversioned' 状态的文件来判断是否有真实变更
      const versionedFiles = fileListStore.files.filter(f => f.status !== 'unversioned')
      if (versionedFiles.length === 0) return t('toolbar.noChanges')
      return '' // 按钮启用状态，无 tooltip
    },
  },
  // diff 按钮：通过 fileListStore.selectedPaths 判断是否有选中文件
  // disabled 条件直接在 ToolBar 中实现，不依赖页面层传递
  { key: 'diff', label: 'toolbar.diff', iconKey: 'FileDiff', action: () => router.push('/workspace/diff'), priority: 8,
    getDisabledTooltip: () => {
      if (globallyDisabled.value) return t('toolbar.operationInProgress')
      if (wpUnavailable.value) return t('toolbar.noWorkingCopy')
      if (fileListStore.selectedPaths.length === 0) return t('toolbar.noSelection')
      return ''
    } },
  { key: 'update', label: 'toolbar.update', iconKey: 'RefreshCw', action: () => emit('open-dialog', 'update'), priority: 7,
    getDisabledTooltip: () => globallyDisabled.value ? t('toolbar.operationInProgress') : t('toolbar.noWorkingCopy') },
  { key: 'updateToRevision', label: 'toolbar.updateToRevision', iconKey: 'ArrowUpCircle', action: () => emit('open-dialog', 'update-to-revision'), priority: 6,
    getDisabledTooltip: () => globallyDisabled.value ? t('toolbar.operationInProgress') : t('toolbar.noWorkingCopy') },
  { key: 'log', label: 'toolbar.log', iconKey: 'History', action: () => router.push('/workspace/log'), priority: 6,
    getDisabledTooltip: () => globallyDisabled.value ? t('toolbar.operationInProgress') : t('toolbar.noWorkingCopy') },
  { key: 'switch', label: 'toolbar.switch', iconKey: 'GitBranch', action: () => emit('open-dialog', 'switch'), priority: 5,
    getDisabledTooltip: () => globallyDisabled.value ? t('toolbar.operationInProgress') : t('toolbar.noWorkingCopy') },
  { key: 'branchTag', label: 'toolbar.branchTag', iconKey: 'Layers', action: () => emit('open-dialog', 'branch-tag'), priority: 4,
    getDisabledTooltip: () => globallyDisabled.value ? t('toolbar.operationInProgress') : t('toolbar.noWorkingCopy') },
  { key: 'merge', label: 'toolbar.merge', iconKey: 'GitMerge', action: () => emit('open-dialog', 'merge'), priority: 3,
    getDisabledTooltip: () => globallyDisabled.value ? t('toolbar.operationInProgress') : t('toolbar.noWorkingCopy') },
  { key: 'cleanup', label: 'toolbar.cleanup', iconKey: 'Broom', action: () => svnStore.cleanupWorkspace(workspaceStore.currentPath), priority: 1,
    getDisabledTooltip: () => globallyDisabled.value ? t('toolbar.operationInProgress') : t('toolbar.noWorkingCopy') },
  { key: 'export', label: 'toolbar.export', iconKey: 'Package', action: () => emit('open-dialog', 'export'), priority: 0,
    getDisabledTooltip: () => globallyDisabled.value ? t('toolbar.operationInProgress') : t('toolbar.noWorkingCopy') },
])

const btnDisabled = computed(() => wpUnavailable.value || globallyDisabled.value)

const visibleButtons = computed(() => {
  const count = Math.max(1, buttons.value.length - overflowCount.value)
  return buttons.value.slice(0, count)
})
const overflowButtons = computed(() => buttons.value.slice(visibleButtons.value.length))

onMounted(() => {
  if (!toolbarRef.value) return
  const observer = new ResizeObserver(([entry]) => {
    const avail = entry.contentRect.width - 80
    overflowCount.value = Math.max(0, buttons.value.length - Math.max(1, Math.floor(avail / 88)))
  })
  observer.observe(toolbarRef.value)
  onUnmounted(() => observer.disconnect())
})
</script>

<template>
  <div ref="toolbarRef" class="h-10 px-3 flex items-center bg-white dark:bg-slate-800 border-b border-slate-200 dark:border-slate-700">
    <div class="flex items-center gap-1 flex-1 overflow-hidden">
      <template v-for="btn in visibleButtons" :key="btn.key">
        <el-tooltip
          :content="btn.getDisabledTooltip()"
          :disabled="!btnDisabled"
          effect="dark"
          placement="bottom"
        >
          <button
            class="inline-flex items-center gap-1 px-2.5 py-1 text-xs rounded-md whitespace-nowrap focus:ring-2 focus:ring-blue-400 focus:outline-none"
            :class="btnDisabled
              ? 'opacity-50 cursor-not-allowed text-slate-400 dark:text-slate-500'
              : 'text-slate-600 dark:text-slate-300 hover:bg-slate-100 dark:hover:bg-slate-700 hover:text-green-600 dark:hover:text-green-400'"
            :disabled="btnDisabled"
            :aria-label="btn.label"
            @click="btn.action"
          >
            <component :is="iconMap[btn.iconKey]" class="w-4 h-4" /><span>{{ t(btn.label) }}</span>
          </button>
        </el-tooltip>
      </template>
      <el-dropdown v-if="overflowButtons.length > 0" trigger="click" placement="bottom-end">
        <button class="px-1.5 py-1 text-xs text-slate-400 focus:ring-2 focus:ring-blue-400 focus:outline-none rounded" :aria-label="t('toolbar.moreOperations')">
          <MoreHorizontal class="w-4 h-4" />
        </button>
        <template #dropdown>
          <el-dropdown-menu>
            <el-dropdown-item v-for="btn in overflowButtons" :key="btn.key" :disabled="btnDisabled" @click="btn.action">
              <!-- V13 修复：溢出菜单中按钮图标使用 Lucide 组件（lucide-vue-next），图标尺寸 w-4 h-4 mr-2 -->
              <component :is="iconMap[btn.iconKey]" class="w-4 h-4 mr-2 inline-block align-middle" />
              <span class="align-middle">{{ t(btn.label) }}</span>
            </el-dropdown-item>
          </el-dropdown-menu>
        </template>
      </el-dropdown>
    </div>
    <el-tooltip :content="t('toolbar.settings')" placement="bottom">
      <button class="p-1.5 rounded-md text-slate-400 hover:text-slate-600 dark:hover:text-slate-300 hover:bg-slate-100 dark:hover:bg-slate-700 focus:ring-2 focus:ring-blue-400 focus:outline-none" :aria-label="t('toolbar.settings')" @click="router.push('/settings')">
        <Settings class="w-4 h-4" />
      </button>
    </el-tooltip>
  </div>
</template>
```

**操作前置条件映射表（完整清单）：**

| 前置条件 | 实现层 | 逻辑说明 | 涉及组件/文件 |
|---------|--------|---------|-------------|
| 操作进行中全局禁用 | ToolBar | 计算 `fileListStore.isOperationRunning`，全部按钮禁用（设置除外），tooltip="操作进行中，请等待" | `ToolBar.vue` — `globallyDisabled` computed |
| 无工作副本时禁用操作按钮 | ToolBar | 检查 `workspaceStore.currentPath` 为 null/空时禁用全部操作按钮（设置除外） | `ToolBar.vue` — `wpUnavailable` computed |
| 提交按钮：无变更时禁用 | ToolBar | 检查 `fileListStore.files` 过滤掉 unversioned 后长度为 0 时提交按钮禁用（考虑 showUnversioned 设置） | `ToolBar.vue` — `commit.getDisabledTooltip()` |
| diff 按钮：需选中文件 | ToolBar | 检查 `fileListStore.selectedPaths.length === 0` 时 diff 按钮禁用，直接在 ToolBar 中实现，不依赖页面层 | `ToolBar.vue` — `diff.getDisabledTooltip()` |
| commit 至少选中一个文件 | 页面层 | CommitPage 模板中提交按钮 `:disabled="selectedCount === 0"` | `CommitPage.vue` — `selectedCount` computed |
| 搜索/筛选过滤后的操作限制 | 页面层 | 文件列表筛选/搜索仅影响显示，不影响选定文件集；commit 面板使用独立副本 | `FileListView.vue` 筛选逻辑 / `CommitPage.vue` |
| 对话框打开时禁止操作 | 页面层 | UpdateRevisionDialog/SwitchDialog 等弹出时，返回按钮和部分操作被禁用（按对话框具体设计） | 各 Dialog 组件 — 通过 emit 控制 |

**`files.length` 触发频率评估（v5 优化说明）：**

当前 `commit.getDisabledTooltip()` 中通过 `fileListStore.files.filter(f => f.status !== 'unversioned').length` 判断是否有已版本化的文件变更。该值通过 `buttons` computed 每次重新计算时都会访问。在 Vue 3 中，computed 的依赖追踪基于响应式系统的自动收集——`getDisabledTooltip` 函数虽然是普通函数而非 computed，但其内部读取的 `fileListStore.files` 会在 `buttons` computed 计算过程中触发响应式追踪。

**优化建议：**
- 将 `hasVersionedChanges` 提取为独立 computed（`computed(() => fileListStore.files.filter(f => f.status !== 'unversioned').length > 0)`），`buttons` computed 仅引用 `hasVersionedChanges.value` 而非直接访问 `files` 数组。这样可以减少响应式依赖粒度（只跟踪布尔值而非整个数组）
- 或者使用 `watchEffect` 替代 computed，在 `files` 变化时手动更新 UI 状态
- 当前方案（每次 `files` 变更都触发 re-computed）在变更较小时性能可接受（按钮列表仅 10 项，`filter` + `length` 成本极低），但批量操作（如一次 add 或 revert 大量文件）可能导致 `buttons` 多次重算。建议在上述独立 computed 优化后，降低高频率 files 变更场景下 ToolBar 的计算开销

| URL 格式校验 | 后端 | checkout/relocate 等命令校验 URL scheme 合法性 | `checkout.rs` / `relocate.rs` — StartsWith 检查 |
| path 合法性校验 | 后端 | validate_path 检查文件路径是否存在、是否为目录、是否有 .svn 子目录 | `svn/executor.rs` — `validate_path()` |
| 冲突仅三种 resolution 取值 | 后端 | resolve_conflict 校验 resolution 参数是否为 mine-full/theirs-full/working | `file_ops.rs` — match 校验 |

**关联说明：**
- ToolBar 层：涉及复用、全局性的条件（操作中/无工作副本），影响所有按钮
- 页面层：涉及特定页面/对话框的内部条件（如文件选中数、对话框状态）
- 后端层：涉及数据完整性、SVN CLI 参数合法性的校验，在 Rust command 中完成
