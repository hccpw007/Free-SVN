# 1.2 前端缺失依赖安装与目录补齐

> 对应开发计划 v2 步骤 1.2：安装前端 Vue 3 生态依赖和 Tauri plugin 的 npm 包
> 前置状态：已有 package.json（vue/pinia/router/element-plus/tailwind/tauri API）

## 涉及文件

| 文件路径 | 操作类型 |
|---------|---------|
| `package.json` | **修改** — dependencies 新增 |
| `src/components/dialogs/` | **新建目录** |
| `public/fonts/` | **新建目录**（Vite 静态资源目录，存放 JetBrains Mono woff2 字体文件）|

## 操作细节

### 1.2A — 安装前端缺失依赖

```bash
cnpm install vue-i18n@^10.0.0 @tauri-apps/plugin-dialog@^2.2.0 @tauri-apps/plugin-shell@^2.2.0 @tauri-apps/plugin-clipboard-manager@^2.2.0 @tauri-apps/plugin-fs@^2.2.0 lucide-vue-next@^0.344.0 diff2html@^3.4.48 highlight.js@^11.10.0
```

> **`@tauri-apps/plugin-fs` 说明：** 用于 HomePage（步骤 6.1）中路径有效性检测，通过 `exists()` 判断工作副本目录是否存在。
>
> **依赖版本兼容性：**
>
> | 包名 | 推荐版本 | 兼容性说明 |
> |------|---------|-----------|
> | `vue-i18n` | 10.0.8 | Vue 3 i18n，^10.0.0 兼容范围 |
> | `@tauri-apps/plugin-dialog` | 2.7.1 | 需同步 Rust 端 `tauri-plugin-dialog` 大版本（2.x）|
> | `@tauri-apps/plugin-shell` | 2.3.5 | 需同步 Rust 端 `tauri-plugin-shell` 大版本（2.x）|
> | `@tauri-apps/plugin-clipboard-manager` | 2.3.2 | 需同步 Rust 端 `tauri-plugin-clipboard-manager` 大版本（2.x）|
> | `@tauri-apps/plugin-fs` | 2.5.1 | Tauri 2 文件系统 API，与 Rust 端独立 |
> | `lucide-vue-next` | 0.344.0 | Vue 3 原生图标库，^0.344.0 兼容 |
> | `diff2html` | 3.4.56 | 差异对比渲染，需配合 highlight.js |
> | `highlight.js` | 11.11.1 | diff2html 语法高亮依赖 |
>
> **版本兼容性要点：**
> - npm 包的 ^ 前缀允许 patch/minor 升级，配合 lock 文件确保可重现构建
> - Rust 端 Tauri 插件版本（dialog/shell/clipboard-manager）必须与 npm 包装器版本**大版本对齐**（均为 2.x），否则 Tauri invoke 调用会因协议不匹配而失败
> - `diff2html` 和 `highlight.js` 需保持版本互不冲突，建议同时升级

### 1.2B — 创建缺失目录

```bash
mkdir -p src/components/dialogs public/fonts
```
