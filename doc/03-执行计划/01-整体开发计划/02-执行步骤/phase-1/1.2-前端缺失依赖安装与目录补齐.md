# 1.2 前端缺失依赖安装与目录补齐

> 对应开发计划 v2 步骤 1.2：安装前端 Vue 3 生态依赖和 Tauri plugin 的 npm 包
> 前置状态：已有 package.json（vue/pinia/router/element-plus/tailwind/tauri API）

## 涉及文件

| 文件路径 | 操作类型 |
|---------|---------|
| `package.json` | **修改** — dependencies 新增 |
| `src/components/dialogs/` | **新建目录** |
| `public/fonts/` | **新建目录**（Vite 静态资源目录，存放 JetBrains Mono woff2 字体文件）|

## 操作细节

### 1.2A — 安装前端缺失依赖

```bash
cnpm install vue-i18n@10 @tauri-apps/plugin-dialog @tauri-apps/plugin-shell @tauri-apps/plugin-clipboard-manager lucide-vue-next diff2html highlight.js
```

### 1.2B — 创建缺失目录

```bash
mkdir -p src/components/dialogs public/fonts
```

## 验收标准

- [ ] `cnpm install` 执行成功无报错
- [ ] `cat package.json \| grep '"vue-i18n"'` 输出非空
- [ ] `ls src/components/dialogs/` 和 `ls public/fonts/` 目录存在
- [ ] `ls node_modules/vue-i18n` 模块已安装
