# 1.4 构建验证

> 对应开发计划 v2 构建验证：确保前后端均编译通过
> 前置状态：1.1（Rust 骨架）、1.2（前端依赖）、1.3（前端骨架）均已完成

## 涉及文件

无新建/修改文件，仅运行编译命令验证

## 操作细节

### 1.4A — Rust 后端编译验证

```bash
cd src-tauri && cargo build 2>&1
```

预期：编译成功，零 error。可接受 `dead_code`、`unused imports` warning（骨架阶段函数暂未调用属正常）。

如果编译失败，按以下顺序排查：
1. 检查 Cargo.toml 中所有 crate 版本是否存在
2. 检查 lib.rs 中 `mod` 声明与文件系统路径一致
3. 检查 models/error.rs 中 `Serialize` 自定义实现
4. 检查各模块 `mod.rs` 中 `pub mod` 声明与子文件匹配
5. 重复 `cargo build` 直到通过

### 1.4B — 前端编译验证

```bash
cnpm run build
```

预期：Vite 构建成功，`dist/` 目录生成。

如果失败：
1. 检查 main.ts 中 vue-i18n 导入路径
2. JSON 导入错误 → 在 tsconfig.app.json 加 `"resolveJsonModule": true`
3. 重复 `cnpm run build` 直到通过

### 1.4C — TypeScript 类型检查

```bash
cnpm run lint
```

预期：vue-tsc 类型检查通过，零 error。

### 1.4D — Tauri 开发模式启动验证（可选）

```bash
cnpm run tauri dev
```

确认窗口标题 "Free-SVN"、尺寸约 1100×720、控制台无异常后 Ctrl+C 停止。

**tauri dev 常见排查场景：**

| 场景 | 现象 | 修复命令 |
|------|------|---------|
| Rust 编译错误 | 前端控制台显示 `[backend] error: ...` 或 `[plugin] xxx build failed`，Tauri 窗口不出现 | 检查 `cargo build` 编译错误，修复后 `cnpm run tauri dev` 自动重编译 |
| 端口占用 | Tauri 输出来回重试 `connecting to dev server...`，`cnpm run tauri dev` 超时 | `lsof -ti:1420 \| xargs kill -9` 后再启动 |
| hot-reload 失效 | 修改前端代码后页面不自动刷新，需手动重启 | 检查 vite.config.ts 中 `server.watch` 配置（Linux/macOS 用 `usePolling: true`），或清除 `.vite/` 缓存后重启 |
| 前端编译阻塞 Rust 编译 | Tauri 先 `vite build` 卡住导致 Rust 侧一直等待 | 关闭所有浏览器标签 + 删除 `src-tauri/target/` 下 `.cargo-lock`，单独执行 `cd src-tauri && cargo build` 确认 Rust 通过后再试 |
| 系统环境缺失 | `Command "cargo" not found` / `Xcode not configured` | 执行 `rustup show` 检查 Rust 安装，macOS 执行 `xcode-select --install` |

## 验收标准

- [ ] `cargo build` 编译成功，零 error
- [ ] `cnpm run build` 构建成功，`dist/` 生成
- [ ] `cnpm run lint` 类型检查通过，零 error
- [ ] `tauri dev` 正常启动（可选验证）
