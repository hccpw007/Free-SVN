### 4.3A — 修改 `src/router/index.ts`

```ts
import { createRouter, createWebHistory } from 'vue-router'
import type { RouteRecordRaw } from 'vue-router'

const routes: RouteRecordRaw[] = [
  {
    path: '/',
    redirect: '/workspace',
  },
  {
    path: '/workspace',
    component: () => import('@/layouts/MainLayout.vue'),
    children: [
      {
        path: '',
        name: 'HomePage',
        component: () => import('@/pages/HomePage.vue'),
      },
      {
        path: 'commit',
        name: 'CommitPage',
        component: () => import('@/pages/CommitPage.vue'),
      },
      {
        path: 'log',
        name: 'LogPage',
        component: () => import('@/pages/LogPage.vue'),
      },
      {
        path: 'diff',
        name: 'DiffViewer',
        component: () => import('@/pages/DiffPage.vue'),
      },
      {
        path: 'resolve',
        name: 'ResolveConflict',
        component: () => import('@/components/dialogs/ResolveConflict.vue'),
      },
    ],
  },
  {
    path: '/settings',
    name: 'SettingsPage',
    component: () => import('@/pages/SettingsPage.vue'),
  },
]

export default createRouter({
  history: createWebHistory(),
  routes,
})
```

**路由表：**

| 路由 | 布局 | 组件 | 说明 |
|------|------|------|------|
| `/` | — | — | 重定向到 /workspace |
| `/workspace` | MainLayout | HomePage | 欢迎页 / 变更列表 |
| `/workspace/commit` | MainLayout | CommitPage | 提交面板 |
| `/workspace/log` | MainLayout | LogPage | 日志视图 |
| `/workspace/diff?file=xxx` | MainLayout | DiffPage | 差异对比（内部使用 DiffViewer 组件） |
| `/workspace/resolve?file=xxx` | MainLayout | ResolveConflict | 冲突解决 |
| `/settings` | 独立 | SettingsPage | 设置页 |

**语言切换联动逻辑：**

settings store 的 `language` 字段值须与 vue-i18n 的 `locale` 联动。在 `src/main.ts` 中 `createI18n` 创建后，在 `App.vue` 的 `onMounted` 中添加以下逻辑：

```ts
// App.vue — 语言切换联动
import { onMounted } from 'vue'
import { useI18n } from 'vue-i18n'
import { useSettingsStore } from '@/stores/settings'

onMounted(() => {
  const settingsStore = useSettingsStore()
  const { locale } = useI18n()

  // 从 settings 中加载语言偏好
  settingsStore.load().then(() => {
    if (settingsStore.language && settingsStore.language !== 'system') {
      locale.value = settingsStore.language
    }
  })
})
```

此外，设置页保存语言变更后，应立即更新 locale：

```ts
// SettingsPage.vue 保存设置后
const { locale } = useI18n()
async function handleSave() {
  await settingsStore.save()
  if (settingsStore.language !== 'system') {
    locale.value = settingsStore.language
  }
}
```

**语言代码映射表（langMap）：**

在 `src/main.ts` 或 `src/i18n.ts` 中定义语言代码映射，确保 settings.language 的值（`'zh-CN'`、`'en'`、`'ja'`、`'ko'`、`'system'`）与 vue-i18n locale 值一致：

```ts
/** 语言代码映射：settings.language 取值 → createI18n locale 值 */
const LANG_MAP: Record<string, string> = {
  'zh-CN': 'zh-CN',
  'en': 'en',
  'ja': 'ja',
  'ko': 'ko',
  'system': 'en', // 跟随系统时默认回退英文
}
```

langMap 说明：
- `ja` 和 `ko` 已在映射表中注册，对应 `locales/ja.json` 和 `locales/ko.json` 文件
- 设置页语言选项中日语/韩语显示为 `"日本語 (Coming Soon)"` / `"한국어 (Coming Soon)"`，提示翻译尚不完整
- `'system'` 表示跟随系统语言，后端通过 `sys-locale` crate 检测系统语言，前端默认回退到 `'en'`

**i18n key 分组规范：**

| 前缀 | 用途 | 示例 | 对应枚举/类型 |
|------|------|------|--------------|
| `app.` | 应用级文案 | welcomeTitle, description | — |
| `error.` | 后端错误码翻译 | SVN_TIMEOUT, SVN_AUTH_FAILED | ErrorCode 枚举值（types/error-codes.ts）|
| `common.` | 通用按钮/提示 | cancel, confirm, save | — |
| `workspace.` | 工作副本视图 | noChanges, pathNotExist | — |
| `file.` | 文件状态/操作 | statusModified, actionDiff | — |

**关键约束：**

1. **`error.` 下的 key 名称必须与 `ErrorCode` 枚举值（`types/error-codes.ts`）完全一致**，例如 `ErrorCode.SVN_EXEC_FAILED` 对应 locale key `error.SVN_EXEC_FAILED`。新增错误码时必须同时更新：
   - `types/error-codes.ts` 的 `ErrorCode` 枚举
   - `types/error-codes.ts` 的 `ERROR_MESSAGE_KEYS` 映射表
   - 所有 locale JSON 文件（en / zh-CN / ja / ko）的 `error` 分类
2. locale JSON 中的 `error` 分类国际化 key 路径格式为 `error.{ErrorCode}`，如 `error.INVALID_INPUT`、`error.SVN_TIMEOUT`
3. 在 `getErrorMessage()` 中通过 `ERROR_MESSAGE_KEYS[errorCode]` 映射到 locale key，组件中使用 `$t(key)` 显示翻译；i18n 不可用时回退链为 `FALLBACK_MESSAGES_ZH` → `FALLBACK_MESSAGES_EN` → 原始错误消息
4. `workspace.` 和 `file.` 分类下的 key 保持 kebab-case 或 camelCase 风格，无枚举绑定约束
