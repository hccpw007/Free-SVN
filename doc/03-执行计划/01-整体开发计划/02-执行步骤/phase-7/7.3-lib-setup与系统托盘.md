### 7.3A — 修改 `src-tauri/src/lib.rs`

完整 setup 函数，包含插件注册、系统托盘、单实例防护：

```rust
use tauri::{
    AppHandle, Manager, WebviewUrl, WebviewWindowBuilder,
    menu::{MenuBuilder, MenuItemBuilder, SubmenuBuilder},
    tray::{TrayIconBuilder, TrayIconEvent, MouseButton, MouseButtonState},
    Emitter,
};
use tauri_plugin_notification::NotificationExt;

#[cfg_attr(mobile, tauri::mobile_entry_point)]
pub fn run() {
    // 插件注册（setup 前必须完成的 5 个插件）
    let builder = tauri::Builder::default()
        .plugin(tauri_plugin_store::Builder::new().build())
        .plugin(tauri_plugin_single_instance::init(|app, args, _cwd| {
            // 第二实例启动时激活首实例窗口
            if let Some(window) = app.get_webview_window("main") {
                let _ = window.show();
                let _ = window.set_focus();

                // 解析 --svn-cmd 参数（右键菜单触发）：格式为 --svn-cmd <command> [file1 file2 ...]
                // 将命令和文件路径通过事件传递给主窗口，由前端 svnStore 处理执行
                if let Some(cmd_idx) = args.iter().position(|a| a == "--svn-cmd") {
                    let cmd = args.get(cmd_idx + 1).cloned().unwrap_or_default();
                    let files: Vec<String> = args.iter()
                        .skip(cmd_idx + 2)
                        .filter(|a| !a.starts_with('--'))
                        .cloned()
                        .collect();
                    if !cmd.is_empty() {
                        let _ = window.emit("svn:shell-cmd", serde_json::json!({
                            "command": cmd,
                            "files": files,
                        }));
                    }
                }
            }
        }))
        .plugin(tauri_plugin_window_state::Builder::default().build())
        .plugin(
            tauri_plugin_log::Builder::new()
                .target(tauri_plugin_log::Target::new(
                    tauri_plugin_log::TargetKind::LogDir { name: "logs" },
                ))
                .level(log::LevelFilter::Info)
                .build(),
        )
        .plugin(tauri_plugin_dialog::init())
        .plugin(tauri_plugin_shell::init())
        .plugin(tauri_plugin_clipboard_manager::init())
        .plugin(tauri_plugin_notification::init())
        .manage(crate::svn::queue::SvnQueue::new())
        .setup(|app| {
            // ── 1. 初始化日志系统 ──
            crate::logging::logger::init(app.handle())?;

            // ── 2. 初始化配置存储 ──
            crate::config::store::init(app.handle())?;

            // ── 3. 构建系统托盘菜单 ──
            // 托盘菜单项使用英文文本：系统托盘菜单是操作系统级别的 UI 元素，
            // 不跟随应用内的 vue-i18n 语言包切换。Tauri 的 TrayMenuBuilder 在
            // setup() 阶段设置文本后无法运行时变更，因此此处直接使用英文。
            // 各操作系统的托盘菜单本身也习惯使用英文（macOS 通知中心菜单通常
            // 为英文，Windows 系统托盘区域混用英文本地化效果不佳）。
            let show = MenuItemBuilder::with_id("show", "Show Main Window")
                .accelerator("CmdOrCtrl+Shift+S").build(app)?;
            let update = MenuItemBuilder::with_id("update", "SVN Update").build(app)?;
            let cleanup = MenuItemBuilder::with_id("cleanup", "SVN Cleanup").build(app)?;
            let about = MenuItemBuilder::with_id("about", "About Free-SVN").build(app)?;
            let quit = MenuItemBuilder::with_id("quit", "Quit").accelerator("CmdOrCtrl+Q").build(app)?;

            let menu = MenuBuilder::new(app)
                .item(&show)
                .separator()
                .item(&update)
                .item(&cleanup)
                .separator()
                .item(&about)
                .separator()
                .item(&quit)
                .build()?;

            // ── 4. 创建系统托盘图标 ──
            // 获取默认窗口图标；若获取失败则创建一个 16×16 透明占位图标
            let tray_icon = app.default_window_icon()
                .map(|icon| icon.clone())
                .unwrap_or_else(|| tauri::image::Image::new(&[0u8; 256], 16, 16)); // 16×16 透明 RGBA
            TrayIconBuilder::new()
                .icon(tray_icon)
                .menu(&menu)
                .tooltip("Free-SVN")
                .on_menu_event(move |app, event| {
                    // 克隆 app handle 供异步 spawn 块使用（避免 lambda 生命周期借用冲突）
                    let app_handle = app.clone();
                    match event.id().as_ref() {
                        "show" => {
                            if let Some(window) = app.get_webview_window("main") {
                                let _ = window.show();
                                let _ = window.set_focus();
                            }
                        }
                        "update" => {
                            let cwd = crate::config::store::current_workspace()
                                .unwrap_or_default();
                            if !cwd.is_empty() {
                                tauri::async_runtime::spawn(async move {
                                    let _ = crate::commands::update::update_workspace(
                                        app_handle.clone(),
                                        crate::commands::update::UpdateParams {
                                            path: cwd, revision: None,
                                            depth: None, ignore_externals: None,
                                            credentials: None,
                                        },
                                        app_handle.state::<crate::svn::queue::SvnQueue>(),
                                    ).await;
                                });
                            }
                        }
                        "cleanup" => {
                            let cwd = crate::config::store::current_workspace()
                                .unwrap_or_default();
                            if !cwd.is_empty() {
                                tauri::async_runtime::spawn(async move {
                                    let _ = crate::commands::cleanup::cleanup_workspace(
                                        cwd,
                                        app_handle.state::<crate::svn::queue::SvnQueue>(),
                                    ).await;
                                });
                            }
                        }
                        "about" => {
                            // 弹出原生"关于"对话框，而非导航到 /about 路由
                            // 方案说明：项目中不存在 /about 路由，因为"关于"信息更适合
                            // 用原生对话框（跨平台一致、无需前端页面）。实现方式：
                            // 方案 A（推荐）：直接调用 tauri-plugin-dialog 的 message API
                            //   let _ = tauri_plugin_dialog::DialogExt::dialog(app)
                            //       .message("Free-SVN — A cross-platform SVN GUI tool\n\nVersion: 1.0.0")
                            //       .title("About Free-SVN")
                            //       .show();
                            // 方案 B：若要保留 emit 方式，需在路由配置中添加 /about 路由，
                            //       并在对应页面（如 src/pages/AboutPage.vue）中显示版本等信息
                            if let Some(window) = app.get_webview_window("main") {
                                let _ = window.show();
                                let _ = window.set_focus();
                                // 当前使用"弹出原生对话框"方案（方案 A），不再 emit navigate
                                let _ = tauri_plugin_dialog::DialogExt::dialog(app)
                                    .message("Free-SVN — A cross-platform SVN GUI tool\n\nVersion: 1.0.0")
                                    .title("About Free-SVN")
                                    .show();
                            }
                        }
                        "quit" => { app.exit(0); }
                        _ => {}
                    }
                })
                .on_tray_icon_event(|tray, event| {
                    // 单击托盘图标 → 激活/显示主窗口
                    if let TrayIconEvent::Click {
                        button: MouseButton::Left,
                        button_state: MouseButtonState::Up, ..
                    } = event {
                        if let Some(app) = tray.app_handle() {
                            if let Some(window) = app.get_webview_window("main") {
                                let _ = window.show();
                                let _ = window.set_focus();
                            }
                        }
                    }
                })
                .build(app)?;

            // ── 5. 注册 Shell 右键菜单 ──
            let _ = crate::shell_integration::register();

            Ok(())
        })
        .invoke_handler(tauri::generate_handler![
            // 3A — 只读操作
            crate::commands::status::get_status,
            crate::commands::status::get_info,
            crate::commands::diff::get_diff,
            crate::commands::log::get_log,
            crate::commands::log::get_blame,
            // 3B — 核心写操作
            crate::commands::checkout::checkout_repo,
            crate::commands::update::update_workspace,
            crate::commands::commit::create_commit,
            crate::commands::file_ops::add_files,
            crate::commands::file_ops::delete_files,
            crate::commands::file_ops::revert_files,
            crate::commands::file_ops::resolve_conflict,

            // 3C — 剩余功能操作

}
```

- Debian/Ubuntu：`apt install libayatana-appindicator3-dev`
- Fedora：`dnf install libappindicator-gtk3-devel`

Tauri 2 通过 `tauri-plugin-tray` 或原生 `tray-icon` 功能自动链接这些库。如果上述依赖未安装，cargo build 会在链接阶段失败。确认依赖后方可执行 `cnpm run tauri build`。

**7.3B — 关闭窗口→最小化到托盘 + 自启动行为控制（App.vue 前端侧）：**

自启动时最小化到托盘的行为说明：
- 应用通过系统自启动机制（OS 启动时自动运行）启动后，不显示主窗口，直接最小化到系统托盘
- 实现方式：在 `tauri.conf.json` 中配置 `"autoStart": false`（不自带自启动配置，而是通过各平台的系统服务管理），或由用户在设置页面开启自动启动功能
- 前端 App.vue 的 `onMounted` 中检测启动参数：如果是从系统自启动启动（无交互启动参数），则自动隐藏窗口到托盘
- 最终行为：用户点击任务栏图标或托盘图标时激活窗口，系统启动时自动运行但不显示窗口

```ts
import { getCurrentWebviewWindow } from '@tauri-apps/api/webviewWindow'

onMounted(() => {
  const win = getCurrentWebviewWindow()
  // 关闭按钮隐藏窗口到托盘而非退出
  win.onCloseRequested(async () => {
    await win.hide()
  })
})
```

**7.3C — 系统托盘操作中徽章（业务设计 §3.4 要求）：**

操作进行中时，在系统托盘图标上显示小徽章（macOS 支持设置 badge 文本，Windows/Linux 可通过动态图标替换实现）：

- **macOS**：使用 `tray.set_icon()` 切换为带红点的图标或通过 `app.get_tray_icon_by_id("main")?.set_tooltip("SVN 操作进行中...")` 修改 tooltip。macOS Dock 徽章通过以下 `#[cfg(target_os = "macos")]` 条件编译实现：

  ```rust
  /// macOS Dock badge 设置（Cargo.toml 已通过 target.'cfg(target_os = "macos")' 添加 objc 依赖）
  ///
  /// # 安全性说明（objc unsafe 内存安全）
  /// 1. `NSApp` 全局指针在 macOS 应用启动后即初始化，setup() 之后调用保证非空
  /// 2. `dockTile` / `badgeLabel` 消息发送不持有返回值 —— 不涉及 Rust 所有权的跨语言传递
  /// 3. `NSString::alloc(nil).init_str("●")` 创建的 NSString 会被 AppKit 自动加入
  ///    当前 autorelease pool，无需手动 release
  /// 4. `msg_send![NSApp, dockTile]` 是 NSApp 的只读属性访问，无副作用
  /// 5. `setBadgeLabel: nil` 在 AppKit 中被定义为"清除 badge"（nil 指针传入是安全的）
  /// 6. 所有 objc 调用在 `#[cfg(target_os = "macos")]` 保护下仅对 macOS 编译，
  ///    不影响 Windows/Linux 的代码安全性
  #[cfg(target_os = "macos")]
  pub fn set_dock_badge(visible: bool) {
      use objc::{msg_send, sel, sel_impl};
      use objc::runtime::{Object, BOOL};
      #[link(name = "AppKit", kind = "framework")]
      extern "C" {
          static NSApp: *mut Object;
      }
      unsafe {
          let dock_tile: *mut Object = msg_send![NSApp, dockTile];
          let badge_label: *mut Object = msg_send![
              dock_tile,
              badgeLabel
          ];
          if visible {
              let label = objc::runtime::NSString::alloc(nil).init_str("●");
              let _: () = msg_send![dock_tile, setBadgeLabel: label];
          } else {
              let _: () = msg_send![dock_tile, setBadgeLabel: *nil];
          }
      }
  }
  ```

  > **注意：** 上述代码使用 `objc` crate。Cargo.toml 已通过 `[target.'cfg(target_os = "macos")'.dependencies]` 添加 `objc = "0.2"`（仅 macOS 编译，不影响其他平台）。

- **Windows/Linux**：准备两套图标资源（正常图标 + 带徽章图标），通过 `tray.set_icon(...)` 在操作开始/完成时切换
- **实现时机**：在 `svnStore.subscribeToOperations()` 中监听 `operation:started` / `operation:completed` / `operation:error` 三个事件，通过 invoke 调用 Rust 端的 `set_tray_badge(visible: bool)` 命令切换图标状态（该命令在 `lib.rs` 或独立的 `tray.rs` 中注册为 tauri command）

**`set_tray_badge` 命令实现（可选放在 `src-tauri/src/commands/tray.rs` 中）：**

```rust
#[tauri::command]
pub fn set_tray_badge(app: AppHandle, visible: bool) -> Result<(), String> {
    if let Some(tray) = app.get_tray_icon_by_id("main") {
        let _ = tray.set_tooltip(if visible { "SVN 操作进行中..." } else { "Free-SVN" });
        // 图标切换（需要准备带徽章的图标资源）
        if visible {
            let icon_bytes = include_bytes!("../../icons/tray-badge.png");
            if let Ok(img) = tauri::image::Image::from_bytes(icon_bytes) {
                let _ = tray.set_icon(Some(img));
            }
        } else {
            let icon_bytes = include_bytes!("../../icons/icon.png");
            if let Ok(img) = tauri::image::Image::from_bytes(icon_bytes) {
                let _ = tray.set_icon(Some(img));
            }
        }
    }
    Ok(())
}
```

**7.3D — 后台操作完成系统通知（P1-4 / 业务设计 §3.4）：**

当窗口隐藏到托盘（后台模式）时，SVN 操作完成后通过操作系统通知中心发出通知，确保用户感知操作结果。使用 `tauri-plugin-notification` 实现跨平台一致性：

```rust
use tauri_plugin_notification::{NotificationExt, NotificationBuilder};

// 在 lib.rs 或独立的 notification.rs 中 — 通知发送辅助函数
fn send_os_notification(app: &AppHandle, title: &str, body: &str) {
    let _ = app.notification()
        .builder()
        .title("Free-SVN")
        .body(format!("{}: {}", title, body))
        .show();
}
```

在长操作完成的回调中触发。为了避免前端轮询，这里采用**纯后端路径**（不依赖前端事件监听）：

**推荐实现路径（后端直接通知，无前端参与）：**
1. 在 `lib.rs` 托盘菜单 `update`/`cleanup` 的异步 spawn 块中，等待命令执行完毕后调用 `send_os_notification()`
2. 具体伪代码：
   ```rust
   // 在 update 菜单事件处理中：
   "update" => {
       let cwd = crate::config::store::current_workspace().unwrap_or_default();
       if !cwd.is_empty() {
           let app_handle = app.clone();
           tauri::async_runtime::spawn(async move {
               let result = crate::commands::update::update_workspace(
                   app_handle.clone(), /* params */, app_handle.state::<crate::svn::queue::SvnQueue>()
               ).await;
               match result {
                   Ok(r) => send_os_notification(&app_handle, "SVN Update", &format!("完成, 版本 {}", r.revision)),
                   Err(e) => send_os_notification(&app_handle, "SVN Update", &format!("失败: {}", e)),
               }
           });
       }
   }
   ```
3. 对于非托盘触发的长操作（例如从前端 UI 触发的 update/checkout），由 Rust 端的命令实现（如 `commands/update.rs`）在操作完成时自行调用 `send_os_notification()` —— 该函数接受 `AppHandle` 参数，通过 `tauri-plugin-notification` 发送通知

**备选路径（前端中转）：**
1. 若因架构限制不能在后端直接发通知，可在前端 `svnStore` 监听 `operation:completed` / `operation:error` 事件
2. 前端检测 `document.hidden === true`（窗口被隐藏）或使用 `@tauri-apps/api/window` 的 `isVisible()` API
3. 当前端确认窗口隐藏时，通过 `invoke('show_notification', { title, body })` 调用后端通知命令
4. 此路径多了一层往返，不推荐作为首选方案

> **建议：** 采用"后端直接通知"路径（推荐方案），因为托盘触发的长操作本身在后端执行，通知也在后端发出，无需前端参与即可完成完整闭环。

**`tauri-plugin-notification` 集成步骤：**

1. 在 `Cargo.toml` 中添加 `tauri-plugin-notification = "2"`
2. 在 `lib.rs` 的 `tauri::Builder::default()` 链中添加 `.plugin(tauri_plugin_notification::init())`
3. 在 `tauri.conf.json` 中配置 `identifier`（已存在）和 `bundle` 设置以确保通知正常工作
4. macOS 上需要确保应用有 `com.apple.developer.usernotifications` 权限（在 `Info.plist` 中配置）

**注：** macOS 需要应用签名后方可发送通知。开发阶段可在终端验证：`osascript -e 'display notification "test" with title "Free-SVN"'` 作为备选方案。
