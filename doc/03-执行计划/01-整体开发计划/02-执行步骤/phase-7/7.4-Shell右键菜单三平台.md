### 7.4B — macos.rs

通过 Automator `.workflow` bundle 注册 Finder 右键服务：

pub fn unregister() -> Result<(), AppError> {
    let svc_dir = dirs::home_dir().ok_or(AppError::Repo("无法获取用户目录".into()))?.join("Library/Services");
    for entry in fs::read_dir(&svc_dir)? {
        let entry = entry?;
        if entry.file_name().to_string_lossy().starts_with("SVN-") && entry.path().extension() == Some("workflow") {
            fs::remove_dir_all(entry.path())?;
        }
    }
    // 刷新服务缓存
    std::process::Command::new("/System/Library/CoreServices/pbs").arg("-flush").output()?;
    Ok(())
}

> 使用方法：在 `macos.rs` 的 `register()` 函数中遍历菜单数组调用此模板即可。确认方式：重启 Finder (`killall Finder`) 后在文件上右键 → 服务 → SVN 提交。12 项菜单需要各自独立的 `.workflow` bundle，每项对应不同的 `--svn-cmd` 参数。


**右键菜单唤醒主窗口参数传递流程（业务功能 §3.2 需求 + P1-3）：**

右键菜单触发后，应用以第二实例启动并携带 `--svn-cmd <command> [file1 file2 ...]` 参数。单实例插件 `tauri-plugin-single-instance` 截获该参数并传递给首实例。完整流程：

```
用户右键 → 选择 SVN 菜单项
       ↓
系统执行: /path/to/free-svn --svn-cmd commit "/path/to/file"
       ↓
tauri-plugin-single-instance 捕获第二实例启动
       ↓
首实例 on_instance_event(|app, args, _cwd| { ... })
  → 解析 args 中 --svn-cmd <command> [files...]
  → window.emit("svn:shell-cmd", { command, files })
       ↓
前端 svnStore 监听 svn:shell-cmd 事件
  → 映射 command 到对应的 store 方法
  → 根据操作类型判断是否需要 UI 交互（commit/log/diff 需要打开对应页面）
  → 无交互后台操作直接执行（update/cleanup/export）
```

**关键点**：
1. 右键菜单执行文件路径参数的传递格式：macOS/WSL 使用 `"$@"` 传递 Finder/Nautilus 选择的文件列表；Windows 使用 `%V` 传递当前目录路径
2. `--svn-cmd` 值列表：`commit`/`update`/`log`/`update-rev`/`diff`/`revert`/`switch`/`branch-tag`/`merge`/`cleanup`/`export`/`checkout`
3. 前端监听 `svn:shell-cmd` 事件后，对于需要用户输入的交互（commit/log/diff/switch/branch-tag/merge/checkout/export），自动导航到对应页面并预填路径参数；对于无交互的后台操作（update/cleanup/revert），直接执行并在完成后发出系统通知
4. 前端的 `svn:shell-cmd` 事件监听在 `App.vue` 的 `onMounted` 中注册，确保应用启动后立即生效

> **`--svn-cmd` 参数格式的跨平台差异详解：**
>
> | 平台 | 右键菜单机制 | 参数占位符 | 传递对象 | 多文件支持 |
> |------|-------------|-----------|---------|-----------|
> | **macOS** | Automator `.workflow` Service | `"$@"` | Finder 选中的文件路径列表 | 是（Automator 自动传递所有选中文件） |
> | **Windows** | Windows 注册表 `shell\command` | `"%V"` / `"%1"` / `"%*"` | `%V`=当前目录路径 / `%1`=首个选中项 / `%*`=所有选中项 | 仅 `%*` 支持多文件 |
> | **Linux** | Nautilus Script | `"${FILES[@]}"` | `$NAUTILUS_SCRIPT_SELECTED_FILE_PATHS` 环境变量中的路径 | 是（通过 bash 数组展开） |
>
> **各平台命令示例：**
> ```
> macOS:    /path/to/free-svn --svn-cmd commit "$@"
> Windows:  "C:\Free-SVN\free-svn.exe" --svn-cmd commit "%V"
> Linux:    /usr/bin/free-svn --svn-cmd commit "${FILES[@]}"
> ```
>
> **Windows 多文件传递的局限性：**
> - `%V` 仅传递当前目录路径（右键目录背景场景），不是选中文件路径
> - `%1` 传递第一个选中项路径（右键文件或文件夹场景）
> - `%*` 传递所有选中项路径（Windows Vista+ 支持）
> - 推荐在 Windows 菜单注册中使用 `"%*"` 以支持多文件选择，对于仅需单文件的操作（diff/log/blame）使用 `"%1"`
>
> **macOS 多文件传递说明：**
> - Automator 的 Run Shell Script action 在"Pass input: as arguments"模式下将 Finder 选中的所有文件路径依次作为参数传递给脚本
> - `$@` 展开为每个文件的带引号字符串，支持包含空格的文件路径


### 7.4C — windows.rs

通过注册表 `HKEY_CLASSES_ROOT\Directory\Background\shell\`、`HKEY_CLASSES_ROOT\*\shell\` 和 `HKEY_CLASSES_ROOT\Directory\shell\` 注册：

```rust
use std::process::Command;

const APP: &str = "Free-SVN";

/// 注册工作副本右键菜单（12 项）
pub fn register() -> Result<(), AppError> {
    // 注册表基路径
    const WC_KEY: &str = r"HKEY_CLASSES_ROOT\Directory\Background\shell";
    const DIR_KEY: &str = r"HKEY_CLASSES_ROOT\Directory\shell";
    const FILE_KEY: &str = r"HKEY_CLASSES_ROOT\*\shell";

    // 辅助：添加菜单项到注册表
    fn add_menu(parent: &str, name: &str, cmd: &str, icon: &str) -> Result<(), AppError> {
        let key = format!(r"{}\SVN-{}", parent, name);
        Command::new("reg")
            .args(&["add", &key, "/ve", "/t", "REG_SZ", "/d", &format!("SVN {}", name), "/f"])
            .output()?;
        Command::new("reg")
            .args(&["add", &format!(r"{}\command", key), "/ve", "/t", "REG_SZ", "/d",
                &format!(r#""{}" --svn-cmd {} "%V""#, std::env::current_exe()?.display(), cmd), "/f"])
            .output()?;
        if !icon.is_empty() {
            Command::new("reg")
                .args(&["add", &key, "/v", "Icon", "/t", "REG_SZ", "/d", icon, "/f"])
                .output()?;
        }
        Ok(())
    }

    // 工作副本 11 项——同时在目录背景(WC_KEY)和文件(FILE_KEY)下注册
    // 菜单项使用英文：右键菜单是操作系统级 UI，各区域 Windows 用户默认使用英文菜单名
    let menus = [
        ("Commit", "commit"), ("Update", "update"), ("Log", "log"),
        ("Update to Rev", "update-rev"), ("Diff", "diff"), ("Revert", "revert"),
        ("Switch", "switch"), ("Branch/Tag", "branch-tag"), ("Merge", "merge"),
        ("Cleanup", "cleanup"), ("Export", "export"),
    ];
    for (name, cmd) in &menus {
        add_menu(WC_KEY, name, cmd, "")?;
        add_menu(FILE_KEY, name, cmd, "")?;  // 文件级右键菜单
    }
    // 普通文件夹 1 项（Directory\shell 下，右键文件夹本身）
    add_menu(DIR_KEY, "检出", "checkout", "")?;

    Ok(())
}

pub fn unregister() -> Result<(), AppError> {
    for parent in &[
        r"HKEY_CLASSES_ROOT\Directory\Background\shell",
        r"HKEY_CLASSES_ROOT\Directory\shell",
        r"HKEY_CLASSES_ROOT\*\shell",
    ] {
        let output = Command::new("reg")
            .args(&["query", parent, "/f", "SVN-*", "/k"]).output()?;
        for line in String::from_utf8_lossy(&output.stdout).lines() {
            let line = line.trim();
            if line.contains("SVN-") {
                Command::new("reg").args(&["delete", line, "/f"]).output()?;
            }
        }
    }
    Ok(())
}
```

> **注意：** Windows 注册表写入需管理员权限。具体提权方案：
> 1. **构建配置提权**：在 `tauri.conf.json` 的 `bundle.windows` 中设置 `requested-execution-level = "requireAdministrator"`，使得每次启动都触发 UAC 弹窗提权
> 2. **首次运行提权（推荐）**：在 `windows.rs` 的 `register()` 中检测到注册表写入失败（无管理员权限）时，通过 `runas` 命令以管理员身份重启自身执行注册。应用入口收到 `--register-shell` 参数时，跳过正常启动流程，仅执行 `windows::register()` 后退出：
>    ```rust
>    // 首次安装时通过 runas 提权注册（避免持续提权问题）
>    fn register_with_elevation() -> Result<(), AppError> {
>        let exe = std::env::current_exe().map_err(|_| AppError::Repo("获取可执行路径失败".into()))?;
>        let status = std::process::Command::new("powershell")
>            .args(&["Start-Process", "cmd", "-Verb", "RunAs",
>                    "-ArgumentList", &format!("/c \"\"{}\" --register-shell\"", exe.display()),
>                    "-Wait"])
>            .status()?;
>        if !status.success() {
>            return Err(AppError::Repo("用户拒绝了提权请求".into()));
>        }
>        Ok(())
>    }
>    ```
> 3. **备选方案（避免提权）**：使用 `rust-winreg` crate 操作 `HKEY_CURRENT_USER\Software\Classes` 路径（而非 `HKEY_CLASSES_ROOT`），该路径无需管理员权限。推荐在 v2 迁移计划中切换到此方案（代码模板已在下方提供）
>
> **区域兼容性：** `reg query` 输出格式随 Windows 区域设置变化。当前匹配逻辑 `line.contains("SVN-")` 理论上兼容所有区域，因为用户菜单名使用英文 `"SVN-{name}"` 格式不依赖区域文字。但 `reg delete` 的行提取仍需验证。
>
> **推荐方案**（使用 `rust-winreg` crate 替代 `reg.exe`，完全消除区域兼容性风险，且无需管理员权限）：
>
> ### 采用 rust-winreg 作为主要方案（在 Windows 上替代 reg.exe）
>
> 相较于 `reg.exe` 命令行方案，`rust-winreg` 有以下优势：
> - **无需管理员权限**：操作 `HKEY_CURRENT_USER\Software\Classes` 路径即可
> - **无区域兼容性问题**：直接操作注册表 API，不依赖命令行输出文本解析
> - **更稳健的清理**：精确删除自身创建的键值，不会误删其他注册表项
> - **无需 UAC 弹窗**：用户体验更好
>
> **Cargo.toml 添加依赖：**
> ```toml
> [target.'cfg(windows)'.dependencies]
> winreg = "0.52"
> ```
>
> **windows.rs 替换实现（建议完全替代 reg.exe 方案）：**
> ```rust
> use winreg::enums::*;
> use winreg::RegKey;
>
> // ── 右键菜单项定义 ──
> const MENUS: &[(&str, &str)] = &[
>     ("Commit", "commit"), ("Update", "update"), ("Log", "log"),
>     ("Update to Rev", "update-rev"), ("Diff", "diff"), ("Revert", "revert"),
>     ("Switch", "switch"), ("Branch/Tag", "branch-tag"), ("Merge", "merge"),
>     ("Cleanup", "cleanup"), ("Export", "export"),
> ];
>
> /// 注册工作副本右键菜单（12 项 + 1 检出）
> pub fn register() -> Result<(), AppError> {
>     let hkcu = RegKey::predef(HKEY_CURRENT_USER);
>     let classes = hkcu.open_subkey_with_flags("Software\\Classes", KEY_WRITE)?;
>
>     // 三组注册表路径（目录背景、目录本身、文件）
>     let parents = [
>         "Directory\\Background\\shell",
>         "Directory\\shell",
>         "*\\shell",
>     ];
>     for (name, cmd) in MENUS {
>         for parent in &parents {
>             let key_path = format!("{}\\SVN-{}", parent, name);
>             let (key, _) = classes.create_subkey(&key_path)?;
>             key.set_value("", &format!("SVN {}", name))?;
>             let (cmd_key, _) = classes.create_subkey(format!("{}\\command", key_path))?;
>             cmd_key.set_value("", &format!(
>                 r#""{}" --svn-cmd {} "%*""#,
>                 std::env::current_exe()?.display(), cmd
>             ))?;
>         }
>     }
>     // 检出菜单（仅用于 Directory\shell）
>     let checkout_path = "Directory\\shell\\SVN-Checkout";
>     let (key, _) = classes.create_subkey(checkout_path)?;
>     key.set_value("", "SVN Checkout")?;
>     let (cmd_key, _) = classes.create_subkey(format!("{}\\command", checkout_path))?;
>     cmd_key.set_value("", &format!(
>         r#""{}" --svn-cmd checkout "%*""#,
>         std::env::current_exe()?.display()
>     ))?;
>     Ok(())
> }
>
> /// 卸载右键菜单（精确删除 SVN 相关键值）
> pub fn unregister() -> Result<(), AppError> {
>     let hkcu = RegKey::predef(HKEY_CURRENT_USER);
>     let classes = hkcu.open_subkey_with_flags("Software\\Classes", KEY_WRITE)?;
>     let parents = [
>         "Directory\\Background\\shell",
>         "Directory\\shell",
>         "*\\shell",
>     ];
>     for (name, _) in MENUS {
>         for parent in &parents {
>             let key_path = format!("{}\\SVN-{}", parent, name);
>             let _ = classes.delete_subkey_all(&key_path);
>         }
>     }
>     let _ = classes.delete_subkey_all("Directory\\shell\\SVN-Checkout");
>     Ok(())
> }
> ```
>
> **迁移说明：** 上述 `rust-winreg` 方案为当前推荐的主要实现方案。`reg.exe` 命令行方案保留在下方供参考，但新开发时优先使用 `rust-winreg` 方案。二者在 `#[cfg(windows)]` 条件编译下互斥，同一编译中只选择一种。
>
> **以下为 reg.exe 方案（保留供参考，不推荐用于新开发）：**
> ```rust

### 7.4D — linux.rs

通过 Nautilus 脚本注册：

```rust
use std::fs;
#[cfg(unix)] use std::os::unix::fs::PermissionsExt;

/// 注册工作副本右键菜单（12 项）
pub fn register() -> Result<(), AppError> {
    let script_dir = dirs::home_dir()
        .ok_or(AppError::Repo("无法获取用户目录".into()))?
        .join(".local/share/nautilus/scripts/SVN");
    fs::create_dir_all(&script_dir)?;

    let exe = std::env::current_exe().map_err(|_| AppError::Repo("无法获取可执行路径".into()))?;

    // 11 项 SVN 操作脚本
    // 菜单项使用英文：Nautilus 脚本菜单中的名称使用英文，因为这是系统级右键菜单，
    // 不跟随应用内的 i18n 语言包切换
    let menus = [
        ("Commit", "commit"), ("Update", "update"), ("Log", "log"),
        ("Update to Rev", "update-rev"), ("Diff", "diff"), ("Revert", "revert"),
        ("Switch", "switch"), ("Branch/Tag", "branch-tag"), ("Merge", "merge"),
        ("Cleanup", "cleanup"), ("Export", "export"),
    ];
    for (label, cmd) in &menus {
        let script = script_dir.join(label);
        let content = format!(
            "#!/bin/bash\nFILES=()\nwhile IFS= read -r file; do FILES+=(\"$file\"); done <<< \"$NAUTILUS_SCRIPT_SELECTED_FILE_PATHS\"\n\"{}\" --svn-cmd {} \"${{FILES[@]}}\"\n",
            exe.display(), cmd
        );
        fs::write(&script, &content)?;
        fs::set_permissions(&script, std::fs::Permissions::from_mode(0o755))?;
    }

    // 普通文件夹 1 项（检出）
    let checkout = script_dir.join("Checkout");
    fs::write(&checkout, format!(
        "#!/bin/bash\nFILES=()\nwhile IFS= read -r file; do FILES+=(\"$file\"); done <<< \"$NAUTILUS_SCRIPT_SELECTED_FILE_PATHS\"\n\"{}\" --svn-cmd checkout \"${{FILES[@]}}\"\n",
        exe.display()
    ))?;
    fs::set_permissions(&checkout, std::fs::Permissions::from_mode(0o755))?;

    Ok(())
}

pub fn unregister() -> Result<(), AppError> {
    let script_dir = dirs::home_dir()
        .ok_or(AppError::Repo("无法获取用户目录".into()))?
        .join(".local/share/nautilus/scripts/SVN");
    if script_dir.exists() {
        fs::remove_dir_all(&script_dir)?;
    }
    Ok(())
}
```

> **注意：** Nautilus 脚本目录 `~/.local/share/nautilus/scripts/` 在 GNOME 文件管理器下生效。其他文件管理器（Dolphin、Thunar）路径不同，可通过 `XDG_CONFIG_HOME` 环境变量检测。
