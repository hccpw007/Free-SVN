## 操作细节

### 概述

当前只有 `checkout_repo` 使用了 `run_svn_with_progress`（试点已验证通过）。需要将其他 4 个写操作命令也改为使用进度推送函数：

| 命令 | 操作名 | 需要追加 AppHandle？ | 特殊注意事项 |
|------|--------|-------------------|-------------|
| update_workspace | "update" | 否（已有 AppHandle） | 需移除手动 event emit（重复风险） |
| switch_branch | "switch" | 是 | 无 |
| merge_branch | "merge" | 是 | 无 |
| copy_branch_tag | "copy" | 是 | 无 |
| export_workspace | "export" | 是 | 无 |

### 改造 update_workspace（update.rs）

1. 将 `svn::executor::run_svn(...)` 替换为 `svn::executor::run_svn_with_progress(...)`，operation 传 `"update"`
2. **移除**以下手动 event emit（run_svn_with_progress 内部已 emit）：
   - `app_handle.emit("operation:started", ...)`（约第 34 行）
   - `app_handle.emit("operation:completed", ...)`（约第 67 行）
   - `app_handle.emit("operation:error", ...)`（约第 72 行）
3. 移除 `Emitter` import 如果无其他用处
4. cwd 参数使用 `&params.path`（当前工作副本路径）

### 改造 branch_ops.rs

三个函数各追加 `app_handle: tauri::AppHandle` 参数，并将 `run_svn` 改为 `run_svn_with_progress`：

```rust
// 改造 switch_branch:
let result = svn::executor::run_svn_with_progress(
    &args.iter().map(String::as_str).collect::<Vec<&str>>(),
    &params.path,
    params.credentials.as_ref(),
    app_handle,
    "switch",
).await;
```

类似改造 `merge_branch`（operation="merge"）和 `copy_branch_tag`（operation="copy"）。

### 改造 cleanup.rs（export_workspace）

追加 `app_handle: tauri::AppHandle` 参数，改用 `run_svn_with_progress`（operation="export"）。

### stdout 管道关闭标记语义化

**问题：** executor.rs 中用 `filePath` 字段承载 `"[管道中断，部分文件列表可能不完整]"` 文本，语义不够清晰。

**修复方案：**

Rust 端（types.rs）为 `OperationLine` 添加 `is_marker` 字段：

```rust
#[serde(default)]
pub is_marker: bool,
```

TS 端（types/svn.ts）为 `OperationLine` 接口添加 `isMarker` 字段：

```typescript
isMarker?: boolean
```

executor.rs 修改 stdout 关闭标记推送（约 634-640 行），使用 `isMarker: true` 替代 hack filePath：

```rust
ah.emit("operation:line", serde_json::json!({
    "operation": operation_owned,
    "filePath": "",
    "status": "completed",
    "isMarker": true,
})).ok();
```
