## 操作细节

### 概述

当前只有 `checkout_repo` 使用了 `run_svn_with_progress`（试点已验证通过）。需要将其他 4 个写操作命令也改为使用进度推送函数：

| 命令 | 操作名 | 需要追加 AppHandle？ | 特殊注意事项 |
|------|--------|-------------------|-------------|
| update_workspace | "update" | 否（已有 AppHandle） | 需移除手动 event emit（重复风险） |
| switch_branch | "switch" | 否（已有 AppHandle） | 直接调用 run_svn_with_progress |
| merge_branch | "merge" | 否（已有 AppHandle） | 直接调用 run_svn_with_progress |
| copy_branch_tag | "copy" | 否（已有 AppHandle） | 直接调用 run_svn_with_progress |
| export_workspace | "export" | 否（已有 AppHandle） | 直接调用 run_svn_with_progress |

### 改造 update_workspace（update.rs）

1. 将 `svn::executor::run_svn(...)` 替换为 `svn::executor::run_svn_with_progress(...)`，operation 传 `"update"`
2. **移除**以下手动 event emit（run_svn_with_progress 内部已 emit）：
   - `app_handle.emit("operation:started", ...)`（约第 34 行）
   - `app_handle.emit("operation:completed", ...)`（约第 67 行）
   - `app_handle.emit("operation:error", ...)`（约第 72 行）
3. 移除 `Emitter` import 如果无其他用处
4. cwd 参数使用 `&params.path`（当前工作副本路径）

### 改造 branch_ops.rs

三个函数已持有 `app_handle: tauri::AppHandle` 参数（无需追加），直接将 `run_svn` 改为 `run_svn_with_progress`：

```rust
// 改造 switch_branch:
let result = svn::executor::run_svn_with_progress(
    &args.iter().map(String::as_str).collect::<Vec<&str>>(),
    &params.path,
    params.credentials.as_ref(),
    app_handle,     // 注意：此处 app_handle 在调用后被 move，如果 switch_branch 函数后续还需要 app_handle，应使用 clone()
    "switch",
).await;
```

类似改造 `merge_branch`（operation="merge"）和 `copy_branch_tag`（operation="copy"）。

### 改造 cleanup.rs（export_workspace）

`export_workspace` 已持有 `app_handle: tauri::AppHandle` 参数（无需追加），改用 `run_svn_with_progress`（operation="export"）。

### stdout 管道关闭标记语义化

**问题：** executor.rs 中用 `filePath` 字段承载 `"[管道中断，部分文件列表可能不完整]"` 文本，语义不够清晰。

**修复方案：**

Rust 端（types.rs）为 `OperationLine` 添加 `is_marker` 字段：

```rust
#[serde(default)]
pub is_marker: bool,
```

TS 端（types/svn.ts）为 `OperationLine` 接口添加 `isMarker` 字段：

```typescript
isMarker?: boolean
```

executor.rs 修改 stdout 关闭标记推送（约 634-640 行），使用 `isMarker: true` 替代 hack filePath：

```rust
ah.emit("operation:line", serde_json::json!({
    "operation": operation_owned,
    "filePath": "",
    "status": "completed",
    "isMarker": true,
})).ok();
```

该修改从语义上将"管道中断标记"与普通文件行区分开，前端据此判断是否显示"部分文件列表可能不完整"提示。

### create_commit 的可选状态说明

`create_commit` 在此阶段**不做改造**，理由如下：

- commit 为原子操作，Sending/Transmitting 阶段信息量有限（文件数目少）
- 设计文档 v4 §5.2 明确将其列为**可选阶段**
- 如后续需要展示 commit 进度，可按同样模式改造（追加 AppHandle + 改用 run_svn_with_progress，operation="commit"）

### cancel.rs 的改造说明

`cancel_operation` command（`src-tauri/src/commands/cancel.rs`）在当前取消流程中，需要确认取消后推送 `operation:cancelled` 事件而非 `operation:completed` + `result:"cancelled"`：

1. 取消操作执行完成后，emit `operation:cancelled` 事件（使用 `CancelledPayload` 结构体）
2. **不再** emit `operation:completed` + `result:"cancelled"` 的旧模式
3. 保持其他取消逻辑不变（kill 子进程 → cleanup → 重置标志）
