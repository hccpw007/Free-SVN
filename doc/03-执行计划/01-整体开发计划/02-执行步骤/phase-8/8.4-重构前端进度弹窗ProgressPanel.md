# 8.4-重构前端进度弹窗ProgressPanel

| 操作 | 文件 |
|------|------|

## 操作细节

### 整体布局改造

将 ProgressPanel.vue 从旧版简单进度卡片（77 行）重构为完整的非模态进度弹窗，布局如下：

```
┌─ {操作名}进行中 ({百分比}%) ──────────── 可拖拽标题栏
│  ████████████████████████░░░░  85%       进度条（el-progress）
│  ⚡ 2.3 MB/s │ 已完成 47 个 │ ~167 待传输 │ 00:12  统计栏
├──────────────────────────────────────────┤
│  ◆  src/main.rs                     暗淡     │
│  ⟳  src/components/App.tsx          高亮     │  文件滚动区
│  ▸  (等待中)                         正常     │
├──────────────────────────────────────────┤
│  操作完成后自动关闭              [取消中...]│  底栏
└──────────────────────────────────────────┘
```

### 容器样式

- 最外层容器：`fixed inset-0 pointer-events-none z-50`（点击穿透，不阻塞其他页面交互）
- 内层拖拽面板：`pointer-events-auto`（仅在面板区域内可交互）
- 不使用 `bg-black/40` 等全屏遮罩样式
- 使用 `<Teleport to="body">` 渲染到 body 最顶层

### 标题栏

- 内容格式：`{操作名}进行中 ({百分比}%)`，使用 `progress?.operation` 和 `progress?.percent`
- 绑定 composable 的拖拽事件：`@mousedown="onDragStart" @mousemove="onDragMove" @mouseup="onDragEnd"`
- 使用 CSS `transform: translate(x, y)` 配合 composable 的 `panelOffset` 实现拖拽位置
- 拖拽过程中 `pointer-events-auto` 区域限制

### 进度条

- 使用 `<el-progress :percentage="percent" :stroke-width="8" striped striped-flow :duration="0.3" />`
- duration="0.3" 表示 Element Plus 中 300ms 动画
- 百分比来源：`svnStore.progress.percent`

### 统计栏

- 四项信息横排：传输速度 / 已完成文件数 / 待传输估算 / 已耗时间
- 数据来源：`svnStore.progress` 的 speed、completedCount、pendingCount、elapsed 字段
- 数字格式化使用 `formatNumber`（来自 `@/utils/format`）
- 时间格式化使用 `formatElapsed`（来自 `@/utils/format`）
- 窄屏（`< 640px`）自动折叠为两行：第一行（已完成 + 待传输），第二行（速度 + 耗时）
- 使用 Tailwind `sm:` breakpoint 实现

### 文件滚动区

- 循环渲染 `svnStore.fileLines` 数组，使用 `<FileLineRow>` 子组件
- 绑定 `fileListRef` 到滚动容器 `<div>` 的 ref
- 在 `watch` 中监听 `svnStore.fileLines` 长度变化，调用 `autoScrollToBottom()`
- 深色模式背景 `bg-slate-900`，浅色模式 `bg-white`
- 等宽字体 `font-mono`
- 自动滚动逻辑：新行加入时自动滚动到底部；用户手动向上滚动时暂停；滚动到底部附近时恢复

### 底栏

- 左侧文字：`t('progress.autoClose')`（"操作完成后自动关闭"）
- 右侧取消按钮：点击后变为 loading 状态，文案变为 `t('progress.cancelling')`
- 取消按钮样式：红色系（bg-red-50 dark:bg-red-900/20）

### 10 秒无响应超时

- 在 `watch` 中监听 `svnStore.isOperationRunning`：启动后设置 10 秒定时器
- 在 `watch` 中监听 `svnStore.fileLines.length` 或 `svnStore.progress`：收到任何行则重置定时器
- 超时后状态栏显示 `t('progress.connecting')`（"正在连接服务器..."）
- 使用 `onUnmounted` 清理定时器

### 500ms 延迟关闭

- 在 `watch` 中监听 `svnStore.isOperationRunning`：从 true → false 时设置 500ms 延迟再隐藏弹窗 UI
- 延迟期间保持弹窗可见，让进度条的 300ms 动画完成
- 使用 `ref<boolean>` 控制弹窗实际显示

### composable 绑定

使用 `useProgressOverlay()` 获取以下方法/状态：

```typescript
const {
  isDragging, isCancelling, panelOffset,
  onDragStart, onDragMove, onDragEnd,
  cancelOperation, autoScrollToBottom, fileListRef,
  isVisible, progress, fileLines,
} = useProgressOverlay()
```
