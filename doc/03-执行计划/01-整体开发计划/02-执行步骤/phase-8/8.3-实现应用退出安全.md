# 8.3-实现应用退出安全

## 涉及文件

| 操作 | 文件 |

## 操作细节

### executor.rs — 添加 has_child_process 辅助函数

在 executor.rs 中添加：

```rust
pub fn has_child_process() -> bool {
    CURRENT_CHILD.lock().map(|guard| guard.is_some()).unwrap_or(false)
}
```

### lib.rs — 添加 on_window_event CloseRequested 处理

在 `tauri::Builder` 的链式调用中插入 `.on_window_event()`：

```rust
.on_window_event(|window, event| {
    if let tauri::WindowEvent::CloseRequested { api, .. } = event {
        let app = window.app_handle();
        let state: tauri::State<crate::svn::queue::SvnQueue> = app.state();
        
        if state.is_locked() {
            api.prevent_default();
            let handle = app.clone();
            let cwd = crate::config::store::current_workspace()
                .unwrap_or_default();
            tauri::async_runtime::spawn(async move {
                // 1. kill 子进程
                crate::svn::executor::kill_current_process();
                // 2. cleanup
                if !cwd.is_empty() {
                    let _ = crate::svn::executor::run_svn(&["cleanup"], &cwd, None).await;
                }
                // 3. 释放写操作锁
                handle.state::<crate::svn::queue::SvnQueue>().unlock();
                // 4. 重置取消标志
                crate::svn::executor::set_cancelled(false);
                // 5. 退出应用
                handle.exit(0);
            });
        }
    }
})
```

### lib.rs — 修改系统托盘 quit

系统托盘 "quit" 菜单项也应使用相同的安全退出逻辑：

```rust
"quit" => {
    let cwd = crate::config::store::current_workspace().unwrap_or_default();
    let state: tauri::State<crate::svn::queue::SvnQueue> = app.state();
    if state.is_locked() {
        crate::svn::executor::kill_current_process();
        if !cwd.is_empty() {
            let handle = app_handle.clone();
            tauri::async_runtime::spawn(async move {
                let _ = crate::svn::executor::run_svn(&["cleanup"], &cwd, None).await;
                handle.state::<crate::svn::queue::SvnQueue>().unlock();
                crate::svn::executor::set_cancelled(false);
                handle.exit(0);
            });
        }
    } else {
        app.exit(0);
    }
}
```

### 注意事项

- `.on_window_event()` 应在 `.setup()` 之后、`.invoke_handler()` 之前添加。原因：Tauri 2 的 Builder 链式调用顺序有依赖关系——`.setup()` 先注册 setup 回调确保窗口初始化完成，`.on_window_event()` 绑定窗口事件监听器（需要窗口已就绪），`.invoke_handler()` 最后注册命令处理。按此顺序可避免运行时事件绑定失败。
- `tauri::WindowEvent::CloseRequested` 需要 `use tauri::WindowEvent;`
- `api.prevent_default()` 阻止默认关闭行为，异步清理完成后 `handle.exit(0)` 才真正退出
- 注意 `CURRENT_CHILD` Mutex guard 的生命周期，避免锁被异步 task 持有

## 验收标准

- [ ] 窗口 X 按钮点击后，如有操作进行中则先 kill + cleanup + unlock 再关闭
- [ ] 系统托盘 quit 也执行相同安全退出逻辑
- [ ] 无操作时窗口正常关闭，不受影响
- [ ] 编译通过，无 deadlock 风险
