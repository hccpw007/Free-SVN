# 8.2-试点改造 checkout_repo 命令

## 涉及文件

| 操作 | 文件 |
|------|------|
| 修改 | `src-tauri/src/commands/checkout.rs` |

## 操作细节

### 改造 checkout_repo command

将 `checkout_repo` 命令改为调用 `run_svn_with_progress` 替代现有的 `run_svn`：

1. 函数签名不变（已有 `app_handle: AppHandle` 参数）
2. 构造 SVN args 的逻辑不变
3. 将 `svn::executor::run_svn(args, cwd, credentials).await` 改为 `svn::executor::run_svn_with_progress(args, cwd, credentials, app_handle, "checkout").await`
4. 注意：`app_handle` 在调用后可能被 move，但 checkout_repo 原本就持有 AppHandle，只需 clone 后传入

### 参数构造

保持现有参数验证和构造逻辑，仅替换执行调用：

```rust
// 改造前:
let output = svn::executor::run_svn(&all_args, &params.target_path, creds.as_ref()).await?;

// 改造后:
let output = svn::executor::run_svn_with_progress(
    &all_args, &params.target_path, creds.as_ref(), 
    app_handle.clone(), "checkout"
).await?;
```

### 注意事项

- `tauri::State<>` 的使用不受影响
- 不需要修改 verify-credentials 逻辑
- 不需要修改错误处理逻辑（run_svn_with_progress 返回相同的 `Result<String, AppError>`）
- 改造后应检查 `use svn::executor::run_svn` 是否仍被其他代码引用，如无引用则可安全移除该 import，保持代码整洁
