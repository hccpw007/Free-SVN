## 操作细节

### 1.3X — 验证 `vite.config.ts` 的 `@` 路径别名

打开 `vite.config.ts`，确认 `resolve.alias` 包含 `@` 到 `src` 目录的映射：

```typescript
// 期望存在的配置（如果不存在则添加）
resolve: {
  alias: {
    '@': resolve(__dirname, 'src'),
  },
},
```

如果不存在，添加 `import { resolve } from 'path'` 和上述 `resolve.alias` 块。

### 1.3Y — 确认 `tsconfig.app.json` 的严格模式 + 路径映射

打开 `tsconfig.app.json`，确认以下配置已存在（如果不存在则追加）：

```json
{
  "compilerOptions": {
    "strict": true,
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    },
    "resolveJsonModule": true,
    "esModuleInterop": true
  }
}
```

### 1.3Z — 确认 `App.vue` 为纯 RouterView

打开 `src/App.vue`，确保其内容仅为：

```vue
<script setup lang="ts">
import { RouterView } from 'vue-router'
</script>

<template>
  <RouterView />
</template>
```

如果 Tauri 初始化生成了计数器模板内容，全部替换为以上内容。

### 1.3B — 修改 `src/main.ts`

在现有基础上追加 vue-i18n 导入和注册：

```typescript
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import { createI18n } from 'vue-i18n'
import router from './router'
import App from './App.vue'
import './style.css'

import zhCN from './locales/zh-CN.json'
import en from './locales/en.json'
import ja from './locales/ja.json'
import ko from './locales/ko.json'

type MessageSchema = typeof zhCN

const i18n = createI18n<[MessageSchema], 'zh-CN' | 'en' | 'ja' | 'ko'>({
  locale: 'zh-CN',
  fallbackLocale: 'en',
  messages: {
    'zh-CN': zhCN,
    'en': en,
    'ja': ja,
    'ko': ko,
  },
})

const app = createApp(App)
app.use(createPinia())
app.use(router)
app.use(i18n)
app.use(ElementPlus)
app.mount('#app')
```

**注**：如果 vue-tsc 报 JSON 导入类型错误，在 `tsconfig.app.json` 的 `compilerOptions` 中追加 `"resolveJsonModule": true, "esModuleInterop": true`。
