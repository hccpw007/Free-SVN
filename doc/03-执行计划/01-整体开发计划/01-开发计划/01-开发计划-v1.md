### 阶段二：Rust 后端核心基础设施

| 步骤 | 功能目标 | 边界 | 技术 | 方法 |
|------|----------|------|------|------|
| 2.1 | 数据模型层 — models/{error,repo,file}.rs：定义 AppError 枚举（含全部 12 个错误码变体）、RepoInfo 结构体、FileItem 结构体（含 status/path/propertyChanges/fileType binary flag 等全部字段）、OperationProgress/OperationResult 结构体 | 纯数据结构，无方法。为后续 blame/lock 预留字段 | serde + thiserror | 错误码按后端规范 §03 表逐一定义。每个变体 #[error("ERROR_CODE")] 返回 key |
| 2.2 | SVN CLI 执行核心 — svn/executor.rs：run_svn()（async + spawn_blocking + 超时 60s）、CURRENT_CHILD/CANCELLED 全局标志、网络检测 check_network(server_url, 5s 超时 via TcpStream)、validate_path()（空字符检测）| 不涉及具体 SVN 操作。取消标志用于所有长操作 | tokio + std::process::Command | 按后端规范 §04 Executor 完整实现。子进程保存句柄供取消时 kill。网络检测仅服务端操作 |
| 2.3 | XML 解析层 — svn/parser.rs：按 XML 格式分 6 组解析函数——parse_status()、parse_info()、parse_log()、parse_diff（文本模式，非 xml）、parse_blame()、parse_mergeinfo() | diff 使用原始 unified diff 文本传给前端 diff2html，不 XML 解析；blame 输出 svn blame --xml | serde + quick-xml | 每种 svn --xml 输出格式一个对应的反序列化结构体。解析失败统一返回 SVN_PARSE_FAILED |
| 2.4 | 并发控制层 — svn/queue.rs：SvnQueue 结构体（Mutex<Option<JoinHandle>>），写操作互斥锁 + try_lock/unlock。只读操作列表(6种): status/info/diff/log/blame/check_network。写操作列表(18种): 其余所有 | 同一工作副本写操作互斥；不同工作副本不互斥。cancel_operation 绕过锁 | std::sync::Mutex | 按后端规范 §04 Queue + 业务设计 §2.4 并发控制规则实现 |
| 2.5 | 共享类型 — svn/types.rs：OperationProgress（operation/percent/stage/fileCount）、OperationResult（result/detail）前后端同步定义 | 仅供前后端 IPC 协议对齐 | serde (Serialize + Deserialize) | Rust struct + TS interface 同名对照 |
| 2.6 | 配置持久化层 — config/store.rs：tauri-plugin-store 封装全部配置项——settings(8 字段) + recentWorkspaces(上限 20) + windowState。try_deserialize 读取+损坏自动备份 .bak + 重置默认 | 不包含 workspace/path 外的缓存数据 | tauri-plugin-store | 按后端规范 §05.2 Store 结构实现。跨版本升级时缺失字段取默认值 |
| 2.7 | 日志系统 — logging/logger.rs：init() 初始化（轮转 3 文件×5MB）、log_dir 路径（各平台 app_log_dir 标准）、四个日志级别/记录内容。get_logs/export_logs 接口 | 不涉及与 svn 操作日志的关联 | tauri-plugin-log + log crate | 按后端规范 §05.1 实现。app 启动时 setup 中 init |

### 阶段三：Rust Command 层（分四批次注册）

#### 批次 3A — 核心只读操作

| 步骤 | 功能目标 | 边界 | 技术 | 方法 |
|------|----------|------|------|------|
| 3A.1 | commands/status.rs：get_status(path) → 文件变更列表（FileItem[]）、get_info(path) → 工作副本信息（RepoInfo） | status 纯本地，不检查网络。info 需网络(获取 URL 等远程信息)时检测 | svn status --xml + svn info --xml | 参数校验 → executor.run_svn() → parser.parse_status/info() |
| 3A.2 | commands/diff.rs：get_diff(path, revision1?, revision2?) → 差异文本 | 二进制文件返回 binary=true 标识，不返回 diff 内容 | svn diff + MIME 检测 | 检测文件 MIME 类型。二进制文件的差异按钮前端仅显示提示 |
| 3A.3 | commands/log.rs：get_log(path, limit?, revision?, search?) → 日志条目列表（每项含变更文件列表）| svn log --search 仅搜索提交信息。limit 默认 50 | svn log --xml | --xml + --verbose 获取变更文件列表。search 参数通过 --search 传递 |
| 3A.4 | commands/log.rs（追加）：get_blame(path, revision?) → BlameLine[]（每行: 版本+作者+日期+行号+代码内容）| 大文件性能考虑：设置单独超时（120s） | svn blame --xml | 与 log 共享同一文件 |

#### 批次 3B — 核心写操作（长操作可取消）

| 步骤 | 功能目标 | 边界 | 技术 | 方法 |
|------|----------|------|------|------|
| 3B.1 | commands/checkout.rs：checkout_repo(url, targetPath, depth, ignoreExternals) → 检出结果+版本号 | 长操作可取消。需网络检测+queue.try_lock | svn checkout | Tauri event 推送进度(operation:started/progress/completed)；循环解析 svn 行输出推 percent |
| 3B.2 | commands/update.rs：update_workspace(path, revision?, depth?, ignoreExternals?) → 更新结果（新版本号+冲突列表） | 长操作可取消。需网络检测+queue.try_lock | svn update --xml | 同 checkout 进度推送模式 |
| 3B.3 | commands/commit.rs：create_commit(paths[], message, keepLocks) → 新版本号 | 原子操作不可取消。需网络检测+queue.try_lock | svn commit | 构造多文件路径 args；返回新 revision |
| 3B.4 | commands/file_ops.rs：add_files(paths[])、delete_files(paths[], keepLocal?)、revert_files(paths[])、resolve_conflict(path, resolution) → 各操作结果 | 批量文件路径数组（非单个文件）。resolve 接受 'mine-full'/'theirs-full'/'working' 等 | svn add/delete/revert/resolve | 每个操作前 queue.try_lock。批量处理返回逐文件结果 |
| 3B.5 | commands/ignore.rs：set_ignore(path, pattern) → 忽略结果 | 调用 svn propget 读取现有 svn:ignore + 追加新 pattern → propset 写入 | svn propget + propset | 先获取已有 ignore 列表，追加后写回 |

#### 批次 3C — 剩余功能操作

| 步骤 | 功能目标 | 边界 | 技术 | 方法 |
|------|----------|------|------|------|
| 3C.1 | commands/branch_ops.rs：switch_branch(path, targetUrl, ignoreAncestry?) → 切换结果 | 长操作可取消。需网络检测+queue.try_lock | svn switch | 同 update 长操作模式 |
| 3C.2 | commands/branch_ops.rs（追加）：copy_branch_tag(srcUrl, dstUrl, message, revision?) → 复制结果 | 需网络检测+queue.try_lock | svn copy | 与 switch 共享同一个文件 |
| 3C.3 | commands/branch_ops.rs（追加）：merge_branch(srcUrl, revStart, revEnd, targetPath) → 合并结果（冲突文件列表） | 长操作可取消。需网络检测+queue.try_lock | svn merge --xml | 合并完成后读取 svn status 获取冲突列表返回 |
| 3C.4 | commands/cleanup.rs：cleanup_workspace(path) + export_workspace(path, targetDir, revision?, ignoreExternals?) | export 长操作可取消。cleanup queue.try_lock | svn cleanup + svn export | export 同 checkout 进度推送模式 |
| 3C.5 | commands/lock.rs：lock_files(paths[], message?) + unlock_files(paths[]) → 各操作结果 | queue.try_lock | svn lock/unlock | 批量路径处理 |
| 3C.6 | commands/relocate.rs：relocate_repo(path, fromUrl, toUrl) → 重定位结果 | queue.try_lock | svn relocate | 参数校验 URL 格式 |
| 3C.7 | commands/ignore.rs（追加）：property_ops(path, propName?, action?) → 属性列表或操作结果 | 增删查三种 action | svn proplist/propget/propset | 与 ignore 共享同一文件 |

#### 批次 3D — 系统操作

| 步骤 | 功能目标 | 边界 | 技术 | 方法 |
|------|----------|------|------|------|
| 3D.1 | commands/cancel.rs：cancel_operation() — 设取消标志 → kill 子进程 → svn cleanup → 重置取消标志 | 特殊操作不经过 queue。清理过程本身不可取消 | AtomicBool + Child::kill | 按业务设计 §2.5 完整取消流程。完成后 emit 'operation:completed' |
| 3D.2 | commands/logs.rs：get_logs() → 日志字符串内容 + export_logs(target_path) → 导出 zip 到指定路径 | get_logs 返回纯文本；export 打包日志+应用版本+系统信息 | std::fs + zip crate | 设置关于页菜单入口 |
| 3D.3 | lib.rs 注册全部 command 到 generate_handler!：按后端规范 §02 命令清单顺序，26 个 command 全部注册 | 编译验证每个 command 可正确发现 | lib.rs | 整理后提交，确保 cargo build 通过 |

### 后端多批次开发的理由

- **3A(只读操作)** 先完成：前端核心功能（显示文件列表/差异/日志）依赖只读操作
- **3B(核心写操作)** 紧随其后：核心工作流（检出→变更→提交→更新）依赖写操作
- **3C(剩余操作)** 最后开发：switch/branch-tag/merge/export/lock 等功能独立，不影响核心链路
- 每个批次完成后可独立测试
