## 1. 总体评价

交互与视觉设计 v2 文档在纯 UI/UX 设计层面质量很高——色彩体系、字体排版、间距系统、暗色模式等设计系统定义完整，页面线框图细致到每个交互状态和颜色值，Element Plus 组件使用约定明确。

## 2. 严重问题

以下问题必须修复，否则 AI 编码会出现方向性错误。

### 2.1 CSS 变量实现方式与开发规范直接冲突

**问题**：视觉设计文档 §5.2 给出的 CSS 变量双主题方案：

```css
:root { --bg-primary: #F8FAFC; }
.dark { --bg-primary: #0F172A; }
```

与 `01-Vue前端开发规范.md` 第 04 节 "核心规则" 冲突——规范明确规定 **仅使用 Tailwind CSS，禁止 `<style>` 标签或任何自定义 CSS 文件**。

CSS 变量要么写在 `<style>` 标签中，要么写在 `.css` 文件中——两者都被禁止。Tailwind CSS 4 的 `@theme` 指令虽然可以在 `style.css` 中定义 Token，但 `.dark .card { @apply bg-slate-800; }` 这类写法的确需要自定义 CSS。

> **影响范围**：全部页面的主题色、卡片色、输入框背景色等约 30 处使用了 CSS 变量的元素。
>
> **修复建议**：二选一——
> - 方案 A：仅使用 Tailwind `dark:` 变体（如 `class="dark:bg-slate-800"`），完全不用 CSS 变量
> - 方案 B：放宽规范，允许在 `src/style.css` 中通过 Tailwind `@theme` / `@layer` 定义设计 Token，仍禁止独立 `<style>` 标签

### 2.2 页面/路由架构与业务设计不一致

| 项目 | 业务设计 | 视觉设计 | 分歧说明 |
|------|---------|---------|---------|
| 欢迎页 vs 主视图 | `HomePage.vue` 负责"空状态条件渲染：有工作副本→文件列表，无→欢迎页" | `HomePage.vue`（欢迎页）+ `WorkspacePage.vue`（文件列表）两个独立页面 | **架构分歧** |
| Diff 视图 | `DiffPage.vue` 路由 `/workspace/diff?file=xxx` | 仅 `DiffViewer.vue` 组件（底部面板），无独立路由 | **路由缺失** |
| 冲突解决 | `ResolvePage.vue` 路由 `/workspace/resolve?file=xxx` | `ResolveDialog.vue` 对话框组件 | **形态不同** |

三个差异意味着 AI 按视觉设计编码后，文件结构和路由会与业务设计的路由定义矛盾。**必须先在两个设计文档之间统一**。

### 2.5 字体加载方案存在离线兼容和规范双重问题

**问题一：Google Fonts CDN `@import` 不适用于桌面离线场景**

视觉设计 §1.4 的字体加载方案：

```css
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap');
@import url('https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@400;500&display=swap');
```

Free-SVN 是 Tauri 桌面应用，用户可能离线操作 SVN。CDN 加载字体在无网络时失败，fallback 到系统字体（macOS 上的 Times/Helvetica），字号和行高不同将打乱整个排版。

> **影响范围**：所有页面的标题/正文/代码/状态栏文字——约 10 个排版层级全部依赖在线加载。

**问题二：`@import` 写法违反 Vue 开发规范**

`@import url(...)` 必须写在 `<style>` 标签或 `.css` 文件中，而 Vue 规范明确禁止两者。

**问题三：字号使用 px 而非 Tailwind 类，且与自身无障碍要求矛盾**

视觉设计 §1.4 全部字号使用 px（32px / 20px / 16px / 14px / 13px / 12px），但 §6 无障碍设计明确要求"使用 rem/em 而非 px"。自身要求冲突。

**问题四：缺少字体 fallback 栈**

仅指定字体名（Inter / JetBrains Mono），未提供 fallback。正确写法应为：

```css
font-family: 'Inter', system-ui, -apple-system, 'Segoe UI', Roboto, sans-serif;
font-family: 'JetBrains Mono', 'Fira Code', 'Cascadia Code', 'Source Code Pro', Consolas, monospace;
```

**问题五：缺少字号到 Tailwind 类的映射**

所有 px 值都可以映射到 Tailwind 内置类，但文档未做映射，导致 AI 需要自行换算：

| 约定层级 | 设计 px | Tailwind 类 |
|---------|---------|------------|
| Display | 32px | `text-3xl` |
| H1 | 20px | `text-xl` |
| H2 / Body | 16px / 14px | `text-base` / `text-sm` |
| Body Small / Mono / Label | 13px | `text-sm` |
| Mono Small / Tooltip | 12px | `text-xs` |

> **修复建议**：
> - 方案 A：将字体文件内嵌打包到 Tauri 资源中，通过 `@font-face` 本地加载（推荐）
> - 方案 B：使用 Tailwind `font-sans` / `font-mono` 类+系统字体栈，完全不需要额外加载
> - 所有字号改用 Tailwind 类（`text-sm`、`text-base` 等）
> - 补充完整的 fallback 字体栈

## 3. 中等问题

虽不影响编码启动，但影响交付质量。

## 5. AI 可编程性评估

### 5.1 各维度评分

| 维度 | 评分 | 依据 |
|------|------|------|
| 设计系统完整性 | A | 色彩、间距、圆角、阴影、图标、动画完整定义；但字体加载方案（CDN @import）有离线兼容和规范冲突问题 |
| 页面线框图详细度 | A- | ASCII 图细到每个状态行和颜色值；合并向导步骤 2-4 缺失、忽略/确认对话框无线框图 |
| 交互状态覆盖 | A | hover / disabled / selected / loading / empty 均有覆盖 |
| 与业务设计一致性 | C | 4 处架构分歧 §2.2、5 处命名分歧 §2.3、2 处功能遗漏 §2.4、对话框输入项差异多处 §3.7-3.11、合并向导步骤计数矛盾 §3.9 |
| 技术可实现性 | B- | CSS 变量与 Tailwind-only 规范矛盾未解决；Google Fonts CDN 不适用于离线桌面场景 |
| 操作反馈覆盖 | C | 视觉设计只覆盖了 8 种操作反馈中的 8 种，但全部 21+ 种 SVN 操作反馈中约 11 种缺失 |

### 5.2 AI 编码关键风险

1. **编码规范冲突**：AI 读到视觉设计中的 CSS 变量方案 → 再读到 Vue 规范禁止 `<style>` → 两条指令都能触发 AI 执行。必须以"规范优先"或"设计优先"固定其中之一。
2. **字体离线不可用**：Google Fonts CDN `@import` 在桌面离线场景会失败，AI 按设计使用 px + CDN 字体将导致排版异常。必须改为内嵌字体或系统字体栈 + Tailwind 类。
3. **文件结构二选一**：AI 无法同时遵循业务设计的 §9.1 文件结构和视觉设计的 §7 组件树——存在 5 处命名差异 3 处架构差异。Plan 阶段必须选定一套。
4. **遗漏功能的编码风险**：导出对话框和更新到版本对话框在视觉设计中缺失，AI 按视觉设计生成的 Plan 不会包含这两个组件。
5. **操作反馈代码缺失**：约 11 种 SVN 操作的操作反馈在视觉设计中未定义，按视觉设计生成的代码将缺少这些反馈的 UI 部分。

### 5.3 执行步骤可贯彻性

视觉设计在 **样式层面** 足够细致（颜色色值/字号/间距/动画参数均有数值），因此 Plan 中的样式描述可以直接引用视觉设计的 Token，无需 AI 自行判断。

但在 **行为逻辑层面**，状态传递规则（提交选择状态）、条件显示逻辑（批量还原按钮）、功能边界提示（日志搜索）等信息的缺失，需要 Plan 步骤补充细节。否则 AI 要么写错、要么做出与业务设计不一致的实现。
