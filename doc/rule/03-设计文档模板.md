# 03-设计文档模板

每个 `doc/design/` 下的设计文件必须按此模板填写，每项缺一不可。

## 功能定义

每个功能点必须表格形式定义以下字段：

| 字段 | 内容要求 |
|------|----------|
| 输入 | 参数名、类型、取值范围、来源 |
| 输出 | 返回值结构、字段说明、成功/错误格式 |
| 预期行为 | 串行步骤描述（先做什么、再做什么、条件分支） |
| 边界处理 | 空数据、超时、权限不足、非法输入的分别处理方式 |

## 文件拆分方案

列出该功能涉及的所有文件路径和职责，按分层结构组织：

### 前端

```
src/pages/           # [页面文件路径] - [职责]
src/components/      # [组件文件路径] - [职责]
src/stores/          # [Store 文件路径] - [职责]
src/services/        # [Service 文件路径] - [职责]
src/types/           # [类型文件路径] - [职责]
src/router/          # [路由配置路径] - [要添加/修改的路由]
```

### 后端

```
src-tauri/src/commands/  # [命令文件路径] - [职责]
src-tauri/src/services/  # [服务文件路径] - [职责]
src-tauri/src/models/    # [模型文件路径] - [职责]
```

每个文件必须说明是新建还是修改。拆分方案须遵循 `doc/rule/01-Vue前端开发规范.md` 和 `doc/rule/02-Rust后端开发规范.md` 中的行数警戒线和拆分规则。
