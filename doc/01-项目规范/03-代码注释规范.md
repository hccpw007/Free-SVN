# 多语言代码注释规范

覆盖前端（TypeScript / Vue / React）和后端（Rust / Java）。

## 01-总则

1. **功能变更同步更新注释** — 过期注释比没注释更危险
2. **不言自明的代码不注释** — 变量名已表达用途、循环索引、初始化值自说明时省略
3. **注释"为什么"而非"是什么"** — 代码表达"做了什么"，注释表达"为什么要这么做"
4. **不注释掉代码** — 不需要直接删，git 可找回

## 02-核心规则（语言无关）

| 元素 | 注释位置 | 说明 |
|------|----------|------|
| Class / Struct / Interface | 上方 | `// 描述`，较长可用 `/* */` |
| 函数 / 方法 | 上方 | 有参数/返回值时说明用途 |
| 变量 / 常量 | 上方 | 复杂逻辑/标志位/魔法数字/配置值**必须**注释 |
| `if` | 正上方 | `// 条件` |
| `else` / `else if` | 右侧同行 | `} else { // 条件` |
| `switch` / `match` | 正上方 | `// 条件` |
| `for` / `while` | 正上方 | `// 条件` |
| `try` | 正上方 | `// 描述` |
| `catch` | 右侧同行 | `} catch { // 条件` |
| Rust `Err` 分支 | 右侧同行 | `Err(e) => { // 原因` |
| 枚举变体 | 上方或右侧 | `// 描述` |
| Vue Template 区块 | 上方 | `<!-- 功能块描述 -->` |
| `v-if` / `v-else` | 正上方 / 右侧同行 | `<!-- 条件 -->` |
| 按钮 / 事件处理 | 上方 | `<!-- 点击后的操作 -->` |
| React JSX | `{/* */}` | 规则同上 |

注释前空行规则：`//` 上一行如果是同级代码（非注释/空白），上方必须空行。连续多条注释只需第一条前空行。

## 03-各语言注释格式

| 场景 | TypeScript | Rust | Java |
|------|-----------|------|------|
| 行内 | `//` | `//` | `//` |
| 公开函数 | `//` 或 `/** */` | `///` | `/** */` |
| 私有函数 | `//` | `//` | `//` |
| Class / Struct | `//` 或 `/** */` | `///` | `/** */` |
| 模块/文件 | `//` | `//!` | `/** */` |
| 接口 / Trait | `//` 或 `/** */` | `///` | `/** */` |
| 参数说明 | `@param` 或行内 | `/// # 参数` | `@param` |
| 返回值说明 | `@returns` 或行内 | `/// # 返回` | `@return` |

### TypeScript 补充

- 默认 `//`，参数较多（≥3）时允许 `/** */` doc block
- Vue 每个 `.vue` 文件 `<script setup>` 顶部注释组件功能
- Vue `<template>` 中每个功能块上方用 `<!-- -->` 注释
- React JSX 中用 `{/* */}`
- Pinia store 定义上方加注释
- 所有 `ref()` / `reactive()` / `useState()` 上方加注释

### Rust 补充

- 公开函数/struct/enum/trait: `///` doc comment
- 模块级文档: `//!`（mod.rs 或文件顶部）
- 私有函数/变量/控制流: `//` 行内注释
- 结构体字段: 每个字段用 `///` 单独注释
- 可省略：getter 方法、trait 实现的显而易见方法、自文档化测试函数名

### Java 补充

- 公开 API: `/** */` Javadoc（含 `@param`、`@return`、`@throws`）
- 私有方法: `//` 行内注释
- 可省略：标准 getter/setter、逻辑简单的 `@Override` 方法、自文档化 `@Test` 方法

## 04-常见反模式

| 反模式 | 正解 |
|--------|------|
| 注释与代码重复（噪声） | 省略，不写废话 |
| `if` 注释在同行 / `else` 注释在上方 | 修正位置 |
| 控制流无注释 | 每个控制流加注释说明条件 |
| 注释掉的代码 | 直接删，git 可找回 |
| 修改代码不更新注释 | 修改时同步更新 |
| Vue Template 无 `<!-- -->` 注释 | 按 05 细则逐级检查补充 |

## 05-Vue Template 注释细则 — AI 编码规则

**核心原则：** 编写或修改 `<template>` 时，逐元素检查 L1-L4 是否缺少注释，有则补。

| 层级 | 需注释的元素 | 注释格式 | 示例 | 豁免规则 |
|------|-------------|----------|------|---------|
| **L1-区块** | 顶层 `div` / `<template>` / `<Transition>` | `<!-- 功能块名 -->` | `<!-- 左侧 Tab 列表 -->` | 单区块 <3 行可省 |
| **L2-控制流** | `v-if` / `v-else` / `v-for` | `<!-- 条件说明 -->` | `<!-- 无工作副本时显示应用名 -->` | — |
| **L3-组件** | `el-*` / 自定义组件 | `<!-- 用途 -->` | `<!-- 文件列表表格 -->` | — |
| **L4-交互** | `<button>` / `<a>` / `<input>` | `<!-- 点击/输入后的动作 -->` | `<!-- 开始合并操作 -->` | 纯展示图标/文本可省 |
| **L5-纯展示** | `<span>` / `<p>` / `<svg>` | **省略** | — | 不言自明时跳过 |

**示例模板（直接套用）：**
```vue
<!-- 顶部导航条 -->
<div class="...">
  <!-- 工作副本时显示路径 -->
  <template v-if="currentPath">...</template>
  <!-- 无工作副本时显示应用名 -->
  <template v-else>...</template>
</div>

<!-- 操作栏 -->
<div class="...">
  <!-- 刷新按钮 -->
  <button @click="refresh">{{ t('common.refresh') }}</button>
</div>
```

**编码后自查：**
```
□ L1-L4 每个元素正上方有 <!-- --> 注释？
□ L5 省略是真的不言自明？
□ 注释在正上方（非同行、非下方）？
□ 写的是"为什么"而非"是什么"？
```
